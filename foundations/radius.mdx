Radius defines the corner rounding system across the TARMAC Design System. It establishes how shape, geometry, and softness are applied to components, creating a consistent visual character that feels functional, human, and approachable.

Rounded corners help differentiate interaction layers, signal affordance, and improve scalability, while maintaining structural clarity across tools and workflows.

## Radius Tokens

| Token | Value | Purpose |
|-------|-------|---------|
| `radius.none` | 0px | Full sharp ends — no rounding |
| `radius.small` | 2px | Small detail elements: badges, checkboxes, status indicators, keyboard hints |
| `radius.default` | 4px | Labels, tags, timestamps, tooltips, compact buttons, small imagery |
| `radius.medium` | 8px | Buttons, inputs, selects, navigation items, cards, panels, dropdown menus, floating UI |
| `radius.large` | 12px | Large layout containers: modals, tables, dashboards, in-page sections |
| `radius.xlarge` | 16px | Rich media containers and immersive content |
| `radius.max` | 999px | Fully circular elements: avatars, user tokens, reaction icons |

| Component | Token | Value | Notes |
|-----------|-------|-------|-------|
| Checkbox, Badge | `radius.small` | 2px | Tight, precise corners |
| Tag, Label, Tooltip | `radius.default` | 4px | Subtle rounding |
| Button, Input, Select | `radius.medium` | 8px | Primary interactive elements |
| Card, Panel, Dropdown | `radius.medium` | 8px | Container-level rounding |

## Principles

<AccordionGroup>
  <Accordion title="Functional Softness">
    Corners are rounded just enough to feel approachable, without sacrificing precision.
  </Accordion>
  <Accordion title="Hierarchy Through Shape">
    Different radius values communicate different levels of interaction, containment, and importance.
  </Accordion>
  <Accordion title="Consistency Over Expression">
    The same semantic intent applies everywhere — not arbitrarily.
  </Accordion>
  <Accordion title="Accessibility Through Recognition">
    Repeated shape language helps users recognise patterns faster.
  </Accordion>
  <Accordion title="Scale with Purpose">
    Larger elements use larger radii to maintain proportion and visual balance.
  </Accordion>
</AccordionGroup>

## Accessibility and Readability

Corner radius contributes to accessibility by:

- Improving perceptual grouping
- Making interactive elements easier to identify
- Reducing visual harshness in dense UI
- Supporting familiarity across repeated patterns

Consistent geometry helps users quickly understand what is clickable, what is content, and what is structural.

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check">
    - Use semantic tokens (`radius.medium`) instead of raw values
    - Match radius to component scale — larger elements get larger radii
    - Keep radius consistent within the same component family
    - Use `radius.max` only for intentionally circular elements
  </Card>
  <Card title="✗ Don't" icon="xmark">
    - Don't use arbitrary radius values outside the token scale
    - Don't mix different radii on the same component
    - Don't apply large radii to small elements (looks distorted)
    - Don't use `radius.none` on interactive elements unless intentional
  </Card>
</CardGroup>

## Implementation

<CodeGroup>
```css CSS Variables
/* Radius tokens */
--radius-none: 0px;
--radius-small: 2px;
--radius-default: 4px;
--radius-medium: 8px;
--radius-large: 12px;
--radius-xlarge: 16px;
--radius-max: 999px;
```

```tsx TypeScript
// Radius token values
export const radius = {
  none: '0px',
  small: '2px',
  default: '4px',
  medium: '8px',
  large: '12px',
  xlarge: '16px',
  max: '999px',
} as const;
```

```tsx Usage
// Button with medium radius
<Button style={{ borderRadius: 'var(--radius-medium)' }}>
  Submit
</Button>

// Avatar with max radius (circular)
<Avatar style={{ borderRadius: 'var(--radius-max)' }} />

// Card with large radius
<Card style={{ borderRadius: 'var(--radius-large)' }}>
  Dashboard content
</Card>
```
