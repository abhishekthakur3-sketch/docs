## Overview

The Tarmac spacing system creates a consistent visual rhythm across products, components, and layouts. Built on an **8px base unit**, spacing establishes clarity, hierarchy, and usability by ensuring elements relate to one another in a predictable way.

<CardGroup cols={3}>
  <Card title="For Designers" icon="pen-ruler">
    Use Figma spacing tokens mapped 1:1 to these values. Never use arbitrary pixel values.
  </Card>
  <Card title="For Developers" icon="code">
    Reference semantic CSS variables or TypeScript token exports. Never hardcode spacing values.
  </Card>
  <Card title="For Product" icon="users">
    Ensures layout consistency, accessibility compliance, and scalable spacing across all products.
  </Card>
</CardGroup>

### 8px Base Unit

All spacing values are multiples of 8px — providing flexibility without fragmentation. The core unit defines the spacing scale, ensuring every screen, component, and interaction maintains consistent visual balance.

<Info>The 8px grid aligns with common device resolutions, simplifies developer handoff, and reduces decision fatigue when choosing spacing values.</Info>

## Size Ranges

<AccordionGroup>
  <Accordion title="Small (0–8px)" icon="compress">
    Use for tight, internal spacing within compact elements:
    - Spacing between small icons and text
    - Inner padding in compact components (badges, icon buttons, table cells)
    - Gaps between repeating UI elements (chips, tag groups, button groups)
    - Form field padding and internal content spacing
    - Tight vertical gaps inside cards and containers
    - Spacing between triggers and elevated elements (menus, dropdowns, tooltips)
  </Accordion>
  <Accordion title="Medium (12–32px)" icon="arrows-left-right">
    Use for standard component and content spacing:
    - Padding inside primary components (buttons, cards, list items)
    - Spacing between content groups within a component
    - Separation between icons/avatars and text
    - Spacing between stacked UI sections
    - Layout spacing in standard page content
  </Accordion>
  <Accordion title="Large (40–80px)" icon="expand">
    Use for structural and section-level spacing:
    - Spacing between major page sections
    - Page top-margins and structural dividers
    - Hero areas and large modules
    - Wide layout breathing room
    - Guiding visual flow across dense dashboards
  </Accordion>
</AccordionGroup>

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#1BA86E">
    - Always use spacing tokens (`space.16`, `space.24`)
    - Follow the 8px grid for all margins and padding
    - Use smaller values (0–8px) to group related items
    - Use larger values (40–80px) to separate major sections
    - Maintain consistent spacing for the same patterns
    - Ensure interactive elements have adequate touch targets (44px min)
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC143C">
    - Don't use arbitrary values (`margin: 13px`, `padding: 7px`)
    - Don't over-space — too much space breaks visual relationships
    - Don't under-space — cramped layouts reduce usability
    - Don't hardcode pixel values in components — use tokens
    - Don't mix spacing systems across the same product
    - Don't rely on spacing alone to convey meaning
  </Card>
</CardGroup>

## Component Spacing

Recommended spacing for common component patterns.

| Component | Internal Padding | Gap Between Items | Margin Bottom |
|-----------|-----------------|-------------------|---------------|
| Button | `space.12` | `space.8` | `space.16` |
| Input | `space.12` | `space.8` | `space.16` |
| Card | `space.24` | `space.16` | `space.24` |
| Modal | `space.24` | `space.16` | — |
| List Item | `space.16` | `space.8` | `space.4` |
| Section | `space.32` | `space.24` | `space.48` |
| Badge | `space.4` | `space.4` | — |
| Tooltip | `space.8` | `space.4` | — |

<Tip>When in doubt, use `space.16` (16px) as your default spacing value — it works well for most component padding and content gaps.</Tip>

## Principles

<Steps>
  <Step title="Use spacing to define hierarchy, not decoration">
    Spacing communicates structure. Larger gaps signal separation between sections; tighter gaps signal grouping. Let spacing do the work of organizing content.
  </Step>
  <Step title="Whitespace is functional, not empty space">
    Every unit of whitespace serves a purpose — improving readability, guiding the eye, or creating breathing room. Treat it as a design element, not leftover area.
  </Step>
  <Step title="Consistency improves learnability">
    When the same spacing patterns repeat across screens, users build mental models faster. Predictable layouts reduce cognitive effort.
  </Step>
  <Step title="Spacing should support clarity at scale">
    Dense dashboards, data tables, and complex workflows all benefit from systematic spacing. The scale is designed to work from compact UI to expansive layouts.
  </Step>
</Steps>

## Accessibility

Consistent spacing improves legibility, comprehension, and interaction accuracy.

<AccordionGroup>
  <Accordion title="Element Separation" icon="grip-lines">
    Maintain clear separation between interactive and content elements. Adequate spacing prevents misclicks and improves scanning in dense interfaces.
  </Accordion>
  <Accordion title="Touch Target Spacing" icon="hand-pointer">
    Interactive elements should be at least 44×44px with a minimum of `space.8` (8px) between adjacent targets. This ensures reliable touch and pointer interaction.
  </Accordion>
  <Accordion title="Reading Flow" icon="book-open">
    Consistent vertical rhythm enhances scan patterns and reading flow. Use predictable spacing between paragraphs, headings, and list items to reduce cognitive load.
  </Accordion>
  <Accordion title="Visual Overload" icon="eye">
    Prevent visual overload in dense workflows by using the medium and large spacing ranges to create breathing room between content groups and page sections.
  </Accordion>
</AccordionGroup>

<Warning>Never reduce spacing below `space.8` (8px) between interactive elements. Tight spacing between buttons, links, or form controls creates accessibility barriers for users with motor impairments.</Warning>

    ```css
    :root {
      --space-0: 0px;
      --space-2: 2px;
      --space-4: 4px;
      --space-6: 6px;
      --space-8: 8px;
      --space-12: 12px;
      --space-16: 16px;
      --space-20: 20px;
      --space-24: 24px;
      --space-32: 32px;
      --space-40: 40px;
      --space-48: 48px;
      --space-64: 64px;
      --space-80: 80px;
    }

    /* Usage */
    .card {
      padding: var(--space-24);
      gap: var(--space-16);
      margin-bottom: var(--space-24);
    }

    .button {
      padding: var(--space-12) var(--space-16);
      gap: var(--space-8);
    }

    .section {
      padding: var(--space-48) 0;
      margin-bottom: var(--space-64);
    }
    ```
