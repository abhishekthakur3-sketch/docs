# Spacing

## Overview

The TARMAC Design System uses a consistent spacing scale based on an 8-point grid system. This creates visual rhythm, improves scanability, and ensures consistent spacing across all components and layouts.

## Usage Guidelines

### Do's and Don'ts

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#10b981">
    **Use the spacing scale**
    - Reference spacing tokens
    - Maintain consistency
    - Follow 8px grid
    
    **Create visual hierarchy**
    - More space = more importance
    - Group related items closely
    - Separate sections clearly
    
    **Be consistent**
    - Same spacing for same patterns
    - Predictable layouts
    - Systematic approach
  </Card>
  
  <Card title="✗ Don't" icon="xmark" color="#ef4444">
    **Don't use arbitrary values**
    - ~~`margin: 13px`~~
    - ~~`padding: 7px`~~
    - Stick to the scale
    
    **Don't over-space**
    - Too much space breaks relationships
    - Maintain visual grouping
    
    **Don't under-space**
    - Ensure touch targets (44px min)
    - Provide breathing room
    - Avoid cramped layouts
  </Card>
</CardGroup>

## Layout Principles

### The 8-Point Grid

<Info>All spacing values are multiples of 8px, creating a consistent and predictable layout system.</Info>

**Benefits:**
- Easier to scale designs
- Consistent visual rhythm
- Simplifies developer handoff
- Works well with common screen sizes
- Reduces decision fatigue

**How to Apply:**
1. Use spacing tokens for all margins and padding
2. Align elements to the 8px grid
3. Size components in multiples of 8px when possible
4. Use 4px (spacing-1) only for tight spacing needs

### Visual Hierarchy

Use spacing to create clear visual hierarchy:

<Steps>
  <Step title="Group related elements">
    Use smaller spacing (4-8px) between related items
  </Step>
  <Step title="Separate sections">
    Use larger spacing (32-48px) between major sections
  </Step>
  <Step title="Create breathing room">
    Use medium spacing (16-24px) for comfortable reading
  </Step>
  <Step title="Emphasize importance">
    More space around an element = more visual weight
  </Step>
</Steps>

### Responsive Spacing

Adjust spacing for different screen sizes:

| Breakpoint | Container Padding | Section Spacing | Component Gap |
|------------|------------------|-----------------|---------------|
| Mobile (< 768px) | `spacing-4` (16px) | `spacing-8` (32px) | `spacing-3` (12px) |
| Tablet (768px+) | `spacing-6` (24px) | `spacing-12` (48px) | `spacing-4` (16px) |
| Desktop (1024px+) | `spacing-8` (32px) | `spacing-16` (64px) | `spacing-6` (24px) |

## Accessibility Considerations

<AccordionGroup>
  <Accordion title="✓ Touch Targets" icon="hand-pointer">
    **Minimum Size: 44x44px**
    - Ensure interactive elements are at least 44x44px
    - Provide adequate spacing between touch targets
    - Use `spacing-2` (8px) minimum between buttons
    
    ```css
    .button {
      min-height: 44px;
      min-width: 44px;
      padding: var(--spacing-3) var(--spacing-4);
    }
    
    .button-group {
      gap: var(--spacing-2); /* 8px minimum */
    }
    ```
  </Accordion>
  
  <Accordion title="✓ Reading Comfort" icon="book-open">
    **Line Height and Spacing**
    - Adequate line height (1.5-1.6 for body text)
    - Comfortable paragraph spacing (16px)
    - Clear section breaks (48px+)
    
    ```css
    .text {
      line-height: 1.5;
      margin-bottom: var(--spacing-4); /* 16px */
    }
    
    .section {
      margin-bottom: var(--spacing-12); /* 48px */
    }
    ```
  </Accordion>
  
  <Accordion title="✓ Focus Indicators" icon="bullseye">
    **Visible Focus States**
    - Provide space for focus rings (2-4px)
    - Don't let focus indicators overlap content
    - Use outline-offset for breathing room
    
    ```css
    .interactive:focus {
      outline: 2px solid var(--color-primary-600);
      outline-offset: 2px; /* spacing-1 */
    }
    ```
  </Accordion>
</AccordionGroup>
