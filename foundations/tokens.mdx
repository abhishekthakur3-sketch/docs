# Design Tokens

## Overview

Design tokens are the visual design atoms of the TARMAC Design System. They are named entities that store visual design attributes, ensuring consistency across platforms and enabling systematic design changes.

## Token Naming Convention

### Structure

Tokens follow a consistent naming pattern:

```
[category]-[property]-[variant]-[state]
```

**Examples:**
- `color-background-primary`
- `spacing-4`
- `button-padding-x-md`
- `input-border-focus`

### Naming Rules

<AccordionGroup>
  <Accordion title="✓ Be Descriptive" icon="tag">
    Use clear, meaningful names that describe purpose, not appearance.
    
    **Good:**
    - `color-text-primary`
    - `spacing-section`
    - `button-height-md`
    
    **Bad:**
    - `color-dark-gray`
    - `spacing-big`
    - `button-40px`
  </Accordion>
  
  <Accordion title="✓ Use Hierarchy" icon="sitemap">
    Organize tokens in a logical hierarchy from general to specific.
    
    ```
    color
      ├── neutral
      │   ├── 50
      │   ├── 100
      │   └── ...
      ├── primary
      └── semantic
          ├── success
          ├── warning
          └── error
    ```
  </Accordion>
  
  <Accordion title="✓ Support Theming" icon="palette">
    Use semantic names that work across themes.
    
    **Good:**
    - `background-primary` (adapts to theme)
    - `text-primary` (adapts to theme)
    
    **Bad:**
    - `background-white` (theme-specific)
    - `text-black` (theme-specific)
  </Accordion>
</AccordionGroup>

## Usage Guidelines

### Do's and Don'ts

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#10b981">
    **Use semantic tokens**
    - Reference by purpose, not value
    - `color-text-primary` not `#000000`
    
    **Create aliases**
    - Component-specific tokens
    - `button-bg-primary: {color.primary.600}`
    
    **Document tokens**
    - Explain usage and context
    - Provide examples
    
    **Version tokens**
    - Track changes
    - Maintain backwards compatibility
  </Card>
  
  <Card title="✗ Don't" icon="xmark" color="#ef4444">
    **Don't hardcode values**
    - ~~`color: #000000`~~
    - Use tokens instead
    
    **Don't create one-off tokens**
    - Maintain consistency
    - Reuse existing tokens
    
    **Don't use presentational names**
    - ~~`color-dark-gray`~~
    - Use semantic names
    
    **Don't skip documentation**
    - Always explain token purpose
    - Provide usage examples
  </Card>
</CardGroup>

### Token Aliasing

Create component-specific aliases that reference base tokens:

```json
{
  "button": {
    "primary": {
      "background": { "value": "{color.primary.600}" },
      "backgroundHover": { "value": "{color.primary.700}" },
      "text": { "value": "{color.text.inverse}" },
      "paddingX": { "value": "{spacing.4}" },
      "paddingY": { "value": "{spacing.3}" },
      "borderRadius": { "value": "{borderRadius.md}" }
    }
  }
}
```

**Benefits:**
- Easier to update component styles
- Clear component-token relationship
- Maintains consistency with base tokens

## Platform Implementation

### CSS Custom Properties

```css
:root {
  /* Colors */
  --color-neutral-900: #171717;
  --color-primary-600: #000000;
  --color-text-primary: var(--color-neutral-900);
  
  /* Spacing */
  --spacing-4: 1rem;
  --spacing-6: 1.5rem;
  
  /* Typography */
  --font-size-base: 1rem;
  --font-weight-medium: 500;
  
  /* Component Tokens */
  --button-bg-primary: var(--color-primary-600);
  --button-padding-x: var(--spacing-4);
}

/* Usage */
.button {
  background: var(--button-bg-primary);
  padding: 0 var(--button-padding-x);
}
```

### Sass/SCSS

```scss
// _tokens.scss
$color-neutral-900: #171717;
$color-primary-600: #000000;
$spacing-4: 1rem;

// Component tokens
$button-bg-primary: $color-primary-600;
$button-padding-x: $spacing-4;

// Usage
.button {
  background: $button-bg-primary;
  padding: 0 $button-padding-x;
}
```

### iOS (Swift)

```swift
// DesignTokens.swift
enum DesignTokens {
    enum Color {
        static let neutral900 = UIColor(hex: "#171717")
        static let primary600 = UIColor(hex: "#000000")
        static let textPrimary = neutral900
    }
    
    enum Spacing {
        static let spacing4: CGFloat = 16
        static let spacing6: CGFloat = 24
    }
    
    enum Button {
        static let primaryBackground = Color.primary600
        static let paddingX = Spacing.spacing4
    }
}

// Usage
button.backgroundColor = DesignTokens.Button.primaryBackground
```

### Android (Kotlin)

```kotlin

button.setBackgroundColor(DesignTokens.Button.primaryBackground)
```

## Theme Support

### Light and Dark Themes

```css
:root {
  --color-background-primary: #ffffff;
  --color-text-primary: #171717;
}

[data-theme="dark"] {
  --color-background-primary: #171717;
  --color-text-primary: #fafafa;
}

/* Components automatically adapt */
.card {
  background: var(--color-background-primary);
  color: var(--color-text-primary);
}
```

### Custom Themes

```json
{
  "theme": {
    "light": {
      "background": {

      },

    }
  }
}
```

## Accessibility Considerations

<AccordionGroup>
  <Accordion title="✓ Contrast Ratios" icon="eye">
    Ensure color tokens meet WCAG contrast requirements:
    
    ```json
    {
      "color": {
        "text": {
          "primary": {
            "value": "{color.neutral.900}",
            "contrast": "21:1",
            "wcag": "AAA"
          },
          "secondary": {
            "value": "{color.neutral.600}",
            "contrast": "7:1",
            "wcag": "AAA"
          }
        }
      }
    }
    ```
  </Accordion>
  
  <Accordion title="✓ Touch Targets" icon="hand-pointer">
    Define minimum sizes for interactive elements:
    
    ```json
    {
      "size": {
        "touchTarget": {
          "minimum": { "value": "44px" }
        }
      }
    }
    ```
  </Accordion>
  
  <Accordion title="✓ Motion Preferences" icon="play">
    Respect user motion preferences:
    
    ```css
    @media (prefers-reduced-motion: reduce) {
      :root {
        --duration-normal: 0ms;
        --duration-slow: 0ms;
      }
    }
    ```
  </Accordion>
</AccordionGroup>
