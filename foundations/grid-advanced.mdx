## Responsive Behavior Rules

How layouts adapt across breakpoints. These rules govern what scales, what stays fixed, and how content reflows.

<AccordionGroup>
  <Accordion title="Column Collapse Behavior" icon="arrows-down-to-line">
    As viewport width decreases, columns collapse and content stacks vertically:
    - **Desktop → Tablet**: 12 columns maintained, column width reduces, gutters shrink from 24px to 16px
    - **Tablet → Mobile**: 12 columns collapse to 4 columns, multi-column layouts stack vertically
    - **Side panels**: Collapse into full-width sections or slide-in drawers on mobile
    - **Navigation**: Side nav becomes a hamburger menu overlay on mobile
  </Accordion>
  <Accordion title="What Scales vs What Stays Fixed" icon="ruler-combined">
    | Element | Behavior | Notes |
    |---------|----------|-------|
    | Column width | Scales (fluid) | Proportional to viewport |
    | Gutter width | Fixed per breakpoint | 24px (large) or 16px (default/small) |
    | Margins | Fixed per breakpoint | 16px on mobile, 80px+ on desktop |
    | Content max-width | Fixed at 1280px | Prevents overly wide layouts on 1920+ |
    | Font sizes | Fixed per breakpoint | Controlled by typography tokens, not grid |
    | Touch targets | Fixed minimum 44px | Never scale below minimum |
  </Accordion>
  <Accordion title="Content Stacking Logic" icon="layer-group">
    When columns collapse on smaller viewports, content stacks in reading order (top to bottom):

    1. **Primary content** stacks first (main body, forms, data)
    2. **Secondary content** stacks below (sidebars, filters, metadata)
    3. **Navigation** moves to overlay or bottom bar
    4. **Actions** (buttons, CTAs) remain visible — never hidden by stacking

    <Warning>Never hide critical actions or primary content when stacking. If content doesn't fit, simplify the layout — don't just stack everything.</Warning>
  </Accordion>
</AccordionGroup>

## Desktop Grid System

### Supported Viewports

| Screen Size | Usage |
|-------------|-------|
| 1920+ | Grid locked to 1280px, centered |
| 1440 × 942px | Primary design frame and baseline grid |
| 1366 × 942 | Widespread enterprise resolution |
| 1280 × 832 | Compact laptop and workspace views |

### Content Grid

The base 12-column content grid used across all desktop breakpoints.

| Breakpoint | Viewport | Columns | Gutter | Column Width | Type |
|------------|----------|---------|--------|--------------|------|

| Small | 1280 × 832 | 12 | 16px | 92px | Centre |

<Info>The content grid is always 12 columns and centre-aligned. Column width remains consistent at 92px — only gutters change between Large (24px) and other breakpoints (16px).</Info>

### Accessibility & Readability

Desktop grid decisions directly impact readability and accessibility. The TARMAC grid is designed to:

<Steps>
  <Step title="Maintain consistent reading widths">
    Content stays within a 1440px grid frame to prevent eye-strain on ultra-wide monitors.
  </Step>
  <Step title="Prevent excessive line lengths">
    On screens wider than 1920px, the grid locks to a centered 1280px container.
  </Step>
  <Step title="Support predictable scan patterns">
    Consistent column alignment creates reliable visual anchors for high-density UIs.
  </Step>
  <Step title="Preserve typographic hierarchy">
    Grid structure reinforces heading and body text relationships through consistent alignment.
  </Step>
  <Step title="Ensure comfortable interaction spacing">
    Gutters and margins provide adequate spacing for motor interactions and touch targets.
  </Step>
  <Step title="Balance contrast and whitespace">
    The grid prevents content from becoming too dense or too sparse at any breakpoint.
  </Step>
</Steps>

## Component Mapping

How the grid applies to common UI components across desktop and mobile.

| Component | Desktop Grid Usage | Mobile Grid Usage |
|-----------|-------------------|-------------------|
| Card | Span 3-4 cols, multiple per row | Span 4 cols (full width), stacked |
| Data Table | Span 12 cols, horizontal scroll if needed | Span 4 cols, horizontal scroll |
| Form | Span 6-8 cols (centered or with sidebar) | Span 4 cols, full width |
| Modal / Dialog | Span 6-8 cols (centered overlay) | Span 4 cols (full width sheet) |
| Side Navigation | Fixed width (collapsed: 72-88px, expanded: 240-280px) | Overlay drawer, 3-column grid |
| Top Navigation | Span 12 cols (full width, single row) | Span 2 cols (logo + actions) |
| Filters Panel | Span 3-4 cols (sidebar) or full-width bar | Full width, collapsible accordion |
| Hero / Banner | Span 12 cols (full bleed) | Span 4 cols (full width) |
| Footer | Span 12 cols, multi-column content | Span 4 cols, stacked sections |
| Tabs | Span 12 cols (tab bar), content follows grid | Span 4 cols, scrollable tab bar |
