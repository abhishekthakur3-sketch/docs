Color in the Tarmac Design System expresses brand, hierarchy, and usability. The palette is built around a small set of core colors, extended into semantic tokens for background, text, icon, and border usage. The system separates raw color values from semantic intent across three layers: Core → Semantic → Usage.

<CardGroup cols={3}>
  <Card title="Accessible" icon="universal-access">
    WCAG 2.1 AA enforced at the semantic layer
  </Card>
  <Card title="Three-Layer Architecture" icon="layer-group">
    Core → Semantic → Usage for scalable theming
  </Card>
  <Card title="Theme Agnostic" icon="circle-half-stroke">
    Light and dark modes without remapping UI logic
  </Card>
</CardGroup>

<Warning>Raw core colors should never be used directly in UI. All product usage must go through semantic tokens.</Warning>

## Semantic Token Mapping

Semantic tokens map core colors to purpose-driven roles. Each token has light and dark theme variants.

| Token | Role | Light Theme | Dark Theme |
|-------|------|-------------|------------|
| `surface.background` | Page backgrounds | White scale | Black scale |
| `surface.neutral` | Neutral UI surfaces | Grey scale | Coal scale |
| `surface.accent` | Brand accent surfaces | Red scale | Red scale (shifted) |
| `surface.info` | Informational | Blue scale | Blue scale (shifted) |
| `surface.success` | Positive states | Green scale | Green scale (shifted) |
| `surface.error` | Error states | Crimson scale | Crimson scale (shifted) |
| `surface.warning` | Caution states | Yellow scale | Yellow scale (shifted) |
| `text.heading` | Heading text | Black.200 / Grey.500 | White.450 / White.25 |
| `text.body` | Body text | Black.25 / Grey.350 | White.400 / White.50 |
| `text.caption` | Caption text | Grey.450 / Grey.300 | White.350 / Grey.200 |
| `border.neutral` | Default borders | White.300 / White.200 | Black.200 / Black.100 |

Each semantic category supports intensity levels: Default, Strongest, Stronger, Strong, Solid, Base, Medium, Subtle, Weak, Weaker, Weakest.

<Tip>In dark mode, semantic tokens automatically shift to darker core values. No UI logic changes needed.</Tip>

## Accessibility

<AccordionGroup>
  <Accordion title="Contrast Requirements" icon="eye">
    Body text: WCAG AA (4.5:1) minimum. Large text: 3:1 minimum. Icon and UI elements: 3:1 against background.
  </Accordion>
  <Accordion title="Color Usage Rules" icon="palette">
    Never rely on color alone to convey meaning. Pair with icons, text, or shape. Avoid low-contrast alpha overlays for critical content.
  </Accordion>
  <Accordion title="State Accessibility" icon="hand-pointer">
    Hover, focus, and active states must remain distinguishable. Error and success must not rely solely on red/green.
  </Accordion>
</AccordionGroup>

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#1BA86E">
    Use semantic tokens, never raw core colors in UI. Ensure sufficient contrast ratios. Use neutrals as the foundation. Let accent and status colors emphasize sparingly.
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC143C">
    Don't reference core colors directly in components. Don't use color alone to convey meaning. Don't stack multiple semantic meanings on one element.
  </Card>
</CardGroup>
