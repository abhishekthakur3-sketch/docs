# Colors

## Overview

Color in the Tarmac Design System expresses brand, hierarchy, and usability. The palette is built around a small set of core colors, extended into semantic, background, text, icon, border, and alpha tokens to create consistency across all products. Use color purposefully: reinforce meaning, support accessibility, and maintain visual clarity.

  <Card title="Accessible" icon="universal-access">

  </Card>

## Color Architecture

The system is built in three layers:

1. Core (Raw) Colors — Foundational color values
2. Semantic Colors — Meaning-driven mapping
3. Usage Layers — Background, text, icon, border

<Warning>Raw colors should never be used directly in UI. All product usage must go through semantic tokens.</Warning>

## Color Principles

<AccordionGroup>
  <Accordion title="Meaning over Decoration" icon="paintbrush">
    Color communicates state and intent, not decoration. Avoid using color purely for visual flair.
  </Accordion>
  <Accordion title="Neutral First" icon="circle">
    Neutrals form the foundation. Accent and status colors are used sparingly for emphasis.
  </Accordion>
  <Accordion title="One Role, One Color" icon="bullseye">
    Each UI element should serve one semantic role at a time. Avoid stacking multiple semantic meanings.
  </Accordion>
  <Accordion title="Theme Agnostic" icon="circle-half-stroke">
    All colors must work in light and dark modes without remapping UI logic.
  </Accordion>
  <Accordion title="Scalable by Design" icon="arrows-up-down-left-right">
    Adding a new brand or theme should only require changes at the core or semantic layer, not UI components.
  </Accordion>
</AccordionGroup>

## Brand Palette Anatomy

The Tarmac core palette is grounded in real-world materials and environments that shape the brand experience.

| Color | Hex | Role | Inspiration |
|-------|-----|------|-------------|
| Tar Black | `#000000` | Primary | Road tar — reliability, infrastructure |
| Optic White | `#FFFFFF` | Primary | Clarity, space, and contrast |
| Velocity Red | `#ED1B36` | Accent | Speed, urgency, forward motion |
| Atlantic Blue | `#2396FB` | Secondary | Ocean trade routes — trust, global connectivity |
| Kinetic Green | `#1BA86E` | Secondary | Progress and efficiency — continuous movement |
| Cardbox | `#B88B5C` | Secondary | Shipping cartons — physical reality of parcels |
| Cargo Grey | `#666666` | Neutral | Structure, UI elements, supporting hierarchy |
| Ash Grey | `#D0CFCF` | Neutral | Surfaces and dividers — depth and contrast |

Semantic tokens are mapped to the Tarmac core palette but referenced by purpose, not hue. If a color needs to change in the future, the meaning and naming remain stable.

## Accessibility Guidelines

Accessibility is enforced at the semantic layer, not manually per screen.

<AccordionGroup>
  <Accordion title="Contrast Requirements" icon="eye">
    **Text contrast**
    - Body text: WCAG AA (4.5:1) minimum
    - Large text: 3:1 minimum

    **Icon & UI elements**
    - Minimum 3:1 against background
  </Accordion>
  <Accordion title="Color Usage Rules" icon="palette">
    - Never rely on color alone to convey meaning
    - Pair color with icons, text, or shape
    - Avoid low-contrast alpha overlays for critical content
  </Accordion>
  <Accordion title="State Accessibility" icon="hand-pointer">
    - Hover, focus, and active states must remain distinguishable
    - Error and success colors should not rely solely on red/green differentiation
  </Accordion>
</AccordionGroup>

<Warning>Never use color as the only means of conveying information. Always pair color with text, icons, or patterns.</Warning>

    - Use neutral colors as the foundation
    - Let accent and status colors emphasize sparingly
    - Reference design tokens for theme switching
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC143C">
    - Don't reference core colors directly in components
    - Don't use color alone to convey meaning
    - Don't stack multiple semantic meanings on one element
    - Don't create custom colors outside the palette
    - Don't ignore contrast requirements
  </Card>
</CardGroup>
