## Token Mapping

Grid values are exposed as design tokens for consistent implementation across design and code.

<Tabs>
  <Tab title="Grid Tokens">
    | Token | Value | Description |
    |-------|-------|-------------|
    | `grid.columns.desktop` | `12` | Number of columns on desktop |
    | `grid.columns.mobile` | `4` | Number of columns on mobile |
    | `grid.columns.mobile.nav` | `2` | Columns for mobile top/bottom nav |
    | `grid.columns.mobile.sidenav` | `3` | Columns for mobile side nav |
    | `grid.gutter.large` | `24px` | Gutter width at 1920px breakpoint |
    | `grid.gutter.default` | `16px` | Gutter width at 1440px and below |
    | `grid.margin.desktop` | `80px` | Outer margin on desktop |
    | `grid.margin.mobile` | `16px` | Outer margin on mobile |
    | `grid.max-width` | `1280px` | Maximum content width on 1920+ screens |
    | `grid.column-width.desktop` | `92px` | Base column width on desktop |
  </Tab>
  <Tab title="Breakpoint Tokens">
    | Token | Value | Description |
    |-------|-------|-------------|
    | `breakpoint.desktop.large` | `1920px` | Large desktop / external monitors |
    | `breakpoint.desktop.default` | `1440px` | Primary design frame |
    | `breakpoint.desktop.medium` | `1366px` | Enterprise laptop resolution |
    | `breakpoint.desktop.small` | `1280px` | Compact laptop |
    | `breakpoint.mobile.large` | `390px` | Default mobile reference |
    | `breakpoint.mobile.default` | `360px` | Secondary modern devices |
    | `breakpoint.mobile.small` | `320px` | POS & constrained environments |
  </Tab>
  <Tab title="Spacing Integration">
    Grid tokens connect directly to the TARMAC spacing scale:

    | Grid Element | Spacing Token | Value |
    |-------------|--------------|-------|
    | Gutter (default) | `spacing-4` | 16px |
    | Gutter (large) | `spacing-6` | 24px |
    | Mobile margin | `spacing-4` | 16px |
    | Section gap | `spacing-8` to `spacing-12` | 32-48px |
    | Card gap | `spacing-4` to `spacing-6` | 16-24px |
    | Form field gap | `spacing-4` | 16px |

    <Info>Always reference spacing tokens for gutters and margins — never hardcode pixel values. This ensures grid spacing stays synchronized with the broader spacing system.</Info>
  </Tab>
</Tabs>

## Edge Cases & Anti-patterns

| Scenario | Problem | Solution |
|----------|---------|----------|
| Overlapping elements | Components extend beyond their column boundaries | Ensure all components respect column span. Use `overflow: hidden` on grid containers. |
| Misaligned components | Elements don't align with adjacent grid items | Check that all items use the same grid context. Avoid mixing grid systems. |
| Breaking grid for "visual fit" | Designer moves element off-grid to "look better" | Revisit content hierarchy or component sizing. The grid is the source of truth. |
| Inconsistent spacing | Different gutters or margins on the same page | Always use grid tokens. Audit pages for hardcoded spacing values. |
| Content overflow on mobile | Desktop layout crammed into 4 columns | Redesign for mobile — simplify, stack, or use progressive disclosure. Don't just shrink. |
| Drawer + side nav conflict | Side drawer opens while side nav is expanded | Define clear z-index and overlay rules. Drawer should overlay content, not push nav. |
| Ultra-wide monitors (2560px+) | Content stretches too wide, poor readability | Grid locks to 1280px max-width, centered. Background fills remaining space. |
| Nested grids | Grid inside a grid creates alignment conflicts | Avoid nesting grids. Use column spans within the parent grid instead. |

## Implementation Guidelines

<Steps>
  <Step title="Set Up the Grid Container">
    Define the grid container with 12 columns on desktop, 4 on mobile. Use CSS Grid or Flexbox with the grid tokens.

    ```css
    .grid-container {
      display: grid;
      grid-template-columns: repeat(12, 1fr);
      gap: var(--grid-gutter-default); /* 16px */
      max-width: var(--grid-max-width); /* 1280px */
      margin: 0 auto;
      padding: 0 var(--grid-margin-desktop); /* 80px */
    }

    @media (max-width: 768px) {
      .grid-container {
        grid-template-columns: repeat(4, 1fr);
        padding: 0 var(--grid-margin-mobile); /* 16px */
      }
    }
    ```
  </Step>
  <Step title="Handle Breakpoints">
    Use breakpoint tokens to define responsive behavior. Content should adapt at each defined breakpoint.

    ```css
    /* Large desktop */
    @media (min-width: 1920px) {
      .grid-container {
        gap: var(--grid-gutter-large); /* 24px */
      }
    }

    /* Medium desktop */
    @media (max-width: 1366px) {
      .grid-container {
        max-width: 100%;
      }
    }

    /* Mobile */
    @media (max-width: 768px) {
      .grid-container {
        grid-template-columns: repeat(4, 1fr);
        gap: var(--grid-gutter-default);
      }
    }
    ```
  </Step>
  <Step title="Use Column Spans">
    Apply column spans to child elements using `grid-column`.

    ```css
    /* Full width */
    .span-12 { grid-column: span 12; }

    /* Half width */
    .span-6 { grid-column: span 6; }

    /* Third width */
    .span-4 { grid-column: span 4; }

    /* Quarter width */
    .span-3 { grid-column: span 3; }

    /* With offset */
    .offset-2 { grid-column-start: 3; }
    .offset-4 { grid-column-start: 5; }

    /* Mobile: everything full width */
    @media (max-width: 768px) {
      [class*="span-"] {
        grid-column: span 4;
      }
    }
    ```
  </Step>
  <Step title="Handle Side Navigation">
    When side navigation is present, wrap the layout in a flex container with the nav and grid as siblings.

    ```css
    .layout {
      display: flex;
    }

    .side-nav {
      width: 280px; /* expanded */
      flex-shrink: 0;
    }

    .side-nav.collapsed {
      width: 72px;
    }

    .main-content {
      flex: 1;
      display: grid;
      grid-template-columns: repeat(12, 1fr);
      gap: var(--grid-gutter-default);
    }
    ```
  </Step>
  <Step title="Test at All Breakpoints">
    Verify layouts at every defined viewport:
    - 1920 × 1080 (large desktop)
    - 1440 × 942 (default)
    - 1366 × 768 (medium)
    - 1280 × 832 (small desktop)
    - 390 × 844 (large mobile)
    - 360 × 800 (default mobile)
    - 320 × 568 (small mobile)

    <Tip>Use browser DevTools responsive mode to test each breakpoint. Pay special attention to column collapse behavior and content stacking order.</Tip>
  </Step>
</Steps>
