Icons provide quick visual cues that support understanding and navigation across the interface. They communicate actions, status, and meaning at a glance, reducing cognitive load and helping users move through tasks efficiently. Icons are designed to complement text — not replace it — and are used consistently to reinforce familiarity and clarity.

The system uses Google's Material Icon library as its core icon set, ensuring visual consistency, scalability, and cross-platform reliability.

## Principles

<AccordionGroup>
  <Accordion title="Clarity First">
    Icons must be immediately recognizable and unambiguous. If an icon's meaning is unclear, pair it with text or avoid using it.
  </Accordion>
  <Accordion title="Consistency Over Variety">
    Use icons from the same style and weight to maintain a unified visual language. Avoid mixing icon styles or custom modifications that break consistency.
  </Accordion>
  <Accordion title="Support, Don't Replace">
    Icons support actions and labels but should not carry meaning alone in critical workflows.
  </Accordion>
  <Accordion title="Purpose-Driven Use">
    Every icon should communicate intent — action, status, navigation, or feedback — never decoration.
  </Accordion>
  <Accordion title="Balanced Visual Weight">
    Icons should align optically with text, spacing, and component size to avoid visual dominance or imbalance.
  </Accordion>
</AccordionGroup>

## Using Material Icon Library

<Steps>
  <Step title="Choose a consistent style">
    Use Material Icons in their intended styles (outlined or filled). Pick one and stay consistent within a context.
  </Step>
  <Step title="Maintain proportions">
    Maintain original proportions and alignment. Never distort, stretch, or recolor icons beyond system rules.
  </Step>
  <Step title="Size to context">
    Size icons according to the component they belong to (12px, 16px, 20px, 24px, 28px, 32px).
  </Step>
  <Step title="Follow the token system">
    Use icon color tokens for consistent theming across light and dark modes.
  </Step>
</Steps>

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check">
    - Use icons to support and reinforce text labels
    - Maintain consistent sizing within a context
    - Provide accessible labels for all interactive icons
    - Use semantic icons that match the action or content
    - Stick to one style (filled or outlined) per context
  </Card>
  <Card title="✗ Don't" icon="xmark">
    - Don't use icons as the sole indicator of meaning
    - Don't mix filled and outlined styles in the same context

    - Don't use icons purely for decoration
    - Don't modify or distort Material Icons
  </Card>

## Accessibility

<Warning>
  Icons should never be the sole indicator of meaning or status. Always pair critical icons with text labels, tooltips, or aria-labels.
</Warning>

| Requirement | Guideline |
|-------------|-----------|
| Labels | Always pair critical icons with text labels, tooltips, or `aria-label` |
| Color contrast | Ensure sufficient contrast between icons and backgrounds (WCAG AA) |
| Color independence | Avoid relying only on color to convey meaning |
| Focus states | Interactive icons must have clear focus and hover states |
| Touch targets | Touch targets should meet minimum size guidelines (44 × 44px) |
| Assistive tech | Icons must include appropriate accessible names and roles |
