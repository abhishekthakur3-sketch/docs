## Overview

Typography in TARMAC is built on Noto Sans — a versatile, humanist sans-serif designed for clarity at every size. The type system uses a Minor Third (1.2) scale to create strong visual hierarchy, consistent rhythm, and accessible reading experiences across all platforms.

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#10b981">
    **Maintain hierarchy**
    - Use one H1 per page
    - Follow heading order (H1 → H2 → H3 → …)
    - Use Display only for hero-level content

    **Optimize readability**
    - Keep body text at 45–75 characters per line
    - Use B1 (16px) as the default reading size
    - Maintain line height ≥ 1.45× for body text

    **Use semantic tokens**
    - Reference `text.heading.primary` not `#0D0D0D`
    - Use weight variants from the type system
    - Pair text roles with their designated color tokens
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#ef4444">
    **Don't skip heading levels**
    - ~~H1 → H3~~ (skipping H2 breaks hierarchy)
    - ~~Display for section titles~~ (use H1 or H2)

    **Don't use tiny text for content**
    - ~~10px for body text~~ (C2 is for legal/constrained UI only)
    - ~~12px for primary content~~ (use B2 at minimum)

    **Don't hardcode values**
    - ~~`font-size: 16px`~~ (use tokens)
    - ~~`color: #1A1A1A`~~ (use semantic color tokens)
    - ~~`font-weight: 600`~~ (use `font-weight-semibold`)
  </Card>
</CardGroup>

<Tip>When in doubt, use B1 (16px, medium weight) for content and H3 (28px, semibold) for section headings. These are the most versatile styles in the system.</Tip>

## Line Height

Line height controls vertical rhythm and readability. TARMAC uses tighter line heights for large text and looser values for body content.

| Text Role | Font Size | Line Height | Ratio |
|-----------|-----------|-------------|-------|
| Caption (C1–C2) | 10–12px | 16px | 1.2–1.4× |
| Body (B1–B2) | 14–16px | 20–24px | 1.3–1.5× |
| Heading (H1–H5) | 20–40px | 26–52px | 1.2–1.4× |
| Display | 48px | 58px | 1.0–1.2× |

<Info>
  **Accessibility rules:**
  - Body text line height must not go below **1.45×** to maintain readability for users with cognitive disabilities.
  - Display text line height must not exceed **1.2×** to keep hero text visually cohesive.
</Info>

## Letter Spacing

Letter spacing (tracking) is adjusted per text role. Larger text uses negative tracking for visual tightness; smaller text uses positive tracking for legibility.

| Text Role | Font Size | Letter Spacing | Direction |
|-----------|-----------|----------------|-----------|
| Caption (C1–C2) | 10–12px | +0.4 to +0.6em | Looser — aids legibility at small sizes |
| Body (B1–B2) | 14–16px | +0.2 to +0.5em | Slightly open — comfortable reading |
| Heading (H1–H5) | 20–40px | -0.1 to -0.5em | Tighter — creates visual density |
| Display | 48px | -0.5 to -1em | Tightest — high-impact, compact feel |

<Warning>
  **Accessibility rules:**
  - Never use negative letter spacing on body or caption text — it reduces readability.
  - Ensure all text remains readable at **200% browser zoom**.
</Warning>

## Accessibility

<AccordionGroup>
  <Accordion title="Contrast Requirements" icon="eye">
    **WCAG 2.1 Level AA**
    - Normal text (&lt; 18px): **4.5:1** contrast ratio minimum
    - Large text (≥ 18px or ≥ 14px bold): **3:1** contrast ratio minimum

    All typography color tokens in TARMAC are pre-validated against these thresholds for both light and dark modes. Disabled states are intentionally below contrast thresholds to signal non-interactivity.
  </Accordion>
  <Accordion title="Minimum Font Sizes" icon="text-height">
    - **Body text**: 16px (B1) minimum for primary content
    - **Secondary text**: 14px (B2) minimum for labels and supporting content
    - **Captions**: 12px (C1) minimum for metadata and helper text
    - **Constrained UI**: 10px (C2) only when absolutely necessary

    Always use `rem` units so text scales with user browser preferences.
  </Accordion>
  <Accordion title="Line Height and Spacing" icon="arrows-up-down">
    - Body text: line height ≥ **1.45×** (WCAG 1.4.12)
    - Paragraph spacing: at least **1.5×** the font size between paragraphs
    - Letter spacing: never negative for body or caption text

    These rules ensure readability for users with dyslexia and low vision.
  </Accordion>
  <Accordion title="Semantic HTML" icon="code">
    Always use proper heading elements (`h1`–`h5`) rather than styled `div` or `span` elements. Screen readers rely on heading structure for navigation.

    ```html
    <!-- Correct -->
    <h1>Page Title</h1>
    <h2>Section Heading</h2>
    <p>Body paragraph with <strong>emphasis</strong>.</p>

    <!-- Incorrect -->
    <div class="h1">Page Title</div>
    <span style="font-size: 34px">Section Heading</span>
    ```
  </Accordion>
  <Accordion title="Zoom and Reflow" icon="magnifying-glass-plus">
    - All text must remain readable at **200% browser zoom**
    - Text must reflow within the viewport — no horizontal scrolling
    - Avoid fixed-width containers that clip text at larger zoom levels
    - Use relative units (`rem`, `em`) instead of `px` for font sizes
  </Accordion>
</AccordionGroup>
