## Font Families

### Primary Font Stack

The TARMAC Design System uses system fonts for optimal performance and native feel.

```css
--font-family-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", 
                     Roboto, "Helvetica Neue", Arial, sans-serif;
```

**Benefits:**
- Zero load time (already installed)
- Native OS appearance
- Excellent readability
- Consistent across platforms

### Monospace Font Stack

Used for code snippets, technical content, and data display.

```css
--font-family-mono: "SF Mono", Monaco, "Cascadia Code", "Roboto Mono",
                     Consolas, "Courier New", monospace;
```

## Font Weights

| Token | Value | Usage |
|-------|-------|-------|
| `font-weight-normal` | 400 | Body text, default |
| `font-weight-medium` | 500 | Emphasis, labels |
| `font-weight-semibold` | 600 | Subheadings, buttons |
| `font-weight-bold` | 700 | Headings, strong emphasis |

<Warning>Avoid using font weights below 400 or above 700 for better cross-platform consistency.</Warning>

## Usage Guidelines

### Do's and Don'ts

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#10b981">
    **Maintain hierarchy**
    - Use one H1 per page
    - Follow heading order (H1 → H2 → H3)
    - Create clear visual distinction
    
    **Optimize readability**
    - 45-75 characters per line
    - Adequate line height (1.5+)
    - Sufficient contrast (4.5:1)
    
    **Use semantic HTML**
    - `<h1>` through `<h6>` for headings
    - `<p>` for paragraphs
    - `<strong>` for emphasis
  </Card>
  
  <Card title="✗ Don't" icon="xmark" color="#ef4444">
    **Don't skip heading levels**
    - ~~H1 → H3~~ (skip H2)
    - Maintain logical structure
    
    **Don't use tiny text**
    - Minimum 12px for UI
    - Minimum 14px for body text
    - Consider accessibility
    
    **Don't use too many weights**
    - Stick to 3-4 weights max
    - Maintain consistency
    - Avoid font weight overload
  </Card>
</CardGroup>

### Line Length

Optimal line length for readability:

| Context | Characters | Pixels (approx) |
|---------|-----------|-----------------|
| Body text | 45-75 | 600-800px |
| Wide layouts | 75-90 | 800-1000px |
| Narrow columns | 40-50 | 400-600px |

```css
.content {
  max-width: 65ch; /* ~65 characters */
}
```

### Text Alignment

| Alignment | Usage |
|-----------|-------|
| Left | Default for body text, headings (LTR languages) |
| Center | Short headings, hero text, marketing content |
| Right | Rarely used, numeric data, RTL languages |
| Justify | Avoid - creates uneven spacing |

<Tip>Use left alignment for most text content. Center alignment works well for short headings and hero sections.</Tip>

## Accessibility Considerations

<AccordionGroup>
  <Accordion title="✓ Contrast Requirements" icon="eye">
    **WCAG 2.1 Level AA**
    - Normal text (< 18px): 4.5:1 contrast ratio
    - Large text (≥ 18px or ≥ 14px bold): 3:1 contrast ratio
    
    **Recommended Combinations**
    ```css
    /* High contrast (AAA) */
    color: var(--color-neutral-900);  /* on white */
    color: var(--color-neutral-50);   /* on black */
    
    /* Medium contrast (AA) */
    color: var(--color-neutral-700);  /* on white */
    color: var(--color-neutral-300);  /* on black */
    ```
  </Accordion>
  
  <Accordion title="✓ Font Size" icon="text-height">
    **Minimum Sizes**
    - Body text: 16px (1rem) minimum
    - Small text: 14px minimum
    - UI elements: 12px minimum (use sparingly)
    
    **Relative Units**
    - Use `rem` for font sizes (scales with user preferences)
    - Use `em` for spacing relative to font size
    - Avoid fixed `px` values when possible
    
    ```css
    body {
      font-size: 1rem; /* 16px, but respects user settings */
    }
    ```
  </Accordion>
  
  <Accordion title="✓ Line Height" icon="arrows-up-down">
    **Recommended Values**
    - Body text: 1.5 (150%)
    - Headings: 1.1-1.3
    - Small text: 1.4-1.5
    
    **Benefits**
    - Improves readability
    - Helps users with dyslexia
    - Easier to track lines
    
    ```css
    p {
      line-height: 1.5;
    }
    ```
  </Accordion>
  
  <Accordion title="✓ Semantic HTML" icon="code">
    **Use Proper Elements**
    ```html
    <!-- Good -->
    <h1>Page Title</h1>
    <h2>Section Heading</h2>
    <p>Body paragraph</p>
    <strong>Important text</strong>
    <em>Emphasized text</em>
    
    <!-- Bad -->
    <div class="heading-1">Page Title</div>
    <span class="bold">Important text</span>
    ```
    
    **Benefits**
    - Screen reader navigation
    - SEO improvements
    - Logical document structure
  </Accordion>
</AccordionGroup>

### Responsive Typography

```css
/* Mobile-first approach */
h1 {
  font-size: var(--text-4xl); /* 32px on mobile */
}

@media (min-width: 768px) {
  h1 {
    font-size: var(--text-5xl); /* 48px on tablet+ */
  }
}

@media (min-width: 1024px) {
  h1 {
    font-size: var(--text-6xl); /* 60px on desktop */
  }
}

/* Fluid typography (advanced) */
h1 {
  font-size: clamp(2rem, 5vw, 3rem);
}
```
