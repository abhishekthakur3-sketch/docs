## The Four Principles

TARMAC follows the four WCAG principles that form the foundation of web accessibility:

<Tabs>
  <Tab title="Perceivable">
    Information and UI components must be presentable in ways users can perceive.

    | Guideline | TARMAC Implementation |
    |-----------|----------------------|
    | Text alternatives | All icons include `aria-label`; image placeholders require alt text |
    | Color contrast | Design tokens guarantee 4.5:1 minimum for text, 3:1 for UI elements |
    | Adaptable content | Components respond to text scaling up to 200% |
    | Distinguishable | Color is never the sole means of conveying information |
  </Tab>
  <Tab title="Operable">
    UI components and navigation must be operable by all users.

    | Guideline | TARMAC Implementation |
    |-----------|----------------------|
    | Keyboard accessible | Every interactive component works without a mouse |
    | Sufficient time | No time-limited interactions without user control |
    | No seizures | No content flashes more than 3 times per second |
    | Navigable | Skip links, landmarks, and logical focus order |
  </Tab>
  <Tab title="Understandable">
    Information and UI operation must be understandable.

    | Guideline | TARMAC Implementation |
    |-----------|----------------------|
    | Readable | Clear typography hierarchy with legible font sizes |
    | Predictable | Consistent component behavior across the system |
    | Input assistance | Inline validation, clear error messages, helper text |
    | Error prevention | Confirmation dialogs for destructive actions |
  </Tab>
  <Tab title="Robust">
    Content must work reliably with assistive technologies.

    | Guideline | TARMAC Implementation |
    |-----------|----------------------|
    | Valid markup | Semantic HTML with proper ARIA attributes |
    | Name, Role, Value | All components expose correct states to assistive tech |
    | Status messages | Dynamic updates use `aria-live` regions |
    | Compatibility | Tested with NVDA, VoiceOver, JAWS, and TalkBack |
  </Tab>
</Tabs>

## What TARMAC Provides

Every TARMAC component ships with these accessibility features built in:

| Feature | Details |
|---------|---------|
| Semantic HTML | Correct elements and roles for every component |
| Keyboard navigation | Tab, arrow keys, Enter, Space, Escape support |
| Focus management | Visible focus indicators, focus trapping in modals |
| ARIA attributes | Labels, descriptions, states, and live regions |
| Color contrast | All color combinations meet AA standards |
| Responsive text | Components support text scaling and zoom |
| Reduced motion | Animations respect `prefers-reduced-motion` |

## Your Responsibility

While TARMAC handles component-level accessibility, you are responsible for:

| Area | What You Need To Do |
|------|---------------------|
| Content | Write descriptive alt text, link text, and headings |
| Page structure | Use landmarks, heading hierarchy, and skip links |
| Forms | Associate labels with inputs, provide error messages |
| Dynamic content | Announce changes with live regions |
| Testing | Test with keyboard, screen readers, and automated tools |
| Custom code | Follow ARIA patterns for any custom widgets |
