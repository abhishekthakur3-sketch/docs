## Overview

These guidelines provide actionable rules for designers and developers working with TARMAC. Every component in the system is built to meet WCAG 2.1 Level AA, but proper usage is essential to maintain accessibility in your application.

## Designer Guidelines

### Color and Contrast

| Rule | Requirement |
|------|-------------|
| Body text contrast | Minimum 4.5:1 against background |
| Large text contrast (18px+ or 14px bold) | Minimum 3:1 against background |
| UI component contrast | Minimum 3:1 for borders, icons, controls |
| Don't rely on color alone | Pair color with text, icons, or patterns |

### Touch Targets

| Rule | Requirement |
|------|-------------|
| Minimum touch target | 44 x 44px (WCAG) / 48 x 48px (recommended) |
| Spacing between targets | Minimum 8px gap |
| Clickable area | Extend beyond visible element if needed |

### Visual Hierarchy

| Rule | Requirement |
|------|-------------|
| Heading order | Use H1 through H6 in sequential order, never skip levels |
| Reading order | Content should make sense when read top-to-bottom, left-to-right |
| Focus order | Interactive elements should follow visual layout order |
| Landmarks | Define clear page regions (header, nav, main, footer) |

### Motion and Animation

| Rule | Requirement |
|------|-------------|
| Reduced motion | Respect `prefers-reduced-motion` media query |
| Auto-playing content | Provide pause/stop controls |
| Flashing content | Never flash more than 3 times per second |
| Transition duration | Keep under 400ms for UI transitions |

## Developer Guidelines

### Semantic HTML

Always use the correct HTML element for its purpose:

```tsx
// Correct
<button onClick={handleClick}>Save</button>
<a href="/settings">Settings</a>
<nav aria-label="Main navigation">...</nav>

// Incorrect
<div onClick={handleClick}>Save</div>
<span onClick={navigate}>Settings</span>
<div class="nav">...</div>
```

### ARIA Usage Rules

| Rule | Details |
|------|---------|
| First rule of ARIA | Don't use ARIA if native HTML can do the job |
| Required attributes | Always include required ARIA attributes for roles |
| State management | Update `aria-expanded`, `aria-selected`, `aria-checked` dynamically |
| Labels | Every interactive element needs an accessible name |
| Live regions | Use `aria-live` for dynamic content updates |

### Keyboard Support

| Rule | Details |
|------|---------|
| All interactive elements | Must be reachable via Tab key |
| Custom widgets | Implement arrow key navigation per WAI-ARIA patterns |
| Focus visible | Never remove `:focus-visible` outlines |
| Focus trapping | Trap focus in modals, restore on close |
| Skip links | Provide "Skip to main content" link |

### Form Accessibility

| Rule | Details |
|------|---------|
| Labels | Every input needs a visible, associated `label` |
| Errors | Use `aria-invalid` and `aria-describedby` for error messages |
| Required fields | Use `aria-required="true"` plus visual indicator |
| Grouping | Use `fieldset`/`legend` for related inputs |
| Error focus | Move focus to first error on form submission failure |

## Component Checklist

Use this checklist when implementing any TARMAC component:

| Check | Description |
|-------|-------------|
| Keyboard operable | Can be fully used without a mouse |
| Focus indicator | Shows visible focus ring when focused via keyboard |
| Screen reader | Announces name, role, and state correctly |
| Color contrast | Meets 4.5:1 for text, 3:1 for UI elements |
| Touch target | Minimum 44x44px interactive area |
| Error handling | Errors are announced and visually indicated |
| Reduced motion | Animations respect user preference |
| Zoom support | Content remains usable at 200% zoom |

## Common Mistakes

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use semantic HTML elements
    - Provide text alternatives for images
    - Ensure keyboard navigation works
    - Test with screen readers
    - Use TARMAC design tokens for colors
    - Include visible focus indicators
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use `div` or `span` for interactive elements
    - Don't remove focus outlines with `outline: none`
    - Don't use `tabindex` values greater than 0
    - Don't auto-play audio or video
    - Don't use title attribute as the only accessible name
    - Don't rely on hover-only interactions
  </Card>
</CardGroup>
