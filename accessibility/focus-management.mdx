## Overview

Proper focus management ensures keyboard users can navigate your application efficiently and predictably. TARMAC components handle focus internally, but you need to manage focus at the page and flow level.

---

## Focus Indicators

All TARMAC interactive elements display a visible focus ring:

| Property | Value |
|----------|-------|
| Style | Box shadow (not outline, for rounded elements) |
| Color | rgba(35, 150, 251, 0.4) — info blue |
| Spread | 2px |
| Offset | 0px (flush with element) |

<Warning>
  Never remove focus indicators with `outline: none` or `box-shadow: none`. If you customize them, ensure the replacement meets 3:1 contrast against adjacent colors.
</Warning>

---

## Focus Order

Focus order must follow the visual reading order:

| Priority | Region | Notes |
|----------|--------|-------|
| 1 | Skip link | Hidden until focused |
| 2 | Header / navigation | App-level controls |
| 3 | Main content | Top to bottom, left to right |
| 4 | Sidebar | Secondary navigation |
| 5 | Footer | Tertiary actions |

<Tip>
  Avoid using positive tabindex values (tabindex="1", "2", etc.). They override natural DOM order and create confusing navigation. Use tabindex="0" to make elements focusable in DOM order, or tabindex="-1" for programmatic focus only.
</Tip>

## Focus Trapping

Trap focus inside modal dialogs and overlays to prevent keyboard users from accessing background content:

```tsx
import { useFocusTrap } from '@tarmac/design-system';

function Dialog({ open, onClose, children }) {
  const trapRef = useFocusTrap();

  if (!open) return null;

  return (
    <div ref={trapRef} role="dialog" aria-modal="true">
      {children}
      <button onClick={onClose}>Close</button>
    </div>
  );
}
```

### When to Trap Focus

| Component | Trap Focus | Reason |
|-----------|-----------|--------|
| Modal / Dialog | Yes | Prevents interaction with background |
| Side Drawer | Yes | Overlay blocks background |
| Bottom Sheet | Yes | Overlay blocks background |
| Dropdown Menu | No | User should be able to Tab away |
| Tooltip | No | Informational only |
| Popover | Depends | Trap if it contains interactive elements |

## Focus Restoration

When closing a modal or overlay, return focus to the element that triggered it:

```tsx
function MyComponent() {
  const triggerRef = useRef(null);

  const handleClose = () => {
    setIsOpen(false);
    triggerRef.current?.focus(); // Restore focus
  };

  return (
    <>
      <button ref={triggerRef} onClick={() => setIsOpen(true)}>
        Open Dialog
      </button>
      <Dialog open={isOpen} onClose={handleClose} />
    </>
  );
}
```

## Programmatic Focus

Move focus programmatically for route changes and dynamic content:

| Scenario | Focus Target |
|----------|-------------|
| Page navigation | Main content heading or skip link target |
| Form submission error | First invalid field |
| Toast/snackbar | Don't move focus (use aria-live instead) |
| New content loaded | First element of new content |
| Item deleted from list | Next item in list, or previous if last |

## Skip Links

Provide a skip link as the first focusable element on every page:

```tsx
<a
  href="#main-content"
  style={{
    position: 'absolute',
    top: '-40px',
    left: '0',
    background: '#0D0D0D',
    color: 'white',
    padding: '8px 16px',
    zIndex: 100,
  }}
  onFocus={(e) => { e.target.style.top = '0'; }}
  onBlur={(e) => { e.target.style.top = '-40px'; }}
>
  Skip to main content
</a>

<main id="main-content" tabIndex={-1}>
  {/* Page content */}
</main>
```

## Testing Checklist

| Check | Pass Criteria |
|-------|---------------|
| Focus indicators visible | Every focused element shows a clear ring |
| Focus order logical | Matches visual layout, no unexpected jumps |
| Focus trapped in modals | Tab cycles within modal, not behind it |
| Focus restored on close | Focus returns to trigger element |
| Skip link works | Appears on focus, jumps to main content |
| No keyboard traps | User can always navigate away (except modals) |
| Programmatic focus correct | Route changes and errors move focus appropriately |
