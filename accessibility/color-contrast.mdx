# Color Contrast

Proper color contrast ensures text and UI elements are readable for all users, including those with visual impairments.

## WCAG Requirements

### Level AA (Minimum)

| Content Type | Contrast Ratio |
|--------------|----------------|
| Normal text (< 18px) | 4.5:1 |
| Large text (≥ 18px or ≥ 14px bold) | 3:1 |
| UI components | 3:1 |
| Graphics | 3:1 |

### Level AAA (Enhanced)

| Content Type | Contrast Ratio |
|--------------|----------------|
| Normal text | 7:1 |
| Large text | 4.5:1 |

## Color Blindness

### Types of Color Blindness

- **Deuteranopia** (red-green, most common)
- **Protanopia** (red-green)
- **Tritanopia** (blue-yellow, rare)
- **Achromatopsia** (complete color blindness, very rare)

### Guidelines

1. Never use color as the only means of conveying information
2. Pair color with text, icons, or patterns
3. Test designs with color blindness simulators
4. Ensure sufficient contrast between adjacent colors

## Code Examples

### CSS with Sufficient Contrast

```css
/* Good - High contrast */
.text-primary {
  color: var(--color-neutral-900); /* 21:1 on white */
}

.text-secondary {
  color: var(--color-neutral-600); /* 7:1 on white */
}

/* Bad - Low contrast */
.text-low-contrast {
  color: #999999; /* Only 2.8:1 on white */
}
```

### Testing in Code

```tsx
// Use design tokens that guarantee contrast
<Text color="neutral-900">High contrast text</Text>
<Text color="neutral-600">Medium contrast text</Text>

// Avoid arbitrary colors
<Text style={{ color: '#aaa' }}>Low contrast - avoid!</Text>
```
