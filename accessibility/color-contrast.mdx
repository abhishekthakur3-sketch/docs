## Overview

Proper color contrast ensures text and UI elements are readable for all users, including those with low vision or color blindness. All TARMAC color token combinations meet WCAG 2.1 Level AA standards.

---

## WCAG Requirements

<Tabs>
  <Tab title="Level AA (Minimum)">
    | Content Type | Contrast Ratio |
    |--------------|----------------|
    | Normal text (below 18px) | 4.5:1 |
    | Large text (18px+ or 14px bold) | 3:1 |
    | UI components and graphics | 3:1 |
  </Tab>
  <Tab title="Level AAA (Enhanced)">
    | Content Type | Contrast Ratio |
    |--------------|----------------|
    | Normal text | 7:1 |
    | Large text | 4.5:1 |
  </Tab>
</Tabs>

## TARMAC Color Combinations

All TARMAC design token combinations are pre-validated for contrast:

### Text on Backgrounds

| Foreground | Background | Ratio | Level |
|------------|------------|-------|-------|
| neutral-900 (#0D0D0D) | white | 19.4:1 | AAA |
| neutral-700 (#454545) | white | 9.7:1 | AAA |
| neutral-600 (#808080) | white | 3.9:1 | AA Large |
| body/primary (#2B2B2B) | white | 14.7:1 | AAA |
| caption/primary (#454545) | white | 9.7:1 | AAA |
| body/disabled (#CDCBCB) | white | 1.5:1 | Fail |

<Warning>
  Disabled text intentionally has low contrast to indicate non-interactive state. Never use disabled color tokens for active content.
</Warning>

### Interactive Elements

| Element | Colors | Ratio |
|---------|--------|-------|
| Primary Button | #0D0D0D on white | 19.4:1 |
| Focus Ring | rgba(35, 150, 251, 0.4) | 3:1 against white |
| Error State | #DC2626 on white | 4.6:1 |
| Success State | #16A34A on white | 3.4:1 |

## Color Blindness

### Types

| Type | Prevalence | Affected Colors |
|------|-----------|-----------------|
| Deuteranopia | ~6% of males | Red-green confusion |
| Protanopia | ~2% of males | Red-green confusion |
| Tritanopia | ~0.01% | Blue-yellow confusion |
| Achromatopsia | ~0.003% | Complete color blindness |

### Guidelines

| Rule | Details |
|------|---------|
| Never use color alone | Pair color with text, icons, or patterns |
| Error indicators | Use icon + red color + error text |
| Success indicators | Use icon + green color + success text |
| Status badges | Include text label, not just color |
| Charts and graphs | Use patterns or labels in addition to color |

## Best Practices

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use TARMAC design tokens for all colors
    - Test all text/background combinations
    - Verify UI component contrast (borders, icons)
    - Check focus indicator visibility
    - Test with color blindness simulators
    - Pair color with secondary indicators
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use arbitrary hex colors
    - Don't rely on color as the only indicator
    - Don't use low-contrast placeholder text as labels
    - Don't forget to test disabled states
    - Don't assume all users see color the same way
    - Don't skip contrast checks on dark backgrounds
  </Card>
</CardGroup>
