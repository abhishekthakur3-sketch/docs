## Overview

Bottom Sheets are mobile overlay panels that slide up from the bottom of the screen. They provide space for contextual actions, confirmations, forms, or supplementary content. The system supports four content types (Standard, Slot, Slot Dual, Slot Mini) with two CTA layout styles (Horizontal and Vertical), all at a fixed 320px width for mobile screens.

## Types

| Type | Description | Use When |
|------|-------------|----------|
| Standard | Title, description, illustration, and action buttons | Simple confirmations, alerts, decisions |
| Slot Mini | Standard content plus a small custom slot area | Compact additional content |
| Slot | Standard content plus one full custom slot area | Forms, selections, custom content |
| Slot Dual | Standard content plus two custom slot areas | Complex multi-section content |

---

## CTA Styles

| Style | Description | Use When |
|-------|-------------|----------|
| CTA Horizontal | Action buttons side by side | Two equal-weight actions |
| CTA Vertical | Action buttons stacked vertically | Primary action emphasis, longer labels |

---

## Sizes

| Property | Value |
|----------|-------|
| Width | 320px (full mobile width with padding) |
| Standard Height | 312px (horizontal) / 356px (vertical) |
| Slot Mini Height | 328px (horizontal) / 372px (vertical) |
| Slot Height | 332px (horizontal) / 376px (vertical) |
| Slot Dual Height | 348px (horizontal) / 392px (vertical) |

---

## Behavior

| Interaction | Result |
|-------------|--------|
| Bottom sheet opens | Slides up from bottom, backdrop appears |
| Swipe down | Dismisses the bottom sheet |
| Click backdrop | Closes bottom sheet |
| Click primary action | Executes action, closes sheet |
| Click secondary action | Cancels or alternative action |
| Content overflow | Content area scrolls |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use for mobile-specific contextual actions
    - Keep content concise and focused
    - Use appropriate type for content complexity
    - Allow swipe-to-dismiss gesture
    - Use vertical CTA for primary action emphasis
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use on desktop (use Popup or Side Drawer instead)
    - Don't stack multiple bottom sheets
    - Don't use for complex multi-step workflows
    - Don't remove the dismiss option
    - Don't overload with too much content
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="dialog"` with `aria-modal="true"` |
| Label | Provide `aria-labelledby` pointing to the title |
| Focus trap | Focus must be trapped inside while open |
| Gesture | Swipe-to-dismiss must have a button alternative |
| Screen reader | Announce title on open |
| Return focus | Return focus to trigger element on close |

## Developer Handoff

<CodeGroup>
```tsx API
interface BottomSheetProps {
  type?: 'standard' | 'slot' | 'slot_dual' | 'slot_mini';
  ctaStyle?: 'horizontal' | 'vertical';
  open: boolean;
  onClose: () => void;
  title: string;
  description?: string;
  illustration?: ReactNode;
  primaryAction: { label: string; onClick: () => void };
  secondaryAction?: { label: string; onClick: () => void };
  slot1?: ReactNode;
  slot2?: ReactNode;
  dismissible?: boolean;
}
```

```tsx Usage
// Standard bottom sheet
<BottomSheet
  type="standard"
  open={showSheet}
  onClose={() => setShowSheet(false)}
  title="Confirm Delivery"
  description="Mark this order as delivered?"
  primaryAction={{ label: 'Confirm', onClick: handleConfirm }}
  secondaryAction={{ label: 'Cancel', onClick: () => setShowSheet(false) }}
/>

// With custom slot content
<BottomSheet
  type="slot"
  ctaStyle="vertical"
  open={showOptions}
  onClose={() => setShowOptions(false)}
  title="Select Option"
  slot1={<OptionsList />}
  primaryAction={{ label: 'Apply', onClick: handleApply }}
/>

// Dual slot
<BottomSheet
  type="slot_dual"
  open={showDual}
  onClose={() => setShowDual(false)}
  title="Payment Method"
  slot1={<PaymentOptions />}
  slot2={<WalletBalance />}
  primaryAction={{ label: 'Pay Now', onClick: handlePay }}
/>
```
</CodeGroup>
