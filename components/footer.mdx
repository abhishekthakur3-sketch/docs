## Overview

The Footer system defines how actions are presented at the bottom of a screen. It consists of two components: Action Footer for general workflows and decisions, and Payment Checkout Footer for payment and transaction flows. Together, they standardize how actions, summaries, and decisions are handled across the product.

## Payment Checkout Footer

Footer for transaction and checkout flows, focusing on clarity, trust, and conversion.

| Type | Description | Use When |
|------|-------------|----------|
| Proceed to Pay | Simple CTA with total amount | Quick checkout, single-step payment |
| Bill Details | Amount summary with bill details access | Order review before payment |
| Bill Details + Wallet | Wallet balance with bill details | Wallet-enabled checkout |
| Bill Details + Wallet + Options | Full payment options with wallet and offers | Complex checkout with multiple payment methods |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Keep the primary action clear and prominent
    - Maintain consistent footer placement
    - Use clear, action-oriented labels
    - Ensure real-time updates for dynamic data (payments)
    - Use Action Footer for workflows, Payment Footer for transactions
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't overload footers with too many actions
    - Don't hide critical information like total amount
    - Don't use vague CTAs
    - Don't mix workflow and payment patterns incorrectly
    - Don't remove the primary action button
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Focus order | Primary CTA should be easily reachable via keyboard |
| Labels | All buttons must have descriptive accessible labels |
| Contrast | CTA buttons must meet minimum contrast requirements |
| Screen reader | Announce total amount and payment method changes |
| Touch targets | All interactive elements must meet 44x44px minimum |

## Developer Handoff

<CodeGroup>
```tsx API
// Action Footer
interface ActionFooterProps {
  variant?: 'standard' | 'with_banner';
  size?: 'web' | 'mobile';
  title?: string;
  subtext1?: string;
  subtext2?: string;
  badge?: BadgeProps;
  slot1?: ReactNode;
  slot2?: ReactNode;
  link?: { label: string; href: string };
  primaryAction: { label: string; onClick: () => void };
  secondaryAction?: { label: string; onClick: () => void };
  tertiaryAction?: { label: string; onClick: () => void };
  banner?: ReactNode;
}

// Payment Checkout Footer
interface PaymentCheckoutFooterProps {
  type?: 'proceed_to_pay' | 'bill_details' | 'bill_details_wallet' | 'bill_details_wallet_options';
  totalAmount: string;
  paymentMethod?: string;
  paymentMethodIcon?: ReactNode;
  walletBalance?: string;
  onPay: () => void;
  onViewBillDetails?: () => void;
  onChangePaymentMethod?: () => void;
}
```

```tsx Usage
// Action Footer - Standard
<ActionFooter
  variant="standard"
  title="Order #12345"
  subtext1="Ready for dispatch"
  primaryAction={{ label: 'Proceed', onClick: handleProceed }}
  secondaryAction={{ label: 'Cancel', onClick: handleCancel }}
/>

// Action Footer - With Banner
<ActionFooter
  variant="with_banner"
  title="Approval Required"
  badge={{ variant: 'warning', children: 'Pending' }}
  banner={<WarningBanner message="This action requires manager approval" />}
  primaryAction={{ label: 'Approve', onClick: handleApprove }}
  tertiaryAction={{ label: 'Deny', onClick: handleDeny }}
/>

// Payment Checkout - Proceed to Pay
<PaymentCheckoutFooter
  type="proceed_to_pay"
  totalAmount="1,299"
  onPay={handlePay}
/>

// Payment Checkout - Full Options
<PaymentCheckoutFooter
  type="bill_details_wallet_options"
  totalAmount="1,299"
  paymentMethod="Google Pay UPI"
  walletBalance="500"
  onPay={handlePay}
  onViewBillDetails={handleBillDetails}
/>
```
</CodeGroup>
