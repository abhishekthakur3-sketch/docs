## Overview

Snackbars are lightweight feedback components used to inform users about system actions, updates, or outcomes. They appear temporarily and typically disappear automatically after a short duration. Unlike alerts, snackbars do not permanently occupy space in the layout — they provide quick contextual feedback while allowing users to continue interacting with the interface.

## Behavior

| Behavior | Description |
|----------|-------------|
| Position | Bottom-center of viewport (default); configurable to top |
| Auto-dismiss | 3–5 seconds; extend to 8–10s when CTAs are present |
| Animation | Slide up + fade in (300ms); slide down + fade out on dismiss |
| Stacking | Max 1 visible at a time; queue subsequent snackbars |
| Manual dismiss | Trailing ✕ icon or swipe gesture (mobile) |
| Z-index | Above all content, below modals |

---

## Snackbar vs Alert vs Toast

| Aspect | Snackbar | Alert (Inline) | Toast |
|--------|---------|---------------|-------|
| Position | Bottom-center overlay | Inline within layout | Corner overlay |
| Duration | Auto-dismiss (3–5s) | Persistent | Auto-dismiss (3–5s) |
| Interaction | Optional CTA + dismiss | Optional CTA + dismiss | Dismiss only |
| Use when | Action confirmation, undo | Contextual feedback near content | System notifications |
| Stacking | 1 at a time (queued) | Multiple inline | Multiple stacked |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Keep messages short and scannable
    - Use for temporary, non-critical feedback
    - Provide undo action for destructive operations
    - Auto-dismiss after appropriate duration
    - Position at bottom-center for consistency
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use for critical errors requiring immediate action
    - Don't show multiple snackbars simultaneously
    - Don't use for long instructions or content
    - Don't dismiss too quickly when CTAs are present
    - Don't rely solely on color — include icon + text
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="status"` with `aria-live="polite"` |
| Timing | Minimum 5 seconds visible; longer with CTAs |
| Screen reader | Message announced automatically via live region |
| Keyboard | CTA buttons focusable; Escape to dismiss |
| Contrast | Text ≥ 4.5:1 against snackbar background |
| Motion | Respect `prefers-reduced-motion` — disable slide animation |
| Focus | Don't steal focus from current task |

## Developer Handoff

<CodeGroup>
```tsx API
interface SnackbarProps {
  variant?: 'black' | 'white' | 'info' | 'success' | 'error' | 'warning';
  style?: 'filled' | 'outlined';
  type?: 'single' | 'dual';
  size?: 'lg' | 'md' | 'sm';
  icon?: ReactNode;
  title?: string;
  description?: string;
  actions?: SnackbarAction[];
  dismissible?: boolean;
  duration?: number;
  position?: 'bottom-center' | 'top-center';
  onDismiss?: () => void;
}

interface SnackbarAction {
  label: string;
  onClick: () => void;
  variant?: 'primary' | 'secondary';
}
```

```tsx Usage
// Success confirmation
<Snackbar variant="success" icon={<CheckIcon />}>
  Order placed successfully.
</Snackbar>

// With undo action
<Snackbar variant="black" actions={[{ label: 'Undo', onClick: handleUndo }]} dismissible>
  Item deleted.
</Snackbar>

// Error with retry
<Snackbar variant="error" icon={<ErrorIcon />} actions={[{ label: 'Retry', onClick: handleRetry }]}>
  Failed to send message.
</Snackbar>

// Dual text
<Snackbar variant="success" title="Order confirmed" description="Your order #4521 has been placed." />

// Outlined style
<Snackbar variant="info" style="outlined">
  New version available.
</Snackbar>

// Custom duration
<Snackbar variant="warning" duration={8000}>
  Connection unstable.
</Snackbar>
```

```tsx ARIA
// Proper live region implementation
<div role="status" aria-live="polite" aria-atomic="true">
  {visible && (
    <div className="snackbar">
      Order placed successfully.
      <button onClick={handleUndo}>Undo</button>
      <button onClick={handleDismiss} aria-label="Dismiss">✕</button>
    </div>
  )}
</div>
```
