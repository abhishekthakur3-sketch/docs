## Overview

The Cards system provides three distinct card categories: Selection Cards for single/multi-select choices, Info Cards for data display with slots, and Content Cards for rich content layouts. Each category supports multiple variants, sizes, and interactive states to cover a wide range of UI patterns.

## Card Categories

### Selection Cards

Selection Cards combine a selectable control (Radio Button or Checkbox) with text content, optional status indicator, and trailing icon. Used for single or multi-select choices within forms.

| Property | Options |
|----------|---------|
| Type | Radio Button, Checkbox |
| Size | Large (100px height), Small (88px height) |
| States | Default, Hover, Pressed, Focused, Disabled, Ghost |
| Optional Elements | Subtext, Status Indicator, Trailing Icon |

### Info Cards

Info Cards display data metrics with flexible slot-based layouts. Three styles support different content arrangements.

| Style | Description | Layout |
|-------|-------------|--------|
| Slots | Horizontal layout with leading slot, text set, and trailing slots | Icon + Title + Value + Trailing |
| Slot Top | Vertical layout with slot above text content | Slot on top, text below |
| Slot Banner | Full-width banner slot above text content | Banner image/slot + text below |

### Content Cards

Content Cards are rich container components for displaying structured content with various layout patterns.

| Variant | Description | Height |
|---------|-------------|--------|
| Standard | Basic card with image area and text content | 212px |
| Standard Type 2 | Alternative standard layout | 212px |
| Slot Banner | Card with full-width banner slot | 300px |
| Standard + Pills | Card with pill/tag elements | 176px |
| Standards + Icon Buttons | Card with icon button actions | 198px |
| Info Sets | Card with structured info key-value pairs | 272px |
| Badge Bottom | Card with badge element at bottom | 188px |
| Buttons Tacked | Card with tacked button actions at bottom | 188px |

## States

| State | Selection Card | Info Card | Content Card |
|-------|---------------|-----------|--------------|
| Default | White background, neutral border | White background, neutral border | White background, neutral border |
| Hover | Light gray background (#F9F9FB) | Light gray background (#F9F9FB) | Light gray background (#F9F9FB) |
| Pressed | White background, selected control | — | — |
| Focused | Blue focus ring (2px spread) | Blue focus ring (2px spread) | Blue focus ring (2px spread) |
| Disabled | Muted text (#CDCBCB), disabled control | Muted text (#CDCBCB) | Muted text (#CDCBCB) |
| Ghost | Gray skeleton placeholder (#EDEDED) | Gray skeleton placeholder (#F2F2F2) | Gray skeleton placeholder |

## Behavior

| Interaction | Result |
|-------------|--------|
| Tap selection card | Toggles radio/checkbox selection |
| Tap info card | Navigates or triggers action (if interactive) |
| Tap content card | Opens detail view or triggers action |
| Hover | Background shifts to light gray |
| Focus (keyboard) | Blue focus ring appears around card |
| Disabled state | All interactions blocked, muted appearance |
| Ghost state | Skeleton loading placeholder shown |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use Selection Cards for form-based single/multi-select choices
    - Use Info Cards for dashboard metrics and data summaries
    - Use Content Cards for browsable content lists
    - Keep card content concise and scannable
    - Use Ghost state for loading placeholders
    - Maintain consistent card sizes within a group
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't nest interactive cards inside other cards
    - Don't mix Selection Card types (radio/checkbox) in the same group
    - Don't overload cards with too much content
    - Don't use disabled state without clear reason
    - Don't use cards when a simple list would suffice
    - Don't remove the border in default state
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Selection Cards: `role="radio"` or `role="checkbox"` with `role="group"` on container |
| Labels | Each card must have an accessible label via `aria-label` or visible text |
| Selection state | `aria-checked="true/false"` for selection cards |
| Keyboard | Tab to navigate between cards, Space/Enter to select |
| Focus indicator | 2px blue focus ring (rgba(35, 150, 251, 0.4)) |
| Disabled | `aria-disabled="true"`, remove from tab order |
| Screen reader | Announce card content, type, and selection state |

## Developer Handoff

<CodeGroup>
```tsx API
// Selection Card
interface SelectionCardProps {
  type: 'radio' | 'checkbox';
  size?: 'large' | 'small';
  states?: 'default' | 'hover' | 'pressed' | 'focused' | 'disabled' | 'ghost';
  title: string;
  subtext?: string;
  statusIndicator?: boolean;
  trailingIcon?: ReactNode;
  selected?: boolean;
  onChange?: (selected: boolean) => void;
}

// Info Card
interface InfoCardProps {
  style: 'slots' | 'slot-top' | 'slot-banner';
  states?: 'default' | 'hover' | 'focused' | 'disabled' | 'ghost';
  title?: string;
  subtextTop?: string;
  subtextBottom?: string;
  slotLeading?: ReactNode;
  slotTrailing?: ReactNode;
  trailingIcon?: boolean;
  onClick?: () => void;
}

// Content Card
interface ContentCardProps {
  variant: 'standard' | 'standard-type-2' | 'slot-banner' | 'standard-pills'
    | 'standard-icon-buttons' | 'info-sets' | 'badge-bottom' | 'buttons-tacked';
  states?: 'default' | 'hover' | 'focused' | 'disabled' | 'ghost';
  title?: string;
  description?: string;
  media?: ReactNode;
  actions?: ReactNode;
  onClick?: () => void;
}
```

```tsx Usage
// Selection Card with Radio Button
<SelectionCard
  type="radio"
  size="large"
  title="Express Delivery"
  subtext="Delivered within 24 hours"
  statusIndicator
  selected={selectedOption === 'express'}
  onChange={() => setSelectedOption('express')}
/>

// Info Card with Slots
<InfoCard
  style="slots"
  title="100"
  subtextTop="Total Orders"
  subtextBottom="This is a sample message"
  slotLeading={<OrderIcon />}
  trailingIcon
/>

// Content Card
<ContentCard
  variant="standard"
  title="Shipment Details"
  description="Track your package status"
  media={<img src="/shipment.png" alt="Shipment" />}
/>
```
</CodeGroup>
