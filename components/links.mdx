## Overview

Links are lightweight navigational elements embedded within text, lists, or UI structures. Unlike buttons, links represent navigation rather than actions. They support optional leading and trailing icons for additional visual context such as external navigation or directional cues. Links should remain visually subtle while clearly indicating interactivity.

| # | Element | Required | Description |
|---|---------|----------|-------------|
| 1 | Leading Icon | No | Icon before the label for visual context |
| 2 | Label | Yes | Clickable text describing the navigation destination |
| 3 | Trailing Icon | No | Icon after the label (external link indicator, arrow, etc.) |

<Warning>Icons should support the link meaning but never replace the text label. The label is always required.</Warning>

## Styles

| Style | Color | Hex | Use Case |
|-------|-------|-----|----------|
| Blue | Blue text | `#2396FB` | Default style, strong visibility, clear interaction cue |
| Black | Black text | `#000000` | Neutral/minimal interfaces, darker UI contexts |
| White | White text | `#FFFFFF` | Dark backgrounds, ensuring readability and contrast |

<Tip>Use Blue for most links. Reserve Black for minimal interfaces and White exclusively for dark backgrounds.</Tip>

## States

| State | Visual Treatment | Interaction |
|-------|-----------------|-------------|
| Default | Colored text, no underline | Clickable, pointer cursor |
| Hover | Underline appears | Visual feedback on pointer hover |
| Focused | Underline + focus ring (2px outline) | Keyboard focus visible |
| Disabled | 40% opacity, no underline | Not interactive, `cursor: default` |

## Behavior

### Navigation

- Clicking a link navigates to the target location
- Links may open content in the same page, a new tab, or an external destination
- Use `target="_blank"` with `rel="noopener noreferrer"` for external links
- Disabled links remain visible but cannot be interacted with

### Icon Usage

| Pattern | Example | Use Case |
|---------|---------|----------|
| Leading icon | 📄 View document | Contextual icon before label |
| Trailing icon | Open in new tab ↗ | External link indicator |
| Both icons | 📄 View document ↗ | Combined context + direction |
| No icons | Learn more | Simple inline navigation |

### Inline vs Standalone

- Inline links are embedded within body text and should match the surrounding font size
- Standalone links appear independently and can use any size
- Inline links should use underline on default state for discoverability within text

---

## Link vs Button

| Feature | Link | Button |
|---------|------|--------|
| Purpose | Navigation | Action |
| Visual weight | Lightweight, text-based | Prominent, filled/outlined |
| Underline | On hover or always | Never |
| Use case | Go to page, open resource | Submit, delete, save |
| HTML element | `<a>` | `<button>` |
| Keyboard | `Enter` to activate | `Enter` or `Space` |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use descriptive link labels that explain the destination
    - Maintain visual distinction between links and regular text
    - Ensure adequate color contrast against the background
    - Use trailing icons to indicate external navigation
    - Keep link text concise and actionable
    - Match link size to surrounding typography
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use links for primary actions (use buttons)
    - Don't use vague labels like "Click here" or "Read more"
    - Don't hide link meaning behind icons only
    - Don't use links without clear context
    - Don't use Blue links on blue backgrounds
    - Don't remove underline from inline links in body text
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Element | Use `<a>` with a valid `href` attribute |
| Label | Link text must clearly describe the destination |
| Keyboard | `Tab` to focus, `Enter` to activate |
| Focus | Visible focus ring (2px outline, matching link color) |
| External links | Add `aria-label` or visible icon indicating new tab |
| Disabled | Use `aria-disabled="true"`, remove `href`, set `tabindex="-1"` |
| Color contrast | Link color must meet 3:1 contrast against surrounding text and 4.5:1 against background |
| Screen readers | Avoid "click here" — use descriptive text like "View order details" |

<Info>For external links, include a trailing icon (↗) and add screen reader text like "opens in a new tab" via `aria-label` or visually hidden text.</Info>

## Developer Handoff

<CodeGroup>
```tsx API
import { Link } from '@tarmac/design-system';

interface LinkProps {
  href: string;                              // Navigation URL (required)
  style?: 'blue' | 'black' | 'white';       // Default: 'blue'
  size?: 'xlarge' | 'large' | 'medium' | 'small' | 'xsmall'; // Default: 'medium'
  disabled?: boolean;                        // Disable interaction
  external?: boolean;                        // Opens in new tab
  leadingIcon?: React.ReactNode;             // Icon before label
  trailingIcon?: React.ReactNode;            // Icon after label
  children: React.ReactNode;                 // Label text
}
```

```tsx Usage
// Basic blue link
<Link href="/orders">View all orders</Link>

// External link with trailing icon
<Link href="https://docs.example.com" external>
  Documentation
</Link>

// Black link, small size
<Link href="/settings" style="black" size="small">
  Account settings
</Link>

// With leading icon
<Link href="/downloads" leadingIcon={<DownloadIcon />}>
  Download report
</Link>

// Disabled link
<Link href="/premium" disabled>
  Premium features
</Link>

// White link on dark background
<div style={{ background: '#1A1A1A', padding: '16px' }}>
  <Link href="/help" style="white" size="xsmall">
    Need help?
  </Link>
</div>
```

```tsx ARIA
// Standard link
<a
  href="/orders"
  style={{ color: '#2396FB', textDecoration: 'none' }}
>
  View all orders
</a>

// External link
<a
  href="https://docs.example.com"
  target="_blank"
  rel="noopener noreferrer"
  style={{ color: '#2396FB' }}
>
  Documentation
  <span aria-hidden="true"> ↗</span>
  <span style={{
    position: 'absolute',
    width: '1px',
    height: '1px',
    overflow: 'hidden',
    clip: 'rect(0,0,0,0)',
  }}>
    (opens in a new tab)
  </span>
</a>

// Disabled link
<a
  aria-disabled="true"
  tabIndex={-1}
  style={{ color: '#2396FB', opacity: 0.4, cursor: 'default' }}
>
  Premium features
</a>
```
