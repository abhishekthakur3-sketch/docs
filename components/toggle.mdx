## Overview

Toggles allow users to switch a system setting between two states instantly. Unlike checkboxes, toggles represent a direct action that takes effect immediately. The thumb slides across the track to visually communicate the current state. Toggles are commonly used in settings panels, feature controls, and configuration interfaces.

## Behavior

### State Switching

- Clicking the toggle moves the thumb from left (off) to right (on), or vice versa
- The state change is applied immediately — no confirmation step
- Visual feedback (track color + thumb position) updates instantly
- The thumb animates smoothly across the track (150–200ms transition)

### Label Pairing

- Toggles should always be paired with a descriptive label
- The label should describe the feature being controlled, not the state
- Place labels to the left or right of the toggle, never above or below

### Form Integration

- Toggles can be used in forms but should not require a submit action
- For settings that need confirmation, use a checkbox instead
- Disabled toggles remain visible to communicate the existence of a feature

---

## Toggle vs Checkbox vs Radio

| Feature | Toggle | Checkbox | Radio |
|---------|--------|----------|-------|
| State change | Immediate | On submit | On submit |
| Options | Binary (on/off) | Multiple selections | Single selection |
| Best for | Settings, preferences | Form fields, multi-select | Mutually exclusive choices |
| Visual feedback | Thumb slides, track changes color | Checkmark appears | Dot fills |
| Confirmation | Not needed | May need submit | May need submit |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Pair toggles with clear, descriptive labels
    - Use toggles for immediate on/off state changes
    - Provide clear visual distinction between on and off states
    - Use consistent toggle style within a settings panel
    - Place the label describing the feature, not the state
    - Use Green for "active" and contextual success states
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use toggles for actions requiring confirmation
    - Don't use toggles without clear context or labels
    - Don't hide the meaning behind icons only
    - Don't use toggles for multi-option selections
    - Don't use toggles in place of radio buttons
    - Don't mix Filled and Outlined styles in the same panel
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Role | `role="switch"` on the toggle element |
| State | `aria-checked="true"` when on, `"false"` when off |
| Label | `aria-labelledby` or `aria-label` describing the controlled feature |
| Keyboard | `Space` or `Enter` to toggle state |
| Focus | Visible focus ring (3px outline) when focused via keyboard |
| Disabled | `aria-disabled="true"`, skip in tab order or use `tabindex="-1"` |
| Announcement | Screen readers should announce: "[Label] switch, on/off" |

<Info>Use `role="switch"` instead of `role="checkbox"` — it provides more accurate semantics for toggle controls and better screen reader announcements.</Info>

## Developer Handoff

<CodeGroup>
```tsx API
import { Toggle } from '@tarmac/design-system';

interface ToggleProps {
  checked?: boolean;                       // Current state
  defaultChecked?: boolean;                // Initial state (uncontrolled)
  onChange?: (checked: boolean) => void;   // State change callback
  style?: 'filled' | 'outlined';          // Default: 'filled'
  color?: 'black' | 'blue' | 'red' | 'green'; // Default: 'black'
  size?: 'large' | 'small';               // Default: 'large'
  disabled?: boolean;                      // Disable interaction
  label?: string;                          // Accessible label text
  labelPosition?: 'left' | 'right';       // Default: 'right'
  id?: string;                             // HTML id for label association
}
```

```tsx Usage
// Basic toggle
<Toggle
  label="Dark Mode"
  checked={isDarkMode}
  onChange={(checked) => setDarkMode(checked)}
/>

  label="Auto-sync"
  color="blue"
  size="small"
  defaultChecked={true}
/>

  label="Active Status"
  style="outlined"
  color="green"
  checked={isActive}
  onChange={setIsActive}

  label="Premium Feature"
  disabled
```
