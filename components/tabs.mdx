## Overview

Tabs are navigation components that organize content into switchable sections. Each tab cell acts as an interactive control, and tab groups arrange these cells horizontally or vertically. The system supports rich optional elements like icons, badges, pills, and subtext for contextual metadata.

A single tab cell can contain up to 9 optional elements:

| # | Element | Required | Description |
|---|---------|----------|-------------|
| 1 | Checkbox | No | Selection control for multi-select tab patterns |
| 2 | Status Indicator | No | Dot or icon showing status (active, error, new) |
| 3 | Leading Icon | No | Icon before the title for visual context |
| 4 | Title | Yes | Primary label text — the only required element |
| 5 | Subtext | No | Secondary description below the title |
| 6 | Badge | No | Status badge attached to the tab |
| 7 | Notification Count | No | Numeric count indicator |
| 8 | Pill | No | Pill label for metadata or category |
| 9 | Trailing Icon | No | Icon after the title (dropdown arrow, close, etc.) |

<Warning>Only the Title is required. All other elements are optional and should be used sparingly to avoid visual clutter.</Warning>

## Tab Count

Tab groups support 2 to 8 tabs. The system adapts layout based on count:

| Count | Layout Behavior |
|-------|----------------|
| 2–4 | Tabs distribute evenly or align left |
| 5–6 | Tabs may require scrolling on mobile |
| 7–8 | Horizontal scroll recommended; consider vertical orientation |

## Behavior

### Selection

- Only one tab can be active at a time within a group
- Selecting a tab immediately switches the visible content panel
- The previously active tab returns to its default (inactive) state
- A tab group must always have exactly one active tab — empty states are not allowed

### Scrolling

- When tabs overflow the container width, horizontal scrolling is enabled
- Scroll indicators (fade or arrow) should appear at the edges
- The active tab should always be scrolled into view automatically

### Content Transition

- Content panels switch instantly (no animation by default)
- Optional: fade or slide transitions can be applied for smoother UX
- Content height should adjust dynamically — avoid fixed-height panels

### Responsive Behavior

| Breakpoint | Behavior |
|-----------|----------|
| Desktop (≥1024px) | Full tab bar, all tabs visible |
| Tablet (768–1023px) | Scrollable if >5 tabs |
| Mobile (<768px) | Scrollable, consider Button Style or vertical orientation |

## Tabs vs Accordion vs Pills

| Feature | Tabs | Accordion | Pills |
|---------|------|-----------|-------|
| Content switching | One panel visible at a time | Multiple panels can be open | Selection-based filtering |
| Orientation | Horizontal or Vertical | Vertical only | Horizontal |
| Best for | Related content sections | FAQ, expandable details | Filters, categories |
| Tab count | 2–8 | Unlimited sections | Unlimited |
| Nesting | Not recommended | Supported | Not applicable |
| Mobile behavior | Scrollable | Stacked | Wrapping |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use tabs to organize related content within the same context
    - Keep tab labels short (1–2 words)
    - Always have one tab selected by default
    - Use consistent tab styles within a page
    - Provide clear, distinct labels for each tab
    - Use icons sparingly and only when they add clarity
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use tabs for sequential steps (use Stepper instead)
    - Don't mix Regular and Button Style in the same group
    - Don't exceed 8 tabs in a single group
    - Don't use tabs for primary navigation (use nav bar)
    - Don't nest tab groups inside tab panels
    - Don't use long labels that cause wrapping
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Role | `role="tablist"` on container, `role="tab"` on each tab, `role="tabpanel"` on content |
| Active state | `aria-selected="true"` on active tab, `"false"` on others |
| Keyboard — Horizontal | `←` `→` arrow keys to move between tabs |
| Keyboard — Vertical | `↑` `↓` arrow keys to move between tabs |
| Home / End | `Home` moves to first tab, `End` moves to last tab |
| Activation | `Enter` or `Space` activates the focused tab |
| Focus management | Focus moves to tab, not to panel — panel is linked via `aria-controls` |
| Disabled tabs | `aria-disabled="true"`, skip in keyboard navigation |
| Labels | Each tab must have a visible text label or `aria-label` |

<Info>Arrow key navigation should wrap — pressing `→` on the last tab moves focus to the first tab, and vice versa.</Info>
