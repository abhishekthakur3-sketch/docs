## Overview

Pills are lightweight UI components used to display selections, filters, or categorized attributes in a compact format. They are visually distinct through their rounded shape and commonly used in filtering systems, metadata displays, and removable selections. Unlike badges, pills are often interactive and may allow users to remove, toggle, or modify the displayed value.

---

## Types

| Type | Emphasis | Visual Treatment | Use When |
|------|----------|-----------------|----------|
| Solid | Highest | Filled background, contrasting text | Selected or active states |
| Subtle | Low | Softer background color | Secondary or supporting information |
| Outlined | Low | Border with transparent background | Minimal visual weight needed |
| Disabled | None | Muted appearance, 40% opacity | Pill is inactive or unavailable |
| Ghost | Minimal | No background, text only | Pill should blend into interface |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Keep pill labels concise and scannable
    - Use semantic variants consistently
    - Place pills near the content they represent
    - Use icons only when they improve clarity
    - Provide close icon for removable pills
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use long phrases inside pills
    - Don't overuse pills in dense interfaces
    - Don't mix too many visual variants in the same context
    - Don't use pills as a replacement for structured navigation
    - Don't rely solely on color for meaning
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Contrast | Text ≥ 4.5:1 against background |
| Color independence | Don't rely on color alone — include text labels |
| Screen reader | Use `aria-label` for icon-only or status-only pills |
| Removable pills | Close button needs `aria-label="Remove [pill name]"` |
| Keyboard | Tab to focus, Enter/Space to activate, Delete to remove |
| Focus ring | Visible 2px focus ring on keyboard navigation |
| Role | Use `role="option"` in filter groups, `role="listbox"` for container |

## Developer Handoff

<CodeGroup>
```tsx API
interface PillProps {
  variant?: 'black' | 'white' | 'coal' | 'blue' | 'success' | 'error' | 'warning' | 'legacy_blue';
  type?: 'solid' | 'subtle' | 'outlined' | 'disabled' | 'ghost';
  size?: 'lg' | 'md' | 'sm';
  icon?: ReactNode;
  iconPosition?: 'left' | 'right';
  statusMark?: boolean;
  closable?: boolean;
  onClose?: () => void;
  children?: ReactNode;
}
```

```tsx Usage
// Solid pills
<Pill variant="black" type="solid">Filter</Pill>
<Pill variant="success" type="solid">Active</Pill>
<Pill variant="error" type="solid">Failed</Pill>

// Subtle
<Pill variant="blue" type="subtle">Draft</Pill>

// Outlined
<Pill variant="coal" type="outlined">Category</Pill>

// With leading icon
<Pill variant="success" icon={<CheckIcon />}>Verified</Pill>

// Removable pill
<Pill variant="black" closable onClose={() => handleRemove()}>React</Pill>

// With status mark
<Pill variant="success" statusMark>Online</Pill>

// Sizes
<Pill variant="black" size="lg">Large</Pill>
<Pill variant="black" size="sm">Small</Pill>

// Disabled
<Pill variant="black" type="disabled">Unavailable</Pill>

// Ghost
<Pill variant="black" type="ghost">Subtle</Pill>
```

```tsx Filter Group
// Pill group for active filters
<PillGroup aria-label="Active filters">
  <Pill variant="blue" closable>Status: Active</Pill>
  <Pill variant="blue" closable>Region: North</Pill>
  <Pill variant="blue" closable>Type: Express</Pill>
</PillGroup>

// Selectable pills
<PillGroup selectable>
  <Pill variant="coal" type="outlined">Today</Pill>
  <Pill variant="coal" type="outlined">This Week</Pill>
  <Pill variant="coal" type="outlined">This Month</Pill>
</PillGroup>
```
</CodeGroup>
