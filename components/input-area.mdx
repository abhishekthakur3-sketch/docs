## Overview

Input areas extend beyond standard input fields to support multiline text entry, tag-based inputs, and segmented OTP verification codes. They share the same three-layer structure (Title Container, Input Area Container, Subtext Container) but provide additional flexibility for structured data entry and larger content. Input areas commonly appear in forms, messaging interfaces, configuration panels, and authentication flows.

## Styles

| Style | Description | Use Case |
|-------|-------------|----------|
| Standard | Multiline text input with optional resize handle | Comments, notes, descriptions, messages |
| With Tags | Input that converts entries into removable tags | Filters, keywords, labels, multi-value entries |
| OTP 4 Cells | 4-digit segmented verification input | Authentication, secure login |
| OTP 6 Cells | 6-digit segmented verification input | Two-factor authentication, security codes |

|-------|----------|

## Behavior

### Standard (Multiline)
- Text wraps to new lines as the user types
- Optional resize handle allows vertical resizing
- Character counter updates dynamically

### With Tags
- Pressing Enter or comma converts text into a tag
- Tags appear as removable chips inside the input
- Clicking × on a tag removes it
- Input area expands vertically as tags accumulate

### OTP Cells
- Each cell accepts a single digit
- Focus automatically advances to the next cell after entry
- Backspace moves focus to the previous cell
- Paste support fills all cells from clipboard

---

## Input Area vs Input Field

| Feature | Input Area | Input Field |
|---------|-----------|-------------|
| Lines | Multiline | Single line |
| Tags | Supported (With Tags style) | Not supported |
| OTP | Supported (4/6 cells) | Not supported |
| Resize | Optional drag handle | Fixed height |
| Best for | Long text, tags, OTP | Short text, names, emails |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use Standard style for long-form text (comments, descriptions)
    - Use With Tags for multi-value inputs (filters, keywords)
    - Use OTP cells for verification codes
    - Provide character counters for text areas
    - Auto-advance focus in OTP inputs
    - Show clear validation feedback
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use multiline input for short single-line text
    - Don't use OTP cells for general numeric input
    - Don't replace labels with placeholders
    - Don't hide validation messages
    - Don't disable resize without good reason
    - Don't mix OTP cell counts in the same flow
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Label | Every input area must have a visible label via `<label>` |
| Keyboard | `Tab` to focus, standard text editing keys |
| OTP focus | Auto-advance focus between cells, support `Backspace` to go back |
| Validation | `aria-invalid="true"` on error, `aria-describedby` for messages |
| Resize | Drag handle must be keyboard accessible |
| Screen readers | Announce character count, tag additions/removals, OTP progress |
| Paste | OTP inputs should support paste from clipboard |

<Info>For OTP inputs, use `inputmode="numeric"` and `autocomplete="one-time-code"` to trigger the correct mobile keyboard and autofill behavior.</Info>

## Developer Handoff

<CodeGroup>
```tsx API
import { InputArea, TagInput, OTPInput } from '@tarmac/design-system';

interface InputAreaProps {
  label?: string;
  placeholder?: string;
  helperText?: string;
  errorMessage?: string;
  type?: 'regular' | 'success' | 'error' | 'info';
  size?: 'large' | 'default' | 'small';
  disabled?: boolean;
  maxLength?: number;
  rows?: number;
  resize?: 'none' | 'vertical' | 'both';
  value?: string;
  onChange?: (value: string) => void;
}

interface TagInputProps extends Omit<InputAreaProps, 'rows' | 'resize'> {
  tags: string[];
  onTagAdd?: (tag: string) => void;
  onTagRemove?: (tag: string) => void;
  maxTags?: number;
}

interface OTPInputProps {
  length: 4 | 6;
  type?: 'regular' | 'success' | 'error' | 'info';
  size?: 'large' | 'default' | 'small';
  disabled?: boolean;
  label?: string;
  helperText?: string;
  onComplete?: (code: string) => void;
  onChange?: (code: string) => void;
  autoFocus?: boolean;
}
```

```tsx Usage
// Standard multiline
<InputArea
  label="Description"
  placeholder="Enter description..."
  maxLength={500}
  rows={4}
/>

// With Tags
<TagInput
  label="Keywords"
  tags={['React', 'TypeScript']}
  onTagAdd={(tag) => addTag(tag)}
  onTagRemove={(tag) => removeTag(tag)}
  placeholder="Add keyword..."
/>

// OTP 4 cells
<OTPInput
  length={4}
  label="Verification Code"
  helperText="Code sent to your phone"
  onComplete={(code) => verifyCode(code)}
  autoFocus
/>

// OTP 6 cells with error
<OTPInput
  length={6}
  type="error"
  label="Enter Code"
  helperText="Invalid code. Please try again."
  onComplete={handleVerify}
/>
```
