## Overview

Tooltips are small contextual overlays that appear when users hover over, focus on, or tap an element. They provide helpful information, clarification, or additional context without cluttering the interface. Tooltips are essential for improving usability by explaining icons, abbreviations, or providing supplementary details.

## Placements

| Placement | Arrow Position | Use When |
|-----------|---------------|----------|
| Top | Bottom center | Default — most common placement |
| Right | Left center | Sidebar elements, left-aligned triggers |
| Bottom | Top center | Top navigation, header elements |
| Left | Right center | Right-aligned triggers, end-of-row actions |
| Auto | Dynamic | Insufficient space — auto-adjusts to viewport |

---

## States

| State | Visual | Behavior |
|-------|--------|----------|
| Hidden | Not visible | Default — no interaction |
| Visible | Fade-in (150ms) | Appears after 300ms hover delay |
| Transitioning | Opacity animation | Smooth fade between hidden/visible |

---

## Interaction Behavior

| Trigger | Show | Hide | Notes |
|---------|------|------|-------|
| Hover (Desktop) | After 300ms delay | When cursor moves away | Remains visible while hovering tooltip |
| Focus (Keyboard) | Immediately on focus | On blur or Escape key | No delay for keyboard users |
| Touch (Mobile) | First tap shows tooltip | Tap outside or scroll | Second tap activates trigger action |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Explain icon-only buttons
    - Show keyboard shortcuts
    - Clarify abbreviations or truncated text
    - Keep content to 1–2 short sentences
    - Support keyboard focus
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use for critical information users must see
    - Don't put interactive elements inside tooltips
    - Don't repeat visible text in the tooltip
    - Don't use for form validation errors
    - Don't use for complex content (use Popover)
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Keyboard | Show on focus, dismiss on Escape |
| Screen reader | Use `role="tooltip"` and `aria-describedby` |
| Icon triggers | Add `aria-label` for icon-only triggers |
| Contrast | Text ≥ 4.5:1 against tooltip background |
| Touch devices | Provide alternative access (tap to show) |
| No traps | Tooltip must not trap keyboard focus |
| Timing | 300ms show delay, no hide delay |

## Developer Handoff

<CodeGroup>
```tsx API
interface TooltipProps {
  content: ReactNode;
  children: ReactElement;
  placement?: 'top' | 'right' | 'bottom' | 'left' | 'auto';
  variant?: 'default' | 'light';
  arrow?: boolean;
  delay?: number;
  disabled?: boolean;
  offset?: number;
}
```

```tsx Usage
// Basic tooltip
<Tooltip content="Save your changes">
  <Button>Save</Button>
</Tooltip>

// Icon with tooltip
<Tooltip content="Delete item">
  <IconButton aria-label="Delete">
    <DeleteIcon />
  </IconButton>
</Tooltip>

// Custom placement
<Tooltip content="User settings" placement="right">
  <Avatar src="/user.jpg" />
</Tooltip>

// Light variant
<Tooltip content="Help documentation" variant="light">
  <HelpIcon />
</Tooltip>

// Without arrow
<Tooltip content="Notification" arrow={false}>
  <BellIcon />
</Tooltip>

// Keyboard shortcut
<Tooltip content="Save (⌘ + S)">
  <Button>Save</Button>
</Tooltip>
```

```tsx ARIA
// Proper ARIA implementation
<button
  aria-describedby="tooltip-save"
  onMouseEnter={showTooltip}
  onFocus={showTooltip}
>
  <SaveIcon />
</button>
<div role="tooltip" id="tooltip-save">
  Save your changes
</div>
```
