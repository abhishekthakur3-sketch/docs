## Overview

Tooltips are small contextual overlays that appear when users hover over, focus on, or tap an element. They provide helpful information, clarification, or additional context without cluttering the interface. Tooltips are essential for improving usability by explaining icons, abbreviations, or providing supplementary details.

## States

| State | Visual | Behavior |
|-------|--------|----------|
| Hidden | Not visible | Default — no interaction |
| Visible | Fade-in (150ms) | Appears after 300ms hover delay |
| Transitioning | Opacity animation | Smooth fade between hidden/visible |

---

## Interaction Behavior

| Trigger | Show | Hide | Notes |
|---------|------|------|-------|
| Hover (Desktop) | After 300ms delay | When cursor moves away | Remains visible while hovering tooltip |
| Focus (Keyboard) | Immediately on focus | On blur or Escape key | No delay for keyboard users |
| Touch (Mobile) | First tap shows tooltip | Tap outside or scroll | Second tap activates trigger action |

---

## Edge Cases

<Warning>These scenarios require special handling in implementation.</Warning>

| Scenario | Expected Behavior | Implementation Note |
|----------|-------------------|---------------------|
| Long content overflow | Text wraps within max-width, never truncates | Set `max-width: 240px` (or 320px for large), `word-wrap: break-word` |
| Viewport boundary | Tooltip repositions to stay fully visible | Use collision detection; flip placement if clipped |
| Scroll container | Tooltip dismisses on scroll | Attach scroll listener to nearest scrollable ancestor |
| Nested tooltips | Only one tooltip visible at a time | Dismiss active tooltip before showing new one |
| Disabled trigger | Tooltip still shows on disabled elements | Wrap disabled element in a `<span>` for event handling |
| Touch + hover device | Prefer hover behavior, fallback to tap | Detect `pointer: fine` media query for hover support |
| Portal rendering | Tooltip renders in document body | Avoids z-index and overflow:hidden issues from parent |
| Rapid hover | Debounce show/hide to prevent flicker | Cancel pending show if cursor leaves before delay |

---

## When to Use (and When Not To)

| Use Tooltip | Use Popover Instead | Use Inline Text Instead |
|-------------|---------------------|------------------------|
| Icon-only button labels | Interactive content (links, buttons) | Critical information users must see |
| Keyboard shortcut hints | Multi-step instructions | Form validation errors |
| Truncated text full value | Content requiring user action | Required field descriptions |
| Abbreviation definitions | Rich media (images, videos) | Persistent help text |
| Chart data point values | Forms or input fields | Status messages |

## Usage Guidelines

  <Card title="✓ Do" icon="check" color="#16A34A">
    - Explain icon-only buttons
    - Show keyboard shortcuts
    - Clarify abbreviations or truncated text
    - Keep content to 1–2 short sentences
    - Support keyboard focus
    - Use sentence case for tooltip text
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use for critical information users must see
    - Don't put interactive elements inside tooltips
    - Don't repeat visible text in the tooltip
    - Don't use for form validation errors
    - Don't use for complex content (use Popover)
    - Don't rely on hover-only for mobile users
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Keyboard | Show on focus, dismiss on Escape |
| Screen reader | Use `role="tooltip"` and `aria-describedby` |
| Icon triggers | Add `aria-label` for icon-only triggers |
| Contrast | Text ≥ 4.5:1 against tooltip background |
| Touch devices | Provide alternative access (tap to show) |
| No traps | Tooltip must not trap keyboard focus |
| Timing | 300ms show delay, no hide delay |
| Persistence | Tooltip remains while hovering tooltip itself |

## Developer Handoff

<CodeGroup>
```tsx API
interface TooltipProps {
  content: ReactNode;
  children: ReactElement;
  placement?: 'top' | 'right' | 'bottom' | 'left' | 'auto';
  variant?: 'default' | 'light';
  arrow?: boolean;
  delay?: number;
  disabled?: boolean;
  offset?: number;
}
```

```tsx Usage
// Basic tooltip
<Tooltip content="Save your changes">
```
