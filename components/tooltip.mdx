# Tooltip

## Overview

Tooltips are small contextual overlays that appear when users hover over, focus on, or tap an element. They provide helpful information, clarification, or additional context without cluttering the interface. Tooltips are essential for improving usability by explaining icons, abbreviations, or providing supplementary details about UI elements.

## Anatomy

The Tooltip component consists of the following elements:

- **Trigger**: The element that activates the tooltip (button, icon, text, etc.)
- **Container**: The floating overlay that contains the tooltip content
- **Content**: Text or simple content displayed in the tooltip
- **Arrow** (optional): Visual pointer connecting tooltip to trigger
- **Backdrop** (optional): Subtle overlay for mobile touch interactions

<Info>Tooltips should contain concise, helpful information - keep content to 1-2 short sentences maximum.</Info>

## Variants

### Default (Dark)
Standard tooltip with dark background and light text.

```jsx
<Tooltip content="Save your changes">
  <Button>Save</Button>
</Tooltip>
```

### Light
Light background variant for use on dark surfaces.

```jsx
<Tooltip content="Delete item" variant="light">
  <IconButton><DeleteIcon /></IconButton>
</Tooltip>
```

### Rich Content
Supports formatted content with multiple lines or elements.

```jsx
<Tooltip content={
  <div>
    <strong>Keyboard Shortcut</strong>
    <p>Cmd + S</p>
  </div>
}>
  <Button>Save</Button>
</Tooltip>
```

## Sizes

Tooltips adapt their size based on content, with a maximum width constraint:

| Size | Max Width | Usage |
|------|-----------|-------|
| **Default** | 240px | Standard tooltips with 1-2 lines |
| **Large** | 320px | Longer descriptions or rich content |

<Tip>Keep tooltip content concise to avoid large, intrusive overlays.</Tip>

## States

### Hidden
Default state when tooltip is not visible.

### Visible
Tooltip appears after hover/focus delay.
- Smooth fade-in animation (150ms)
- Positioned relative to trigger
- Remains visible during interaction

### Transitioning
Animation state during show/hide.
- 150-200ms fade transition
- Slight vertical movement (optional)

## Interaction Behavior

### Hover (Desktop)
- Tooltip appears after 300ms delay
- Remains visible while hovering trigger or tooltip
- Dismisses when cursor moves away
- Smooth fade-in animation (150ms)

### Focus (Keyboard)
- Tooltip appears immediately on focus
- Remains visible while element has focus
- Dismisses when focus moves away
- Dismisses on Escape key press

### Touch (Mobile)
- First tap shows tooltip
- Second tap activates trigger action
- Tap outside dismisses tooltip
- Scroll dismisses tooltip

### Positioning
- Default placement: top
- Auto-adjusts if insufficient space
- Maintains 8px offset from trigger
- Arrow points to trigger center

<Warning>Tooltips automatically reposition to stay within viewport boundaries.</Warning>

## Usage Guidelines

<AccordionGroup>
  <Accordion title="When to Use" icon="check">
    - Explaining icon-only buttons
    - Providing additional context for truncated text
    - Clarifying abbreviations or technical terms
    - Showing keyboard shortcuts
    - Displaying full values for charts or data visualizations
    - Offering help text for form fields
  </Accordion>
  
  <Accordion title="When Not to Use" icon="xmark">
    - For critical information users must see
    - For complex interactive content (use Popover)
    - For mobile-primary experiences (consider alternatives)
    - When the information should always be visible
    - For form validation errors (use inline error messages)
  </Accordion>
</AccordionGroup>

### Do's and Don'ts

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#10b981">
    **Be concise**
    - "Save changes"
    - "Delete item"
    - "⌘ + S"
    
    **Add value**
    - Explain icon meaning
    - Show keyboard shortcuts
    - Clarify abbreviations
    
    **Support keyboard**
    - Show on focus
    - Dismiss with Escape
  </Card>
  
  <Card title="✗ Don't" icon="xmark" color="#ef4444">
    **Avoid long text**
    - ~~"Click this button to save all your changes to the database"~~
    
    **Don't repeat visible text**
    - ~~Button: "Save" Tooltip: "Save"~~
    
    **Don't use for critical info**
    - Error messages
    - Required instructions
    - Essential context
  </Card>
</CardGroup>

<Tip>Keep tooltip content to 1-2 short sentences maximum for best readability.</Tip>

<Warning>Never put interactive elements inside tooltips - use a Popover instead.</Warning>

### Content Guidelines

<Steps>
  <Step title="Be concise and specific">
    Use clear, direct language. Avoid unnecessary words.
  </Step>
  <Step title="Use sentence case">
    "Save your changes" not "Save Your Changes"
  </Step>
  <Step title="Don't repeat visible text">
    Add new information, don't duplicate what's already shown
  </Step>
  <Step title="Include units or formats">
    "Last updated: 2 hours ago" or "Size: 2.4 MB"
  </Step>
</Steps>

## Accessibility

<Info>Tooltips must be accessible via keyboard and screen readers to meet WCAG 2.1 AA standards.</Info>

### Keyboard Navigation

<ResponseField name="Tab" type="key">
  Focus trigger element to show tooltip
</ResponseField>

<ResponseField name="Escape" type="key">
  Dismiss tooltip and return focus
</ResponseField>

<ResponseField name="Shift + Tab" type="key">
  Move to previous element (dismisses tooltip)
</ResponseField>

### ARIA Implementation

<CodeGroup>

### Accessibility Checklist

<AccordionGroup>
  <Accordion title="✓ Keyboard Support" icon="keyboard">

  <Accordion title="✓ Screen Reader Support" icon="audio-description">

    - [ ] No redundant announcements
  </Accordion>
  
  <Accordion title="✓ Visual Requirements" icon="eye">
    - [ ] Sufficient color contrast (4.5:1)
    - [ ] Visible against all backgrounds
    - [ ] Doesn't obscure important content
    - [ ] Readable font size (14px minimum)
  </Accordion>
  
  <Accordion title="✓ Touch Device Support" icon="mobile">
    - [ ] Tap to show tooltip
    - [ ] Tap outside to dismiss
    - [ ] Alternative access method provided
    - [ ] No hover-only tooltips
  </Accordion>
</AccordionGroup>

### Touch Device Considerations

<Warning>Hover-only tooltips don't work on touch devices. Always provide alternative access methods.</Warning>

**Recommended approaches:**
- Show tooltip on first tap, activate on second tap
- Provide info icon for explicit tooltip access
- Display information inline on mobile
- Use expandable sections instead of tooltips

## Engineering Implementation

### Component API

```typescript
interface TooltipProps {
  // Content
  content: ReactNode;
  children: ReactElement;
  
  // Appearance
  placement?: 'top' | 'right' | 'bottom' | 'left' | 'auto';
  variant?: 'default' | 'light';
  arrow?: boolean;
  
  // Behavior
  delay?: number;
  disabled?: boolean;
  offset?: number;
  
  // Styling
  className?: string;
}
```

### Implementation Notes

**Technical Requirements:**
- Use portal rendering to avoid z-index issues
- Implement collision detection for viewport boundaries
- Calculate optimal positioning dynamically
- Handle scroll and resize events
- Clean up event listeners on unmount
- Support both controlled and uncontrolled modes
- Debounce show/hide for better UX
- Use pointer-events: none to prevent tooltip interference
- Implement proper focus management
- Support touch devices with tap interactions

**Positioning Logic:**
- Default to top placement
- Auto-adjust if insufficient space
- Maintain consistent offset from trigger
- Center align with trigger when possible
- Keep arrow pointing to trigger center
