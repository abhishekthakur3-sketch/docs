## Overview

The Search component is an input designed specifically for searching. It enables users to quickly locate information by entering keywords or phrases. Search fields often trigger dynamic suggestions, filters, or result lists via a dropdown. The system is built from three foundational elements: Search Field, Search Dropdown Cells, and Search Dropdown Lists.

## Sub-Component: Search Field

The input control where users type search queries. Supports optional filter tags for advanced search.

| Property | Options | Description |
|----------|---------|-------------|
| Variant | Standard, With Tags | Basic input or input with filter tag chips |
| State | Default, Hover, Active, Focused, Disabled, Ghost | Interaction feedback |
| Size | Default, Large | Dimensional scale |
| Leading Icon | Boolean | Search icon on the left |
| Trailing Icon | Boolean | Clear/action icon on the right |
| Text | Boolean | Placeholder or user input text |

---

## Sub-Component: Search Dropdown Cells

Individual search result items displayed inside the dropdown. Each cell may contain icons, text, or subtext.

| Property | Options | Description |
|----------|---------|-------------|
| Style | Regular, Blue | Visual differentiation for highlighted results |
| State | Default, Hover, Pressed, Disabled | Selection feedback |
| Size | Default, Large, Small | Matches search field context |
| Leading Icon | Boolean | Icon before result text |
| Trailing Icon | Boolean | Action icon after result text |
| Subtext | Boolean | Secondary description line |

---

## Sub-Component: Search Dropdown Lists

Container grouping search results into logical sections (search history, suggestions, recent items).

| Property | Options | Description |
|----------|---------|-------------|
| List Count | 1–5 Lists | Number of result groups |
| Tags | Boolean | Filter tags at the top of the dropdown |
| Size | Default, Large, Small | Matches search field size |

---

## Search Dropdown

The combined component pairing the search field with a results dropdown.

| Property | Options | Description |
|----------|---------|-------------|
| Variant | Collapsed, Expanded | Dropdown visibility state |
| Size | Default, Large, Small | Dimensional scale |

## States

| State | Visual Treatment | Behavior |
|-------|-----------------|----------|
| Default | Neutral border, placeholder text | Ready for input |
| Hover | Subtle border/background change | Cursor over the field |
| Active | User is typing, cursor visible | Input in progress |
| Focused | Focus ring visible | Keyboard navigation active |
| Disabled | 40% opacity, non-interactive | Search unavailable |
| Ghost | Skeleton/loading appearance | Data is loading |

---

## Behavior

| Interaction | Result |
|-------------|--------|
| Click search field | Places cursor, ready for input |
| Type query | Triggers suggestions/results dynamically |
| Enter key | Confirms search query |
| Clear input | Resets search results |
| Arrow keys (in dropdown) | Navigate between results |
| Click result | Selects and applies the result |
| Click outside | Closes dropdown |
| Escape | Closes dropdown |

## Variants

| Variant | Description | Use When |
|---------|-------------|----------|
| Standard | Basic search input for entering queries | Global search, simple dataset filtering |
| With Tags | Includes filter tag chips alongside the query | Advanced filtering, multi-category search |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Provide clear placeholder text (e.g., "Search...")
    - Use a search icon to indicate purpose
    - Support keyboard input and navigation
    - Allow users to clear the search easily
    - Group dropdown results logically
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't replace labels with placeholder text only
    - Don't hide search results without feedback
    - Don't overload search fields with too many controls
    - Don't display too many results at once
    - Don't use for non-search input purposes
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="search"` on the container, `role="listbox"` on dropdown |
| Keyboard | Arrow keys navigate results, Enter selects, Escape closes |
| Focus | Visible focus ring on input and active result |
| Screen reader | Announce result count, selected result, expanded/collapsed state |
| Label | Associate with `aria-label="Search"` or visible `<label>` |
| Live region | Use `aria-live="polite"` for dynamic result updates |

## Developer Handoff

<CodeGroup>
```tsx API
interface SearchFieldProps {
  variant?: 'standard' | 'with_tags';
  size?: 'lg' | 'md' | 'sm';
  state?: 'default' | 'hover' | 'active' | 'focused' | 'disabled' | 'ghost';
  placeholder?: string;
  value?: string;
  tags?: string[];
  onChange?: (value: string) => void;
  onSearch?: (query: string) => void;
  onClear?: () => void;
  disabled?: boolean;
  leadingIcon?: boolean;
  trailingIcon?: boolean;
  suggestions?: SearchResult[];
  showDropdown?: boolean;
}

interface SearchResult {
  value: string;
  label: string;
  subtext?: string;
  leadingIcon?: ReactNode;
  trailingIcon?: ReactNode;
  cellStyle?: 'regular' | 'blue';
  disabled?: boolean;
}

interface SearchDropdownProps {
  variant?: 'collapsed' | 'expanded';
  size?: 'lg' | 'md' | 'sm';
  results?: SearchResultGroup[];
  tags?: string[];
  onSelect?: (result: SearchResult) => void;
}

interface SearchResultGroup {
  label?: string;
  icon?: ReactNode;
  items: SearchResult[];
}
```

```tsx Usage
// Basic search
<SearchField
  placeholder="Search..."
  onSearch={(query) => console.log(query)}
/>

// With tags
<SearchField
  variant="with_tags"
  tags={['Shipments', 'Active']}
  placeholder="Search orders..."
/>

// With dropdown suggestions
<SearchField
  placeholder="Search..."
  suggestions={[
    { value: '1', label: 'Order #12345', subtext: 'Delhi → Mumbai' },
    { value: '2', label: 'Order #67890', subtext: 'Bangalore → Chennai' },
  ]}
  showDropdown
/>

// Large size
<SearchField size="lg" placeholder="What are you looking for?" />

// Disabled
<SearchField disabled placeholder="Search unavailable" />
```
</CodeGroup>
