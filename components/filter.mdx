## Overview

The Filter system provides structured data filtering across web and mobile platforms. It includes Filter Inputs (trigger buttons), Filter List Cells (selectable options), Filter Lists (grouped options), Filter Dropdowns (collapsed/expanded selectors), and a Filter Set Bottom Sheet for mobile. Together they enable users to refine data views efficiently.

## Sub-Components

| Component | Description |
|-----------|-------------|
| Filter Input | Trigger button with leading icon, label, and trailing chevron |
| Filter List Cell | Individual selectable option within a filter list |
| Filter List | Grouped list of filter options (2-6 items) |
| Filter Dropdown | Combined input + list as a collapsible dropdown |
| Filter Set Bottom Sheet | Mobile full-screen filter panel with tabs |
| Filter Set Mobile | Tab-based filter categories for mobile |

## Filter Input States

| State | Description |
|-------|-------------|
| Default | Neutral, ready for interaction |
| Hover | Pointer over the input |
| Active | Filter has active selections |
| Focused | Keyboard focus active |
| Disabled | Filter unavailable |
| Ghost | Skeleton/loading state |

## Behavior

| Interaction | Result |
|-------------|--------|
| Click filter input | Expands dropdown list |
| Select an option | Marks option as active, updates input |
| Click outside | Closes dropdown |
| Clear filter | Removes active selection |
| Mobile: tap filter | Opens bottom sheet with tab categories |
| Mobile: select tab | Shows filter options for that category |
| Mobile: apply | Closes bottom sheet, applies filters |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use filters for narrowing large data sets
    - Keep filter labels short and descriptive
    - Show active filter count or selection
    - Use bottom sheet for mobile filter experiences
    - Allow clearing all filters easily
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't overload with too many filter options
    - Don't hide critical filters behind secondary menus
    - Don't use filters for navigation purposes
    - Don't mix filter and sort controls without clear separation
    - Don't use filters when the data set is small
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="listbox"` on filter list, `role="option"` on cells |
| Keyboard | Arrow keys navigate options, Enter selects, Escape closes |
| Focus | Visible focus ring on input and active option |
| Screen reader | Announce active filter count and selected options |
| Label | Associate each filter with descriptive `aria-label` |

## Developer Handoff

<CodeGroup>
```tsx API
interface FilterDropdownProps {
  variant?: 'collapsed' | 'expanded' | 'active';
  size?: 'lg' | 'sm';
  label: string;
  options: FilterOption[];
  selectedValues?: string[];
  onChange?: (values: string[]) => void;
  disabled?: boolean;
}

interface FilterOption {
  value: string;
  label: string;
  disabled?: boolean;
}

interface FilterBottomSheetProps {
  open: boolean;
  onClose: () => void;
  categories: FilterCategory[];
  onApply: (filters: Record<string, string[]>) => void;
}

interface FilterCategory {
  label: string;
  options: FilterOption[];
}
```

```tsx Usage
// Single filter dropdown
<FilterDropdown
  label="Status"
  options={[
    { value: 'active', label: 'Active' },
    { value: 'pending', label: 'Pending' },
    { value: 'completed', label: 'Completed' },
  ]}
  onChange={(vals) => setFilters(vals)}
/>

// Active filter with selection
<FilterDropdown
  variant="active"
  label="City"
  selectedValues={['delhi']}
  options={cityOptions}
/>

// Mobile bottom sheet
<FilterBottomSheet
  open={showFilters}
  onClose={() => setShowFilters(false)}
  categories={[
    { label: 'Status', options: statusOptions },
    { label: 'City', options: cityOptions },
    { label: 'Date', options: dateOptions },
  ]}
  onApply={handleApplyFilters}
/>
```
