## Overview

Progress bars are visual indicators used to represent the progress of ongoing tasks, processes, or workflows. They help users understand how much of an operation has been completed and how much remains. By providing visual feedback, progress bars reduce uncertainty during uploads, downloads, data processing, and system operations.

## Types

| Type | Height | Visual | Use When |
|------|--------|--------|----------|
| Filled | 8px (Large), 4px (Small) | Thicker bar, stronger emphasis | Page loading, major task indicators |
| Line | 8px (Large), 4px (Small) | Thinner indicator on track | Compact interfaces, subtle progress |

---

## Variants

| Variant | Color | Usage |
|---------|-------|-------|
| Black | `#000000` | Neutral progress, most interfaces |
| Coal | `#3D445C` | Soft neutral, subtle system feedback |
| Blue | `#2396FB` | Informational, primary progress states |
| Green | `#1BA86E` | Positive, successful completion |
| DLV Red | `#ED1B36` | Brand-specific, strong visual emphasis |

---

## Sizes

| Size | Track Height | Border Radius | Usage |
|------|-------------|---------------|-------|
| Large | 8px | 4px | Prominent areas — page loading, major tasks |
| Small | 4px | 2px | Compact — cards, list items, dense layouts |

---

## Progress States

| Value | Description | Visual |
|-------|-------------|--------|
| 0% | Task started, no visible progress | Empty track |
| 25% | Early stage | Quarter filled |
| 50% | Halfway completed | Half filled |
| 75% | Nearing completion | Three-quarters filled |
| 100% | Process completed | Fully filled |

---

## Behavior

| Behavior | Description |
|----------|-------------|
| Direction | Progress expands left to right (LTR); right to left in RTL |
| Animation | Smooth width transition (300ms ease-out) on value change |
| Completion | Bar reaches full width at 100%; may transition to success state |
| Indeterminate | For unknown duration, use a shimmer/pulse animation instead |
| Update frequency | Update at meaningful intervals; avoid rapid micro-updates |

---

## Progress Bar vs Spinner vs Skeleton

| Component | Use When | Progress Known | Duration |
|-----------|----------|---------------|----------|
| Progress Bar | Measurable task progress | Yes | Medium–long |
| Spinner | Unknown duration, brief wait | No | Short |
| Skeleton | Content loading, layout preview | No | Short–medium |

---

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Use for measurable processes with known progress
    - Update smoothly as the task progresses
    - Place near the associated content
    - Provide clear contrast between track and indicator
    - Add percentage label for long operations
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use when progress cannot be estimated
    - Don't display misleading progress values
    - Don't use for very short operations (under 1 second)
    - Don't overuse across the interface
    - Don't rely solely on color — provide text alternative
  </Card>
</CardGroup>

---

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="progressbar"` |
| Value | Set `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"` |
| Label | Add `aria-label="Upload progress"` or `aria-labelledby` |
| Contrast | Indicator ≥ 3:1 contrast against track |
| Screen reader | Announce progress at meaningful intervals (25% increments) |
| Indeterminate | Omit `aria-valuenow` for indeterminate state |
| Color independence | Don't rely on color alone — provide text percentage |

## Developer Handoff

<CodeGroup>
```tsx API
interface ProgressBarProps {
  value: number; // 0–100
  variant?: 'black' | 'coal' | 'blue' | 'green' | 'dlv-red';
  type?: 'filled' | 'line';
  size?: 'lg' | 'sm';
  label?: string;
  showPercentage?: boolean;
  indeterminate?: boolean;
}
```

```tsx Usage
// Basic progress bar
<ProgressBar value={65} variant="blue" />

// With label and percentage
<ProgressBar value={65} variant="blue" label="Uploading file..." showPercentage />

// Green completion
<ProgressBar value={100} variant="green" label="Processing complete" />

// Small line type
<ProgressBar value={40} type="line" size="sm" variant="coal" />

// Indeterminate (unknown duration)
<ProgressBar indeterminate variant="blue" />

// DLV Red brand variant
<ProgressBar value={75} variant="dlv-red" size="lg" />
```

```tsx ARIA
// Proper ARIA implementation
<div
  role="progressbar"
  aria-valuenow={65}
  aria-valuemin={0}
  aria-valuemax={100}
  aria-label="Upload progress"
>
  <div style={{ width: '65%' }} />
</div>

// Indeterminate
<div
  role="progressbar"
  aria-label="Loading..."
>
  <div className="indeterminate-animation" />
</div>
```
