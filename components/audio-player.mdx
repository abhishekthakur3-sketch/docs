## Overview

Audio Player allows users to record, preview, and manage audio inputs. It transitions smoothly between states — from idle recording to active playback — while displaying progress, duration, and action controls.

## Anatomy

| Element | Description |
|---------|-------------|
| Control Button | Mic, play, pause, or stop icon depending on state |
| Progress Bar | Shows playback or recording progress |
| Timer / Duration | Displays elapsed or total time |
| Save Action | Allows saving the recorded audio |
| Delete Action | Removes the current recording |
| Helper Text | Optional top and bottom labels for context |

---

## Variants

| Variant | Description |
|---------|-------------|
| Default | Idle state, ready to start recording |
| Recording | Active recording in progress |
| Play | Recorded audio ready to play |
| Pause | Playback is paused |
| Saved | Recording has been saved |
| Disabled | Component is non-interactive |

## States & Behaviour

<AccordionGroup>
  <Accordion title="Default → Recording">
    User taps the mic button to begin recording. The component transitions to the Recording variant, showing a live timer and waveform or progress indicator.
  </Accordion>
  <Accordion title="Recording → Play / Saved">
    When recording stops, the component transitions to the Play variant. The user can preview the audio or save it.
  </Accordion>
  <Accordion title="Play → Pause">
    During playback, tapping the control button pauses audio and transitions to the Pause variant. The progress bar retains its position.
  </Accordion>
  <Accordion title="Saved">
    Once saved, the component shows the Saved variant with options to re-play or delete.
  </Accordion>
  <Accordion title="Disabled">
    All interactions are blocked. Opacity is reduced to indicate non-interactivity.
  </Accordion>
</AccordionGroup>

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Use within forms where audio input is required
    - Always show a timer during recording and playback
    - Provide helper text to guide users on what to record
    - Allow users to delete and re-record before saving
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use for long-form audio (use a dedicated media player instead)
    - Don't hide the delete action after saving
    - Don't skip helper text in ambiguous contexts
    - Don't disable without communicating why
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Labels | All control buttons must have `aria-label` (e.g. "Start recording", "Play audio") |
| Keyboard | Full keyboard support — Tab to focus, Enter/Space to activate |
| Screen readers | Announce state changes (recording started, paused, saved) |
| Progress | Use `aria-valuenow`, `aria-valuemin`, `aria-valuemax` on the progress bar |
| Non-visual feedback | Timer and status labels must be readable by assistive tech |
| Visible states | Ensure each variant has a visually distinct appearance |

## Developer Handoff

<CodeGroup>
```tsx Component API
interface AudioPlayerProps {
  variant?: 'default' | 'recording' | 'play' | 'pause' | 'saved' | 'disabled';
  size?: 'large' | 'small';
  style?: 'light' | 'dark';
  helperTextTop?: string;
  helperTextBottom?: string;
  duration?: string;        // e.g. "00:48"
  progress?: number;        // 0–100
  onRecord?: () => void;
  onPlay?: () => void;
  onPause?: () => void;
  onSave?: () => void;
  onDelete?: () => void;
}
```

```tsx Usage
import { AudioPlayer } from '@tarmac/design-system';

// Default — ready to record
<AudioPlayer
  variant="default"
  size="large"
  style="light"
  helperTextTop="Record your instruction"
  helperTextBottom="Max 60 seconds"
  onRecord={() => startRecording()}
/>

// Playback state
<AudioPlayer
  variant="play"
  size="large"
  style="light"
  duration="00:48"
  progress={50}
  onPlay={() => playAudio()}
  onDelete={() => deleteRecording()}
/>

// Saved state
<AudioPlayer
  variant="saved"
  size="small"
  style="dark"
  duration="00:48"
  onPlay={() => playAudio()}
  onDelete={() => deleteRecording()}
/>

// Disabled
<AudioPlayer
  variant="disabled"
  size="large"
  style="light"
/>
```
</CodeGroup>
