## Overview

Avatars represent people, entities, or ownership within the interface. They provide quick visual identification, support recognition in collaborative contexts, and help humanize data-heavy experiences.

## Anatomy

| Background | Visual fill when image is not available |
| Content | Image, initials, icon, or numeric indicator |
| Status Indicator | Optional presence dot (online, idle, offline) |

---

## Variants

| Variant | Content | When to Use |
|---------|---------|-------------|
| Image | User photo | Preferred when image is available |
| Initials | 1–2 characters | Fallback when no image exists |
| Icon | Generic icon | System accounts or placeholders |
| Numeric | Count (e.g. +2) | Overflow in Avatar Groups |

## Shapes

| Shape | Usage |
|-------|-------|
| Circle | Default for people and user identities |
| Rounded Square | Structured layouts, grids, container alignment |

---

## Sizes

| Size | Dimensions | Usage |
|------|------------|-------|
| S | 24px | Compact lists, inline mentions, dense tables |
| M | 28px | Standard lists, compact navigation |
| L | 36px | Default size, user profiles |
| XL | 40px | Prominent displays, headers |
| XXL | 48px | Profile pages, hero sections |

---

## States

| State | Behavior |
|-------|----------|
| Default | Standard representation, no interaction |
| Hover | Subtle background change, pointer cursor, 150ms transition |
| Focused | Visible 2px focus ring for keyboard navigation |
| Disabled | 40–50% opacity, no hover effects, not-allowed cursor |
| Ghost | Ultra-light variant with minimal visual weight |
| Loading | Skeleton shimmer, maintains dimensions, prevents layout shift |

---

## Status Indicators

| Status | Color | Usage |
|--------|-------|-------|
| Active | Green | User is online and active |
| Inactive | Gray | User is offline |
| Idle | Yellow | User is away |
| Orange | Orange | Busy, in meeting |
| Cyan | Cyan | Available, focus mode |
| Purple | Purple | Do not disturb |
| Pink | Pink | Custom state |

Status indicator sizes: 12px (L/XL/XXL avatars), 10px (M), 8px (S).

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Provide meaningful alt text ("John Doe", not "avatar")
    - Use fallback chain: Image → Initials → Icon
    - Keep same size and shape within a context
    - Handle loading states with skeleton
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use for decorative images unrelated to identity
    - Don't mix circle and square shapes in the same list
    - Don't use low-resolution images
    - Don't skip alt text or aria-labels
  </Card>
</CardGroup>

---

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Alt text | Descriptive — include person's name, not "avatar" |
| ARIA labels | `aria-label="John Doe"` for initials/icon variants |
| Keyboard | Tab to focus, Enter/Space to activate (if interactive) |
| Focus ring | 2px visible outline with high contrast |
| Status | Communicate via `aria-label` — don't rely on color alone |
| Contrast | Initials text ≥ 4.5:1 ratio against background |

## Developer Handoff

<CodeGroup>
```tsx Component API
interface AvatarProps {
  src?: string;
  alt?: string;
  initials?: string;
  icon?: string;
  numeric?: string;
  size?: 's' | 'm' | 'l' | 'xl' | 'xxl';
  shape?: 'circle' | 'square';
  status?: 'active' | 'inactive' | 'idle' | 'orange' | 'cyan' | 'purple' | 'pink';
  state?: 'default' | 'hover' | 'focused' | 'disabled' | 'ghost';
  onClick?: () => void;
  loading?: boolean;
}

interface AvatarGroupProps {
  children: ReactNode;
  max?: number;
  size?: 's' | 'm' | 'l' | 'xl' | 'xxl';
  shape?: 'circle' | 'square';
}
```

```tsx Usage
// Image avatar with status
<Avatar src="/user.jpg" alt="John Doe" size="l" status="active" />

// Initials fallback
<Avatar initials="JD" size="l" />

// Avatar Group with overflow
<AvatarGroup max={4} size="l">
  <Avatar src="/user1.jpg" alt="User 1" />
  <Avatar src="/user2.jpg" alt="User 2" />
  <Avatar src="/user3.jpg" alt="User 3" />
  <Avatar src="/user4.jpg" alt="User 4" />
  <Avatar src="/user5.jpg" alt="User 5" />
</AvatarGroup>
```
