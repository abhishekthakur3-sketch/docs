# Avatar

## Overview

Avatars represent people, entities, or ownership within the interface. They provide quick visual identification, support recognition in collaborative contexts, and help humanize data-heavy experiences. Avatar Groups extend this by showing multiple participants together, communicating presence, scale, or participation at a glance while conserving space.

## Variants

Avatars support multiple content types to adapt to context and data availability:

### Image Avatar
User or entity photo. This is the preferred variant when images are available.

```jsx
<Avatar src="/user-photo.jpg" alt="John Doe" />
```

<Tip>Image avatars always take priority when available.</Tip>

### Initials Avatar
Fallback using 1-2 characters representing the user's name.

```jsx
<Avatar initials="JD" />
```

<Note>Initials are centered and truncated to fit size.</Note>

### Icon Avatar
Generic or role-based representation for system accounts or placeholders.

```jsx
<Avatar icon="user" />
```

<Note>Icon avatars act as neutral placeholders when no user data exists.</Note>

### Numeric Avatar
Count indicator (e.g., +2) used in grouped or overflow contexts.

```jsx
<Avatar numeric="+2" />
```

<Warning>Numeric avatars are used only in grouped or overflow contexts, typically within Avatar Groups.</Warning>

## Shapes

Avatars are available in two shape variants:

### Rounded (Circle)
Default for people and user-related entities. Provides a friendly, approachable appearance.

```jsx
<Avatar shape="circle" src="/user.jpg" />
```

### Rounded Square
Used where alignment with containers or grids is required. Provides better visual alignment in structured layouts.

```jsx
<Avatar shape="square" src="/user.jpg" />
```

## Sizes

Avatars are available in a consistent size scale:

| Size | Dimensions | Usage |
|------|------------|-------|
| **S (Small)** | 24px | Compact lists, inline mentions, dense tables |
| **M (Medium)** | 28px | Standard lists, compact navigation |
| **L (Large)** | 36px | Default size, user profiles |
| **XL (Extra Large)** | 40px | Prominent displays, headers |
| **XXL (2X Large)** | 48px | Profile pages, hero sections, featured content |

## States

All avatar types support the following states:

### Default
Standard representation with no user interaction.

### Hover
Interactive feedback where applicable.
- Subtle background color change
- Cursor changes to pointer
- Smooth transition (150ms)

### Focused (Selected)
Indicates active or chosen state.
- Visible focus ring for keyboard navigation
- 2px outline with high contrast
- Maintains accessibility standards

### Disabled
Reduced emphasis and non-interactive.
- Reduced opacity (40-50%)
- No hover effects
- Cursor shows not-allowed
- Not keyboard focusable

### Ghost
Ultra-light variant with minimal visual weight.
- Very low opacity background
- Used for secondary or tertiary contexts
- Maintains content visibility

## Interaction Behavior

### Non-Interactive (Default)
- No hover or click effects
- Static display element
- Used for identification only
- No keyboard focus

### Interactive (Clickable)
- Hover state shows clickability
- Cursor changes to pointer
- Focus ring for keyboard navigation
- Triggers profile view or action on click
- Smooth transition effects (150ms)

### Loading State
- Skeleton shimmer animation
- Maintains avatar dimensions
- Smooth transition when image loads
- Prevents layout shift

### Do's and Don'ts

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#10b981">

    - "John Doe's profile picture"
    - "Sarah Smith"
    
    **Use fallback chain**
    - Image → Initials → Icon
    
    **Maintain consistency**
    - Same size in same context
    - Consistent spacing (8px)

    - Show skeleton while loading

  <Card title="✗ Don't" icon="xmark" color="#ef4444">

    - Optimize file sizes

  </Card>

### Color Contrast Requirements

- Initials text: ≥ 4.5:1 contrast ratio (WCAG AA)
- Focus indicators: ≥ 3:1 contrast ratio
- Status indicators: Don't rely on color alone

## Status Indicators

Avatars can display status indicators to show user availability or state:

| Status | Color | Usage |
|--------|-------|-------|
| **Active** | Green | User is online and active |
| **Inactive** | Gray | User is offline |
| **Idle** | Yellow | User is away or idle |
| **Orange** | Orange | Custom status (busy, in meeting) |
| **Cyan** | Cyan | Custom status (available, focus mode) |
| **Purple** | Purple | Custom status (do not disturb) |
| **Pink** | Pink | Custom status (custom state) |

Status indicators are available in two sizes:
- **Large**: 12px diameter (for XL and XXL avatars)
- **XLarge**: 16px diameter (for XXL avatars)

```jsx
<Avatar src="/user.jpg" status="active" />
<Avatar initials="JD" status="idle" />
```

## Engineering Implementation

### Component API

```typescript
interface AvatarProps {
  // Content variants (mutually exclusive)
  src?: string;           // Image URL
  alt?: string;           // Image alt text
  initials?: string;      // 1-2 characters
  icon?: string;          // Icon identifier
  numeric?: string;       // Count (e.g., "+5")
  
  // Appearance
  size?: 's' | 'm' | 'l' | 'xl' | 'xxl';
  shape?: 'circle' | 'square';
  
  // Status indicator (optional)
  status?: 'active' | 'inactive' | 'idle' | 'orange' | 'cyan' | 'purple' | 'pink';
  
  // States
  state?: 'default' | 'hover' | 'focused' | 'disabled' | 'ghost';
  
  // Interaction
  onClick?: () => void;
  loading?: boolean;
  
  // Styling
  className?: string;
}

interface AvatarGroupProps {
  children: ReactNode;
  max?: number;           // Maximum avatars before overflow
  size?: 's' | 'm' | 'l' | 'xl' | 'xxl';
  shape?: 'circle' | 'square';
  state?: 'default' | 'hover' | 'focused' | 'disabled';
  className?: string;
}
```

### Implementation Notes

**Content Priority:**
1. Image avatars always take priority when available
2. Initials are centered and truncated to fit size
3. Numeric avatars are used only in grouped or overflow contexts
4. Icon avatars act as neutral placeholders when no user data exists

**Technical Requirements:**
- Use `<img>` with proper error handling for image avatars
- Implement lazy loading for performance optimization
- Provide graceful fallback chain: image → initials → icon
- Use CSS `object-fit: cover` for image scaling
- Implement skeleton loading state for async content
- Cache loaded images to prevent flickering

**Status Indicators:**
- Position absolutely at bottom-right of avatar
- Use 2px white border for contrast
- Size: 12px for L/XL/XXL, 10px for M, 8px for S
