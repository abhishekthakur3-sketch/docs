## Overview

Side Drawers are panel overlays that slide in from the right edge of the screen. They provide space for contextual content, detail views, forms, or secondary workflows without full-page navigation. The system supports two width variants (Narrow and Extended) across two viewport aspect ratios, with shared Header, Footer, Content Slot, optional Snackbar, and optional Tab Group sub-components.

## Variants

| Variant | Width (764px aspect) | Width (942px aspect) | Use When |
|---------|---------------------|---------------------|----------|
| Narrow | 328px | 416px | Simple detail views, compact forms |
| Extended | 464px | 596px | Complex forms, data-heavy content, multi-section layouts |

## Behavior

| Interaction | Result |
|-------------|--------|
| Drawer opens | Slides in from right, backdrop appears |
| Click close button | Slides out, returns to previous view |
| Click backdrop | Closes drawer (if dismissible) |
| Press Escape | Closes drawer |
| Content overflow | Content area scrolls independently |
| Tab navigation | Switches content sections within drawer |
| Focus trap | Focus contained within drawer while open |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use for detail views that don't require full navigation
    - Choose Narrow for simple content, Extended for complex
    - Include clear header titles and close actions
    - Use tabs for multi-section content
    - Allow content to scroll independently
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use drawers for primary navigation
    - Don't stack multiple drawers
    - Don't use Extended variant on small viewports
    - Don't remove the close/dismiss option
    - Don't use for simple confirmations (use Dialog Box)
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="dialog"` with `aria-modal="true"` |
| Label | Provide `aria-labelledby` pointing to the header title |
| Focus trap | Focus must be trapped inside the drawer while open |
| Keyboard | Escape closes, Tab cycles through elements |
| Screen reader | Announce drawer title on open |
| Return focus | Return focus to trigger element on close |

## Developer Handoff

<CodeGroup>
```tsx API
interface SideDrawerProps {
  variant?: 'narrow' | 'extended';
  open: boolean;
  onClose: () => void;
  title: string;
  subtitle?: string;
  snackbar?: ReactNode;
  tabs?: TabProps[];
  children: ReactNode;
  primaryAction?: { label: string; onClick: () => void };
  secondaryAction?: { label: string; onClick: () => void };
  dismissible?: boolean;
}
```

```tsx Usage
// Narrow drawer for order details
<SideDrawer
  variant="narrow"
  open={showDrawer}
  onClose={() => setShowDrawer(false)}
  title="Order Details"
  subtitle="Order #12345"
  primaryAction={{ label: 'Update', onClick: handleUpdate }}
  secondaryAction={{ label: 'Cancel', onClick: () => setShowDrawer(false) }}
>
  <OrderDetails orderId="12345" />
</SideDrawer>

// Extended drawer with tabs
<SideDrawer
  variant="extended"
  open={showExtended}
  onClose={() => setShowExtended(false)}
  title="Shipment Tracking"
  tabs={[
    { label: 'Timeline', content: <Timeline /> },
    { label: 'Documents', content: <Documents /> },
  ]}
  primaryAction={{ label: 'Close', onClick: () => setShowExtended(false) }}
>
  {activeTabContent}
</SideDrawer>

// With snackbar notification
<SideDrawer
  variant="narrow"
  open={showWithSnackbar}
  onClose={() => setShowWithSnackbar(false)}
  title="Edit Profile"
  snackbar={<Snackbar type="info" message="Changes auto-saved" />}
  primaryAction={{ label: 'Save', onClick: handleSave }}
>
  <ProfileForm />
</SideDrawer>
```
</CodeGroup>
