## Overview

Buttons are interactive elements used to initiate actions such as submitting data, confirming decisions, or triggering system processes. They communicate importance through visual hierarchy and help users clearly understand what action will occur next.

## Anatomy

| Element | Description |
|---------|-------------|
| Container | Clickable area with background, border, and shape |
| Label | Text describing the action — concise and action-oriented |
| Leading Icon | Optional icon before the label (add, upload, download) |
| Trailing Icon | Optional icon after the label (arrow, expand) |
| Loading Indicator | Spinner during async operations (left or right position) |
| Focus Ring | 2px outline for keyboard navigation |

---

## Variants (Color)

| Variant | Usage | Background |
|---------|-------|------------|
| Black | High-emphasis actions, strong visual prominence | `#000000` |
| White | Dark backgrounds, neutral action style | `#FFFFFF` |
| Blue | Default primary action in most interfaces | `#2396FB` |
| Success | Positive confirmations, successful operations | `#1BA86E` |
| Error | Destructive actions — delete, remove, cancel | `#DC143C` |
| Warning | Caution-required actions, user awareness | `#CF9F02` |

---

## Styles

| Style | Emphasis | Description |
|-------|----------|-------------|
| Primary (Standard) | High | Filled button for primary and high-emphasis actions |
| Secondary (Outlined) | Medium | Visible border, transparent background |
| Tertiary (Text) | Low | No container fill or border — subtle actions |

---

## Types

| Type | Description | Use Case |
|------|-------------|----------|
| Standard Button | Single-line with label, optional icons | Primary actions, form submissions |
| Icon Button | Icon only, no label | Toolbars, dense interfaces, utility actions |
| Two-Liner Button | Two lines of text for additional context | Actions requiring more explanation |
| Slider Button | Swipe-to-confirm interaction | High-stakes confirmations |

---

## Sizes

| Size | Height | Usage |
|------|--------|-------|
| XLarge | 48px | High prominence, important CTAs |
| Large | 44px | Primary actions, touch-friendly interfaces |
| Regular | 36px | Default size for most interfaces |
| Small | 28px | Dense layouts, toolbars, compact panels |
| XSmall | 24px | Tables, utility controls, highly compact areas |

---

## States

| State | Behavior |
|-------|----------|
| Default | Resting state, ready for interaction |
| Hover | Background darkens, pointer cursor, 150ms transition |
| Pressed | Darker than hover, reduced elevation |
| Focused | 2px focus ring with high contrast for keyboard nav |
| Disabled | 40% opacity, not-allowed cursor, not focusable |
| Ghost | Minimal visual presence for subtle/secondary contexts |
| Loading | Spinner replaces/accompanies label, button disabled during processing |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Use clear, action-oriented labels ("Save", "Delete", "Create")
    - Maintain one primary button per section
    - Provide loading states for async operations
    - Ensure 44px minimum touch target on mobile
    - Keep labels to 1–3 words
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use vague labels ("Click Here", "Submit", "OK")
    - Don't use multiple primary buttons in the same context
    - Don't hide critical actions in low-emphasis styles
    - Don't disable without explaining why
    - Don't write long sentences inside button labels
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Semantic HTML | Always use native `<button>` element |
| Keyboard | Tab to focus, Enter/Space to activate, Escape to close popover |
| Focus ring | 2px visible outline, ≥ 3:1 contrast ratio |
| Contrast | Text ≥ 4.5:1 against background (WCAG AA) |
| Touch target | Minimum 44×44px on mobile |
| Icon-only | Must have `aria-label` describing the action |
| Loading | Use `aria-busy="true"` during async operations |
| Disabled | Not in tab order, visually communicates unavailability |

## Developer Handoff

<CodeGroup>
```tsx Component API
interface ButtonProps {
  variant?: 'black' | 'white' | 'blue' | 'success' | 'error' | 'warning';
  style?: 'primary' | 'secondary' | 'tertiary';
  size?: 'xl' | 'lg' | 'md' | 'sm' | 'xs';
  type?: 'button' | 'submit' | 'reset';
  disabled?: boolean;
  loading?: boolean;
  loadingPosition?: 'left' | 'right';
  icon?: ReactNode;
  iconPosition?: 'left' | 'right';
  fullWidth?: boolean;
  onClick?: (event: MouseEvent) => void;
  children: ReactNode;
}
```

```tsx Usage
// Primary black button
<Button variant="black" style="primary" size="lg">
  Save Changes
</Button>

// Secondary outlined
<Button variant="black" style="secondary">Cancel</Button>

// Error destructive
<Button variant="error" style="primary">Delete</Button>

// With loading
<Button variant="blue" loading loadingPosition="left">
  Saving...
</Button>

// Icon button
<Button variant="black" style="primary" icon={<PlusIcon />} />
```
</CodeGroup>
