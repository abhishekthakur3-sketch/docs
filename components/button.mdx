## Overview

Buttons are interactive elements used to initiate actions such as submitting data, confirming decisions, or triggering system processes. They communicate importance through visual hierarchy and help users clearly understand what action will occur next.

## States Reference

| State | Visual Change | Interaction |
|-------|--------------|-------------|
| Default | Standard appearance | Ready for interaction |
| Hover | Background darkens | Pointer cursor, 150ms transition |
| Pressed | Darker than hover | Active click/tap feedback |
| Focused | 2px focus ring | Keyboard navigation (Tab) |
| Disabled | 40% opacity | Not-allowed cursor, not focusable |
| Ghost | Minimal bg (5% opacity) | Subtle secondary contexts |
| Loading | Spinner + disabled | Async operation in progress |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Use clear, action-oriented labels ("Save", "Delete", "Create")
    - Maintain one primary button per section
    - Provide loading states for async operations
    - Ensure 44px minimum touch target on mobile
    - Use icons only when they improve clarity
    - Keep labels to 1–3 words
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use vague labels ("Click Here", "Submit", "OK")
    - Don't use multiple primary buttons in the same context
    - Don't write long sentences inside button labels
    - Don't hide critical actions in low-emphasis styles
    - Don't disable without explaining why
    - Don't use disabled buttons without communicating the reason
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Semantic HTML | Always use native `<button>` element |
| Keyboard | Tab to focus, Enter/Space to activate |
| Focus ring | 2px visible outline, ≥ 3:1 contrast |
| Text contrast | ≥ 4.5:1 against background (WCAG AA) |
| Touch target | Minimum 44×44px on mobile |
| Icon-only | Must have `aria-label` |
| Loading | Use `aria-busy="true"` |
| Disabled | Not in tab order |

## Developer Handoff

<CodeGroup>
```tsx Component API
interface ButtonProps {
  variant?: 'black' | 'white' | 'blue' | 'success' | 'error' | 'warning';
  style?: 'primary' | 'secondary' | 'tertiary';
  size?: 'xl' | 'lg' | 'md' | 'sm' | 'xs';
  type?: 'button' | 'submit' | 'reset';
  disabled?: boolean;
  loading?: boolean;
  loadingPosition?: 'left' | 'right';
  icon?: ReactNode;
  iconPosition?: 'left' | 'right';
  fullWidth?: boolean;
  onClick?: (event: MouseEvent) => void;
  children: ReactNode;
}
```

```tsx Usage Examples
// Primary black
<Button variant="black" style="primary" size="lg">Save Changes</Button>

// Secondary outlined blue
<Button variant="blue" style="secondary">Cancel</Button>

// Error destructive
<Button variant="error" style="primary">Delete Account</Button>

// With loading
<Button variant="blue" loading loadingPosition="left">Saving...</Button>

// Icon button
<Button variant="black" style="primary" icon={<PlusIcon />} aria-label="Add item" />

// Two-liner (custom)
<Button variant="black" size="xl">
  <span>Confirm Order</span>
  <span className="text-xs opacity-70">3 items · ₹2,450</span>
</Button>

// Full width
<Button variant="black" style="primary" fullWidth>Continue</Button>
```
