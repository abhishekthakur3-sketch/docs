## Overview

Skeleton loaders, also known as shimmer loaders, are visual placeholders that mimic the structure of the content being loaded. Instead of showing blank space or spinners, skeletons give users a preview of the layout while data is loading. They improve perceived performance by showing the structure of upcoming content.

## Animation Types

| Type | Description | Use When |
|------|-------------|----------|
| Animated (Shimmer) | Moving shimmer effect sweeps across the placeholder | Default for most loading states — communicates active loading |
| Static | Skeleton remains without animation | Motion needs to be minimized, performance constraints, or subtle loading states |

---

## Modes

| Mode | Description | Background |
|------|-------------|------------|
| Light | Neutral light gray placeholders | Light themed interfaces |
| Dark | Darker neutral tone placeholders | Dark themed interfaces |

## Common Patterns

Skeleton loaders should match the structure of the content they represent.

| Pattern | Description | Use Case |
|---------|-------------|----------|
| Text Lines | Horizontal bars of varying widths | Loading text content, paragraphs |
| Avatar Circles | Circular placeholders | User profiles, avatars |
| Card Blocks | Rectangular placeholders | Loading cards, modules |
| Table Rows | Repeating row placeholders | Loading tabular data |
| Image Placeholders | Large rectangular areas | Loading media content |

---

## Behavior

| Interaction | Result |
|-------------|--------|
| Content begins loading | Skeleton appears immediately |
| Skeleton displayed | Matches approximate layout of final content |
| Content loads | Skeleton disappears, replaced seamlessly |
| Layout shift | Should be minimal — skeleton matches final dimensions |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Match skeleton shape to the final content layout
    - Use animated shimmer for most loading states
    - Show skeletons immediately when loading begins
    - Keep skeleton dimensions close to actual content
    - Use appropriate mode (light/dark) for the theme
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use skeletons for very short loading times
    - Don't show skeletons that differ significantly from final layout
    - Don't leave skeletons visible indefinitely without feedback
    - Don't use animated shimmer when motion should be reduced
    - Don't interpret skeletons as actual content
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Screen readers | Hide skeletons from assistive technologies using `aria-hidden="true"` |
| Loading announcement | Use `aria-live="polite"` on the container to announce when content loads |
| Reduced motion | Respect `prefers-reduced-motion` — use static skeleton when enabled |
| Role | Use `role="status"` on the loading container |
| No content interpretation | Skeletons must not be read as meaningful content |

## Developer Handoff

<CodeGroup>
```tsx API
interface SkeletonProps {
  animated?: boolean;
  mode?: 'light' | 'dark';
  width?: string | number;
  height?: string | number;
  borderRadius?: string | number;
  className?: string;
  count?: number;
  circle?: boolean;
}
```

```tsx Usage
// Basic animated skeleton (default)
<Skeleton width="100%" height={48} />

// Static skeleton
<Skeleton animated={false} width="100%" height={48} />

// Dark mode
<Skeleton mode="dark" width="100%" height={48} />

// Circle (avatar placeholder)
<Skeleton circle width={40} height={40} />

// Multiple text lines
<Skeleton count={3} width="100%" height={16} />

// Card skeleton
<div style={{ display: 'flex', gap: 12 }}>
  <Skeleton circle width={48} height={48} />
  <div style={{ flex: 1 }}>
    <Skeleton width="60%" height={16} />
    <Skeleton width="80%" height={12} />
  </div>
</div>

// Table row skeleton
{Array.from({ length: 5 }).map((_, i) => (
  <Skeleton key={i} width="100%" height={44} />
))}
```
</CodeGroup>
