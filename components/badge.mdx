# Badge

## Overview

Badges are compact visual indicators used to display status, counts, categories, or labels. They draw attention to specific information and help users quickly identify important attributes or states. Badges are typically small, non-interactive elements that complement other UI components or standalone content.

## Anatomy

The Badge component consists of the following elements:

- **Container**: Rounded background that contains the badge content
- **Label**: Text or numeric content
- **Icon** (optional): Visual indicator supporting the label
- **Dot** (optional): Simple indicator without text

<Info>Most badges are non-interactive. Only use interactive badges when they trigger actions like filtering or navigation.</Info>

## Variants

### Default
Standard badge for general labeling and categorization.

```jsx
<Badge>New</Badge>
```

### Status
Indicates state or condition with semantic colors.

```jsx
<Badge variant="success">Active</Badge>
<Badge variant="warning">Pending</Badge>
<Badge variant="error">Failed</Badge>
<Badge variant="info">Draft</Badge>
```

### Count
Displays numeric values, often used for notifications.

```jsx
<Badge variant="count">5</Badge>
<Badge variant="count">99+</Badge>
```

### Dot
Minimal indicator showing presence or status without text.

```jsx
<Badge variant="dot" />
```

### Outlined
Subtle variant with border and transparent background.

```jsx
<Badge variant="outlined">Beta</Badge>
```

## Sizes

Badges are available in three sizes:

| Size | Height | Padding | Font Size | Usage |
|------|--------|---------|-----------|-------|
| **SM (Small)** | 18px | 6px horizontal | 11px | Compact spaces, inline text |
| **MD (Medium)** | 22px | 8px horizontal | 12px | Default size for most use cases |
| **LG (Large)** | 26px | 10px horizontal | 13px | Prominent displays, headers |

<Tip>Use MD (Medium) as the default size for most badges.</Tip>

## States

### Default
Standard appearance with no interaction.

### Hover (if interactive)
Subtle visual feedback for clickable badges.
- Slight background color change
- Cursor changes to pointer

### Focus (if interactive)
Visible focus ring for keyboard navigation.
- 2px focus outline with appropriate contrast

<Warning>Most badges should be non-interactive. Only make badges clickable when they trigger filtering, navigation, or other actions.</Warning>

## Interaction Behavior

### Non-Interactive (Default)
- No hover or click effects
- Static display element
- Used for status indication
- No keyboard focus

### Interactive (Optional)
- Hover state shows clickability
- Cursor changes to pointer
- Focus ring for keyboard navigation
- Triggers action on click

## Usage Guidelines

<AccordionGroup>
  <Accordion title="When to Use" icon="check">
    - Displaying status or state (active, pending, completed)
    - Showing notification counts
    - Labeling categories or tags
    - Highlighting new or updated content
    - Indicating user roles or permissions
    - Showing availability or capacity
  </Accordion>
  
  <Accordion title="When Not to Use" icon="xmark">
    - For primary actions (use Button)
    - For long text content (use Chip or Tag)
    - When color alone conveys meaning (ensure text/icon support)
    - For critical alerts (use Alert component)
  </Accordion>
</AccordionGroup>

### Do's and Don'ts

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#10b981">
    **Keep text concise**
    - "New"
    - "Active"
    - "5"
    
    **Use semantic colors**
    - Green for success/active
    - Red for error/critical
    - Yellow for warning
    
    **Position logically**
    - Near related content
    - Consistent placement
  </Card>
  
  <Card title="✗ Don't" icon="xmark" color="#ef4444">
    **Avoid long text**
    - ~~"This is a new feature"~~
    - ~~"Currently processing"~~
    
    **Don't overuse**
    - Too many badges lose impact
    - Maintain visual hierarchy
    
    **Don't rely on color alone**
    - Include text or icons
    - Ensure accessibility
  </Card>
</CardGroup>

<Tip>For counts over 99, use "99+" format to maintain consistent badge sizing.</Tip>

### Color Usage Guide

| Color | Use Case | Examples |
|-------|----------|----------|
| Gray/Neutral | Default, informational | "Beta", "Draft" |
| Blue | Informational, in progress | "Processing", "Info" |
| Green | Success, active, available | "Active", "Online", "Verified" |
| Yellow/Orange | Warning, pending | "Pending", "Attention" |
| Red | Error, critical, unavailable | "Failed", "Offline", "Error" |
| Purple | Premium, featured | "Pro", "Featured" |

## Accessibility

<Info>Badges must be accessible to all users, including those using screen readers.</Info>

### Color Contrast Requirements

<AccordionGroup>
  <Accordion title="✓ Text Contrast" icon="eye">
    - [ ] Text contrast ratio ≥ 4.5:1 (WCAG AA)
    - [ ] Don't rely solely on color to convey meaning
    - [ ] Include descriptive text with color indicators
    - [ ] Test with color blindness simulators
  </Accordion>
</AccordionGroup>

## Engineering Implementation

### Component API

```typescript
interface BadgeProps {
  // Appearance
  variant?: 'default' | 'success' | 'warning' | 'error' | 'info' | 'count' | 'dot' | 'outlined';
  size?: 'sm' | 'md' | 'lg';
  
  // Content
  children?: ReactNode;
  icon?: ReactNode;
  max?: number; // Maximum count before showing "+"
  
  // Interaction
  onClick?: () => void;
  
  // Styling
  className?: string;
  ariaLabel?: string;
}
```

### Implementation Notes

**Technical Requirements:**
- Use `<span>` for non-interactive badges
- Implement count truncation (e.g., 99+)
- Support both text and numeric content
- Handle empty/zero counts appropriately
- Use semantic color classes
- Ensure proper text alignment
- Support RTL layouts
- Optimize for small sizes with appropriate font rendering

**Count Handling:**
- Display actual count up to max value
- Show "max+" when count exceeds max (default: 99)
- Hide badge when count is 0 (optional behavior)
