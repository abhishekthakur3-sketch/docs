## Overview

Badges are lightweight components used to communicate contextual information — status, classification, or system feedback — without occupying significant visual space. They are typically attached to cards, list items, tables, or headers.

## Types

| Type | Emphasis | Visual Treatment | Use When |
|------|----------|-----------------|----------|
| Solid | Highest | Filled background, white text | Important status, primary classification |
| Subtle | Low | Light tinted background, colored text | Secondary context, supporting metadata |
| Outlined | Low | Border only, transparent background | Minimal visual weight needed |
| Disabled | None | 40% opacity, non-interactive | Status is unavailable or inactive |
| Ghost | Minimal | No background, text only | Badge should blend into interface |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Keep labels short (1–2 words)
    - Use semantic variants consistently
    - Place badges near related content
    - Use subtle styles for secondary metadata
    - Pair color with text — don't rely on color alone
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use badges for long phrases
    - Don't overuse — too many lose impact
    - Don't use multiple high-emphasis badges together
    - Don't rely solely on color for meaning
    - Don't use badges for primary actions (use Button)
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Contrast | Text ≥ 4.5:1 against background |
| Color independence | Don't rely on color alone — include text |
| Screen reader | Use `aria-label` for dot/icon-only badges |
| Semantic HTML | Use `<span role="status">` for status badges |
| Interactive | If clickable, use `<button>` with focus ring |

## Developer Handoff

<CodeGroup>
```tsx API
interface BadgeProps {
  variant?: 'black' | 'white' | 'coal' | 'dlv_red' | 'info' | 'success' | 'warning' | 'error' | 'cardbox';
  type?: 'solid' | 'subtle' | 'outlined' | 'disabled' | 'ghost';
  size?: 'lg' | 'md' | 'sm';
  icon?: ReactNode;
  iconPosition?: 'left' | 'right';
  dot?: boolean;
  children?: ReactNode;
}
```

```tsx Usage
// Solid status badges
<Badge variant="success" type="solid">Active</Badge>
<Badge variant="error" type="solid">Failed</Badge>

// Subtle
<Badge variant="warning" type="subtle">Pending</Badge>

// Outlined
<Badge variant="info" type="outlined">Draft</Badge>

// With prefix icon
<Badge variant="success" icon={<CheckIcon />}>Verified</Badge>

// With suffix icon (dismissible)
<Badge variant="black" icon={<CloseIcon />} iconPosition="right">React</Badge>

// Dot indicator
<Badge variant="success" dot>Online</Badge>

// Sizes
<Badge variant="black" size="lg">Large</Badge>
<Badge variant="black" size="sm">Small</Badge>
```
</CodeGroup>
