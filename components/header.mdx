## Overview

The Header system defines the structural hierarchy of interfaces across mobile and web platforms. It helps users understand where they are, what the page represents, and how content is organized. The system includes Mobile App Headers, Section Headers (Mobile), Web Page Headers, and Section Headers (Web).

## Mobile App Header

The mobile header anchors navigation at the top of the screen. It remains fixed while content scrolls beneath.

| Property | Options | Description |
|----------|---------|-------------|
| Style | Brand, Title + Subtext, Subtext + Title, Slots, Title + Dropdown | Center section content |
| Type | Avatar Right, Avatar Left | Avatar placement |

---

## Mobile App Header Styles

| Style | Description | Use When |
|-------|-------------|----------|
| Brand | App logo with global navigation and utility icons | Home screen, dashboard, landing pages |
| Title + Subtext | Primary title with supporting subtitle below | Main screens, contextual navigation |
| Subtext + Title | Subtitle above primary title | Contextual hierarchy emphasis |
| Slots | Flexible slot for custom center content | Custom layouts requiring non-standard content |
| Title + Dropdown | Title with dropdown selector | Location or context switching |

---

## Section Header Mobile

Organizes content sections within mobile screens.

| Property | Options | Description |
|----------|---------|-------------|
| Type | Standard | Section layout type |
| Size | Large (68px), Medium (60px), Small (48px), XSmall (40px) | Dimensional scale |

---

## Web Page Header

Introduces the entire page and provides high-level context. Appears once at the top of a page.

| Size | Description |
|------|-------------|
| XLarge | Maximum emphasis for primary landing pages |
| Large | High emphasis for dashboards and product pages |
| Medium | Standard page headers |
| Small | Compact page headers |
| XSmall | Minimal page headers for dense layouts |

---

## Section Header Web

Organizes content groups within a web page. Appears multiple times to separate content areas.

| Property | Options | Description |
|----------|---------|-------------|
| Variant | Standard, Tabs | With or without tab navigation |
| Size | Large (84px), Medium (76px), Small (68px) | Dimensional scale |

## Behavior

| Interaction | Result |
|-------------|--------|
| Tap menu icon (mobile) | Opens navigation drawer |
| Tap action icons | Triggers contextual actions |
| Tap breadcrumbs | Navigates to previous hierarchy level |
| Tap tabs | Switches between screen sections |
| Tap search | Activates search interface |
| Scroll content | Header remains fixed at top |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Keep navigation icons consistent across screens
    - Use descriptive screen titles
    - Maintain consistent spacing and alignment
    - Limit the number of actions displayed
    - Use the appropriate header style for the screen hierarchy
    - Use Web Page Header once per page
    - Use Section Headers to organize large pages
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't overcrowd the header with too many icons
    - Don't replace navigation hierarchy with unclear titles
    - Don't use inconsistent header styles between related screens
    - Don't hide important navigation controls
    - Don't use multiple page headers on a single page
    - Don't overload headers with too many actions
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Semantic hierarchy | Web Page Header uses H1, Section Header uses H2/H3 |
| Touch targets | Interactive elements must meet minimum 44x44px touch target |
| Labels | Icons must include descriptive accessibility labels |
| Keyboard | Buttons, links, and tabs must be keyboard accessible |
| Screen reader | Navigation hierarchy should be clearly announced |
| Contrast | Sufficient contrast for text and interactive elements |

## Developer Handoff

<CodeGroup>
```tsx API
// Mobile App Header
interface MobileAppHeaderProps {
  style?: 'brand' | 'title_subtext' | 'subtext_title' | 'slots' | 'title_dropdown';
  type?: 'avatar_right' | 'avatar_left';
  title?: string;
  subtitle?: string;
  leadingIcon?: ReactNode;
  actions?: ReactNode[];
  avatar?: AvatarProps;
  onMenuPress?: () => void;
}

// Section Header Mobile
interface SectionHeaderMobileProps {
  type?: 'standard';
  size?: 'lg' | 'md' | 'sm' | 'xs';
  title: string;
  subtitle?: string;
  leadingIcon?: ReactNode;
  trailingIcon?: ReactNode;
  link?: { label: string; href: string };
  badge?: BadgeProps;
}

// Web Page Header
interface WebPageHeaderProps {
  type?: 'xlarge' | 'large' | 'medium' | 'small' | 'xsmall';
  title: string;
  description?: string;
  actions?: ReactNode;
  badges?: BadgeProps[];
  pills?: PillProps[];
}

// Section Header Web
interface SectionHeaderWebProps {
  variant?: 'standard' | 'tabs';
  size?: 'lg' | 'md' | 'sm';
  title: string;
  subtitle?: string;
  titleIcon?: ReactNode;
  subtitleIcon?: ReactNode;
  actions?: ReactNode;
  badges?: BadgeProps[];
  link?: { label: string; href: string };
  tabs?: TabProps[];
  slot?: ReactNode;
}
```

```tsx Usage
// Mobile Brand Header
<MobileAppHeader
  style="brand"
  type="avatar_right"
  leadingIcon={<MenuIcon />}
  actions={[<NotificationIcon />, <CameraIcon />, <SettingsIcon />]}
  avatar={{ src: '/avatar.png', size: 'sm' }}
/>

// Mobile Title + Subtext
<MobileAppHeader
  style="title_subtext"
  title="Orders"
  subtitle="415, 8th main road, AECS Layout"
/>

// Web Page Header
<WebPageHeader
  type="large"
  title="Dashboard"
  description="Overview of your shipment activity"
  actions={<Button>Create Order</Button>}
/>

// Section Header Web with Tabs
<SectionHeaderWeb
  variant="tabs"
  size="lg"
  title="Analytics"
  subtitle="Performance metrics"
  tabs={[
    { label: 'Overview', active: true },
    { label: 'Revenue' },
    { label: 'Users' },
  ]}
/>
```
