## Overview

Status indicators are compact visual elements that communicate real-time state information. Each indicator consists of a colored dot and a text label, making status immediately scannable across tables, lists, cards, and dashboards. The color-coded system ensures users can quickly identify success, failure, warnings, and other operational states.

## Behavior

### Display

- Status indicators are non-interactive — they display state, not trigger actions
- The dot and label always appear together as a single unit
- Indicators should update in real-time when the underlying state changes
- Use consistent sizing within a single context (e.g., all rows in a table use the same size)

### Placement

- In tables: align within a dedicated "Status" column
- In cards: place near the top or in a metadata section
- In lists: align consistently on the left or right side
- Inline: use XSmall size within body text

### Transitions

- When status changes, update the dot color and label simultaneously
- Optional: brief fade transition (150ms) for smooth visual updates
- Avoid animation on the dot itself — keep it static for scannability

## Status Indicator vs Badge vs Tag

| Feature | Status Indicator | Badge | Tag |
|---------|-----------------|-------|-----|
| Purpose | Real-time state | Count or label | Categorization |
| Interactive | No | No | Sometimes (removable) |
| Visual | Dot + text | Filled pill | Filled/outlined chip |
| Best for | Process status, system state | Notification counts, labels | Filtering, categorization |
| Updates | Real-time | On event | Static or user-driven |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Always pair the dot with a text label
    - Use consistent variant colors across the product
    - Use the same size within a single table or list
    - Update indicators in real-time when state changes
    - Use semantic colors that match the status meaning
    - Place indicators in scannable, predictable positions
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use a dot without a label (color alone is not accessible)
    - Don't use status indicators as interactive elements
    - Don't mix sizes within the same table or list
    - Don't create custom colors outside the defined palette
    - Don't use status indicators for categorization (use Tags)
    - Don't animate the dot — keep it static for readability
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Color independence | Always include a text label — never rely on color alone |
| Screen readers | Use `aria-label` or visible text to announce the status |
| Contrast | Dot colors must meet 3:1 contrast ratio against background |
| Role | Use `role="status"` for live-updating indicators |
| Live regions | Add `aria-live="polite"` for real-time status changes |
| Decorative dot | The dot is decorative — hide from screen readers with `aria-hidden="true"` |

<Info>The text label is the primary accessible element. The colored dot is supplementary visual information and should be hidden from screen readers.</Info>
