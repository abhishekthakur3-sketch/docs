## Overview

Spinners provide visual feedback that the system is actively working — loading content, processing requests, or fetching data. Unlike progress bars, spinners represent an indeterminate state where the duration is unknown. They consist of a circular track with a rotating indicator segment that animates continuously until the operation completes.

## Variants

The spinner has 4 animation frame variants (01–04) representing different positions of the rotating tail. In implementation, these are rendered as a continuous CSS animation.

| Variant | Description |
|---------|-------------|
| 01 | Tail at 12 o'clock position |
| 02 | Tail at 3 o'clock position |
| 03 | Tail at 6 o'clock position |
| 04 | Tail at 9 o'clock position |

## Behavior

### Animation

- The tail rotates continuously clockwise around the track
- Animation speed: 1 full rotation per ~800ms (smooth, not jarring)
- Animation should be smooth and continuous using CSS `animation: spin`
- The spinner appears immediately when loading begins and disappears when complete

### Display Patterns

| Pattern | Description | Spinner Size |
|---------|-------------|-------------|
| Inline | Within a button or input | 16–20px |
| Card loading | Centered within a card or container | 24–32px |
| Section loading | Replaces a content section | 32–36px |
| Full page | Centered overlay on the entire viewport | 40–44px |
| Skeleton + spinner | Combined with skeleton placeholders | 24–28px |

### Timing

- Show spinner immediately after action (no delay)
- For very fast operations (<300ms), consider not showing a spinner at all
- For operations >3 seconds, pair with a text message (e.g., "Loading...")
- For operations >10 seconds, provide additional context or a cancel option

---

## Spinner vs Progress Bar

| Feature | Spinner | Progress Bar |
|---------|---------|-------------|
| Duration | Unknown (indeterminate) | Known (determinate) |
| Visual | Rotating circle | Filling bar |
| Feedback | "Something is happening" | "X% complete" |
| Best for | API calls, data fetching | File uploads, downloads |
| Size range | 16–44px | Full width |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use spinners for indeterminate loading states
    - Pair with loading text for operations >3 seconds
    - Use appropriate size based on context
    - Show spinner immediately when loading begins
    - Hide spinner as soon as content is ready
    - Center spinners within their loading container
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use spinners for operations with measurable progress
    - Don't show spinners for extremely short loading durations (<300ms)
    - Don't leave spinners active indefinitely without feedback
    - Don't rely solely on visual animation — add text for accessibility
    - Don't use oversized spinners for small loading areas
    - Don't use multiple spinners simultaneously in the same view
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Role | `role="status"` on the spinner container |
| Label | `aria-label="Loading"` or descriptive text |
| Live region | `aria-live="polite"` to announce loading state |
| Completion | Announce "Content loaded" or remove `aria-busy` when done |
| Busy state | `aria-busy="true"` on the loading container |
| Text pairing | Pair with visible "Loading..." text when possible |
| Motion | Respect `prefers-reduced-motion` — show static indicator instead |

<Info>Always pair spinners with accessible text. Screen readers cannot perceive animation — they need `aria-label` or visible text to announce the loading state.</Info>

## Developer Handoff

<CodeGroup>
```tsx API
import { Spinner } from '@tarmac/design-system';

interface SpinnerProps {
  size?: 16 | 20 | 24 | 28 | 32 | 36 | 40 | 44; // Default: 24
  color?: 'dark' | 'light';                        // Default: 'dark'
  label?: string;                                   // Accessible label
  showLabel?: boolean;                              // Show visible text
}
```

```tsx Usage
// Basic spinner
<Spinner />

// Small inline spinner (in a button)
<button disabled>
  <Spinner size={16} color="light" />
  Submitting...
</button>

// Card loading state
<div style={{ display: 'flex', justifyContent: 'center', padding: '40px' }}>
  <Spinner size={32} label="Loading orders" showLabel />
</div>

// Full page loading
<div style={{
  position: 'fixed', inset: 0,
  display: 'flex', alignItems: 'center', justifyContent: 'center',
  background: 'rgba(255,255,255,0.8)', zIndex: 50,
}}>
  <Spinner size={44} label="Loading your dashboard" showLabel />
</div>
```

```tsx ARIA
// Spinner with accessible announcement
<div role="status" aria-live="polite" aria-label="Loading">
  <svg
    width="24"
    height="24"
    viewBox="0 0 24 24"
    style={{ animation: 'spin 800ms linear infinite' }}
  >
    <circle
      cx="12" cy="12" r="10"
      stroke="#E5E5E5"
      strokeWidth="3"
      fill="none"
    />
    <circle
      cx="12" cy="12" r="10"
      stroke="#000000"
      strokeWidth="3"
      fill="none"
      strokeDasharray="31.4 31.4"
      strokeLinecap="round"
    />
  </svg>
  <span>Loading...</span>
</div>

{/* Loading container with aria-busy */}
<div aria-busy="true" aria-describedby="loading-msg">
  <Spinner size={32} />
  <span id="loading-msg">Loading content...</span>
</div>
```

```css Animation
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

/* Respect reduced motion preference */
@media (prefers-reduced-motion: reduce) {
  .spinner {
    animation: none;
    opacity: 0.7;
  }
}
```
