## Overview

Checkboxes are interactive selection controls that enable users to choose one or more items independently from a group of options. Unlike radio buttons, checkboxes allow multiple selections simultaneously. They support checked, unchecked, and indeterminate (partial) states for hierarchical selection patterns.

## States

| State | Visual | Behavior |
|-------|--------|----------|
| Default (Unchecked) | Empty container, `#D4D4D4` border | Ready for input |
| Hover | Darker border `#A3A3A3` | Cursor pointer, 150ms transition |
| Checked | Filled with variant color, white ✓ | Selected |
| Indeterminate | Filled with variant color, white — | Some children selected |
| Focused | 2px focus ring, 2px offset | Keyboard navigation (`:focus-visible`) |
| Disabled | 40% opacity, `#E5E5E5` border | Non-interactive, removed from tab order |
| Error | `#DC2626` border, error message below | Validation failed |

---

## Interaction Behavior

| Behavior | Description |
|----------|-------------|
| Toggle | Click/tap checkbox or label toggles state |
| Indeterminate logic | Parent shows — when some children checked; clicking checks all |
| Group behavior | Multiple checkboxes independent; wrap in `fieldset` + `legend` |
| Validation | Required checkboxes show error on submit if unchecked |
| Disabled | Cannot toggle; removed from tab order; 40% opacity |

---

## Checkbox vs Radio vs Toggle

| Control | Selection | Behavior | Use Case |
|---------|-----------|----------|----------|
| Checkbox | Multiple | Independent toggle per item | Multi-select filters, preferences |
| Radio Button | Single | Mutually exclusive within group | Single choice from options |
| Toggle Switch | Binary | Immediate on/off action | Feature switches, settings |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Use clear, descriptive labels
    - Group related checkboxes with headings
    - Use indeterminate state for parent-child hierarchies
    - Maintain 8px spacing between checkboxes
    - Ensure 44×44px minimum touch target
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use for single-choice selections (use radio)
    - Don't use for immediate on/off actions (use toggle)
    - Don't omit labels (use `aria-label` if no visible label)
    - Don't mix color variants within the same form
    - Don't use long scrollable checkbox lists (5–7 max)
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Semantic HTML | Use native `<input type="checkbox">` |
| Keyboard | Tab to focus, Space to toggle |
| Indeterminate | Set via ref: `aria-checked="mixed"` |
| Focus ring | 2px solid, 2px offset, ≥ 3:1 contrast |
| Screen reader | Announces: role, label, state (checked/unchecked/mixed) |
| Groups | Use `<fieldset>` + `<legend>` for checkbox groups |
| Disabled | `aria-disabled="true"`, removed from tab order |
| Error | Link error message via `aria-describedby` |

```tsx API
interface CheckboxProps {

  style?: 'box' | 'rounded';
  checked?: boolean;
  indeterminate?: boolean;

  required?: boolean;

  helperText?: string;
  error?: string;

  value?: string;

```

```tsx Usage
// Basic checkbox
<Checkbox label="Accept terms and conditions" />

// Controlled
<Checkbox label="Dark mode" checked={isDark} onChange={(e) => setIsDark(e.target.checked)} />

// Variants
<Checkbox variant="blue" label="Option A" />
<Checkbox variant="green" label="Confirmed" />

// Sizes and styles
<Checkbox size="lg" style="rounded" label="Large rounded" />
<Checkbox size="sm" label="Compact option" />

// With helper text
<Checkbox label="Marketing emails" helperText="Receive updates about new features" />

// Required with error
<Checkbox label="I accept the terms" required error="You must accept to continue" />
```

```tsx Indeterminate
function SelectAllExample() {
  const [items, setItems] = useState([false, true, false]);
  const allChecked = items.every(Boolean);
  const someChecked = items.some(Boolean);

  return (
    <fieldset>
      <legend>Notifications</legend>
      <Checkbox
        label="Select all"
        checked={allChecked}
        indeterminate={someChecked && !allChecked}
        onChange={() => setItems(items.map(() => !allChecked))}
      />
      {items.map((checked, i) => (
        <Checkbox
          key={i}
          label={`Item ${i + 1}`}
          checked={checked}
          onChange={() => {
            const next = [...items];
            next[i] = !next[i];
            setItems(next);
          }}
        />
      ))}
    </fieldset>
  );
}
```
