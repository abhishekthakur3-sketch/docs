## Overview

Steppers break complex workflows into manageable steps and provide clear progress indicators. They are composed of two foundational elements — Stepper Icon and Stepper Text — which combine to represent step status, order, and contextual information. Steppers support both horizontal and vertical orientations with 2–8 steps.

## Component Structure

The Stepper system consists of two master components:

| Component | Description |
|-----------|-------------|
| Stepper Icon | Visual marker for each step — communicates step number, completion status, or current state |
| Stepper Text | Title and optional description for each step — provides context about what the step represents |

These combine into two assembled components:

| Component | Layout | Description |
|-----------|--------|-------------|
| Stepper Horizontal | Left-to-right | Steps flow horizontally with dividers between them |
| Stepper Vertical | Top-to-bottom | Steps stack vertically with connecting lines |

## Icon Variants

| Variant | Visual Treatment | Use Case |
|---------|-----------------|----------|
| Solid | Filled circle with white text/icon | Active or completed steps |
| Outlined | Border circle with colored text | Current step (alternative) |
| Focused | Solid with focus ring | Keyboard-focused step |
| Disabled | Gray fill, muted text | Unavailable steps |
| Ghost | Minimal, no background | Ultra-lightweight contexts |

## Step Count

Steppers support 2 to 8 steps in both orientations:

| Count | Recommendation |
|-------|---------------|
| 2–4 | Ideal for most workflows |
| 5–6 | Acceptable, keep labels short |
| 7–8 | Consider simplifying the workflow |

<Warning>Keep workflows under 8 steps. If more are needed, consider breaking the process into sub-workflows or using a different navigation pattern.</Warning>

## Behavior

### Progression

- Steps progress sequentially from first to last
- Completed steps display a checkmark icon (green) and remain clickable for review
- The active step is visually highlighted with a solid icon
- Upcoming steps appear muted/disabled until reached

### Navigation

- Users typically move forward through steps sequentially
- Completed steps can be clicked to navigate back for review
- Disabled steps cannot be clicked or focused
- The divider line between steps changes color to indicate progress

### Responsive

| Breakpoint | Behavior |
|-----------|----------|
| Desktop (≥1024px) | Horizontal stepper preferred |
| Tablet (768–1023px) | Either orientation works |
| Mobile (<768px) | Vertical stepper recommended |

---

## Stepper vs Tabs vs Breadcrumbs

| Feature | Stepper | Tabs | Breadcrumbs |
|---------|---------|------|-------------|
| Purpose | Sequential workflow | Content switching | Hierarchical navigation |
| Order | Sequential (must follow) | Any order | Hierarchical path |
| Progress | Shows completion state | No progress concept | No progress concept |
| Best for | Multi-step forms, checkout | Related content sections | Page hierarchy |
| Step count | 2–8 | 2–8 | 2–8 levels |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use steppers for sequential multi-step processes
    - Keep step labels short and descriptive (1–3 words)
    - Show clear visual distinction between completed, active, and pending steps
    - Use vertical orientation on mobile
    - Allow users to navigate back to completed steps
    - Use subtext sparingly for additional context
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use steppers for non-sequential navigation
    - Don't exceed 8 steps in a single stepper
    - Don't use steppers for simple single-step actions
    - Don't hide step labels — always show text alongside icons
    - Don't allow skipping ahead to incomplete steps
    - Don't mix horizontal and vertical steppers on the same page
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Role | `role="list"` on stepper container, `role="listitem"` on each step |
| Current step | `aria-current="step"` on the active step |
| Completed | `aria-label="Step 1: Account — Completed"` |
| Disabled | `aria-disabled="true"` on unavailable steps |
| Keyboard | `Tab` to navigate between clickable steps |
| Focus | Visible focus ring on focused step icon |
| Labels | Each step must have a visible text label |
| Reading order | Steps must follow logical DOM order matching visual order |

<Info>Screen readers should announce each step's number, label, and status (e.g., "Step 1, Account, completed").</Info>

## Developer Handoff

<CodeGroup>
```tsx API
import { Stepper, Step } from '@tarmac/design-system';

interface StepperProps {
  orientation?: 'horizontal' | 'vertical';  // Default: 'horizontal'
  size?: 'large' | 'medium' | 'small';      // Default: 'large'
  activeStep: number;                        // 0-indexed active step
  children: React.ReactNode;
}

interface StepProps {
  label: string;                             // Step title (required)
  description?: string;                      // Optional subtext
  icon?: React.ReactNode;                    // Custom icon override
  iconStyle?: 'black' | 'coal' | 'red' | 'blue' | 'green'; // Default: 'black'
  completed?: boolean;                       // Mark as completed
  disabled?: boolean;                        // Mark as disabled
  onClick?: () => void;                      // Click handler (for completed steps)
}
```

```tsx Usage
// Horizontal stepper — checkout flow
<Stepper orientation="horizontal" size="large" activeStep={1}>
  <Step label="Account" description="Create account" completed />
  <Step label="Shipping" description="Enter address" />
  <Step label="Payment" description="Add payment" disabled />
  <Step label="Review" description="Confirm order" disabled />
</Stepper>

// Vertical stepper — onboarding
<Stepper orientation="vertical" size="medium" activeStep={2}>
  <Step label="Welcome" completed />
  <Step label="Profile Setup" completed />
  <Step label="Preferences" description="Choose your settings" />
  <Step label="Get Started" disabled />
</Stepper>

// With custom icon styles
<Stepper orientation="horizontal" activeStep={0}>
  <Step label="Upload" iconStyle="blue" />
  <Step label="Process" iconStyle="green" disabled />
  <Step label="Complete" iconStyle="green" disabled />
</Stepper>
```
