## Overview

Tags are compact UI elements used to represent attributes, filters, or categories associated with content. They help users quickly identify grouped information and interact with filtering or classification systems. Unlike badges, tags are often interactive and may support actions such as removal or filtering.

## States

| State | Visual | Interaction |
|-------|--------|-------------|
| Default | Normal appearance | Idle, no interaction |
| Hover | Slight background shift | Cursor over tag |
| Pressed | Darker background | Active click/tap |
| Focused | Focus ring (2px offset) | Keyboard navigation |
| Disabled | 40% opacity | Non-interactive, unavailable |
| Ghost | Text only, no background | Minimal presence |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Keep labels short (1–3 words)
    - Use consistent color types across similar contexts
    - Provide close icon for removable/filter tags
    - Use outlined style for secondary or dense layouts
    - Ensure tags are keyboard accessible
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use tags for long phrases or sentences
    - Don't mix too many color types in one group
    - Don't use tags for primary actions (use Button)
    - Don't rely solely on color for meaning
    - Don't nest tags inside other tags
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Contrast | Text ≥ 4.5:1 against background |
| Color independence | Don't rely on color alone — include text labels |
| Screen reader | Use `aria-label` for icon-only or dot-only tags |
| Removable tags | Close button needs `aria-label="Remove [tag name]"` |
| Keyboard | Tab to focus, Enter/Space to activate, Delete to remove |
| Focus ring | Visible 2px focus ring on keyboard navigation |
| Role | Use `role="option"` in filter groups, `role="listbox"` for container |

## Developer Handoff

<CodeGroup>
```tsx API
interface TagProps {
  type?: 'black' | 'white' | 'coal' | 'blue' | 'success' | 'error' | 'warning' | 'legacy_blue' | 'dlv_red';
  variant?: 'standard' | 'outlined';
  size?: 'xl' | 'lg' | 'md' | 'sm';
  state?: 'default' | 'hover' | 'pressed' | 'focused' | 'disabled' | 'ghost';
  icon?: ReactNode;
  closable?: boolean;
  onClose?: () => void;
  dot?: boolean;
  children?: ReactNode;
}
```

```tsx Usage
// Standard tags
<Tag type="black">Category</Tag>
<Tag type="success">Active</Tag>
<Tag type="error">Failed</Tag>

// Outlined style
<Tag type="blue" variant="outlined">Draft</Tag>

// Sizes
<Tag type="black" size="xl">XLarge</Tag>
<Tag type="black" size="sm">Small</Tag>

// With leading icon
<Tag type="success" icon={<CheckIcon />}>Verified</Tag>

// Removable tag with close icon
<Tag type="coal" closable onClose={() => handleRemove()}>React</Tag>

// With status dot
<Tag type="success" dot>Online</Tag>

// Disabled
<Tag type="black" state="disabled">Unavailable</Tag>

// Ghost
<Tag type="black" state="ghost">Subtle</Tag>
```

```tsx Filter Group
// Tag group for filtering
<TagGroup aria-label="Filter by status">
  <Tag type="success" closable>Active</Tag>
  <Tag type="warning" closable>Pending</Tag>
  <Tag type="error" closable>Failed</Tag>
</TagGroup>

// Selectable tags
<TagGroup selectable>
  <Tag type="blue" variant="outlined">React</Tag>
  <Tag type="blue" variant="outlined">Vue</Tag>
  <Tag type="blue" variant="outlined">Angular</Tag>
</TagGroup>
```
</CodeGroup>
