# Tags & Chips

## Overview

Tags and chips are compact UI elements that represent discrete pieces of information such as categories, attributes, filters, or user selections. They help users quickly scan, organize, and interact with content. Tags are typically read-only labels, while chips are interactive and can be selected, removed, or toggled.

## When to Use

<AccordionGroup>
  <Accordion title="When to Use" icon="check">
    - Displaying categories or labels on content items
    - Showing selected filters in a filter bar
    - Representing user input as tokens (e.g., email recipients)
    - Displaying status indicators (active, pending, archived)
    - Showing metadata like tags on articles or products
    - Enabling multi-select in compact spaces
  </Accordion>

  <Accordion title="When Not to Use" icon="xmark">
    - For primary actions (use buttons instead)
    - For navigation links (use links or tabs)
    - For toggling binary states (use toggle switches)
    - When content requires detailed descriptions (use cards)
    - For single-select from a long list (use a dropdown)
  </Accordion>
</AccordionGroup>

### Tags vs Chips vs Badges

| Element | Purpose | Interaction | Use Case |
|---------|---------|-------------|----------|
| **Tag** | Label or categorize | Read-only or removable | Content categories, metadata |
| **Chip** | Represent a selection | Selectable, removable, toggleable | Filters, input tokens |
| **Badge** | Show count or status | Non-interactive | Notification counts, status dots |

## Anatomy

The Tags & Chips component consists of the following elements:

- **Container**: The outer wrapper that defines the shape, background, and border of the tag or chip
- **Label**: The text content describing the tag or chip value
- **Leading Icon** (optional): An icon placed before the label for additional context
- **Trailing Icon / Remove Button** (optional): An icon or close button placed after the label, typically used for dismissing or removing the element
- **Avatar** (optional): A small avatar image used in user-related chips (e.g., assignee chips)

<Info>Tags and chips should maintain a compact footprint. Keep label text short and descriptive — ideally 1-3 words.</Info>

## Variants

<Tabs>
  <Tab title="Filled">
    Solid background with contrasting text. Used for high-emphasis tags that need to stand out.

    ```jsx
    <Tag variant="filled" label="Category" />
    <Tag variant="filled" color="blue" label="Active" />
    <Tag variant="filled" color="green" label="Completed" />
    <Tag variant="filled" color="red" label="Urgent" />
    ```
  </Tab>

  <Tab title="Outlined">
    Transparent background with a visible border. Used for medium-emphasis tags that blend with the layout.

    ```jsx
    <Tag variant="outlined" label="Category" />
    <Tag variant="outlined" color="blue" label="In Progress" />
    ```
  </Tab>

  <Tab title="Subtle">
    Light tinted background without a border. Used for low-emphasis tags in dense layouts.

    ```jsx
    <Tag variant="subtle" label="Draft" />
    <Tag variant="subtle" color="green" label="Published" />
    ```
  </Tab>

  <Tab title="Removable">
    Includes a close/remove button. Used when users need to dismiss or remove the tag.

    ```jsx
    <Tag
      label="JavaScript"
      removable
      onRemove={() => handleRemove('javascript')}
    />
    ```
  </Tab>

  <Tab title="Selectable (Chip)">
    Toggleable selection state. Used for filter chips or choice chips.

    ```jsx
    <Chip
      label="Electronics"
      selected={isSelected}
      onClick={() => toggleSelection('electronics')}
    />
    ```
  </Tab>

  <Tab title="With Icon">
    Includes a leading icon for additional visual context.

    ```jsx
    <Tag icon={<StarIcon />} label="Featured" />
    <Tag icon={<ClockIcon />} label="Pending" />
    ```
  </Tab>
</Tabs>

<Warning>Use variants consistently within the same context. Avoid mixing filled and outlined tags in the same group.</Warning>

## Colors

Tags support semantic color options to convey meaning:

| Color | Token | Usage |
|-------|-------|-------|
| **Default (Neutral)** | `tag-color-default` | General-purpose labels |
| **Blue** | `tag-color-blue` | Informational, in-progress states |
| **Green** | `tag-color-green` | Success, completed, active states |
| **Red** | `tag-color-red` | Error, urgent, critical states |
| **Yellow** | `tag-color-yellow` | Warning, pending, attention states |
| **Purple** | `tag-color-purple` | Special categories, premium features |
| **Gray** | `tag-color-gray` | Disabled, archived, inactive states |

<Tip>Use color meaningfully and consistently. Don't rely on color alone to convey information — always pair with descriptive label text.</Tip>

## Sizes

Tags adapt to different interface densities:

| Size | Height | Font Size | Padding | Usage |
|------|--------|-----------|---------|-------|
| **Small** | 24px | 12px | 4px 8px | Dense layouts, tables, compact lists |
| **Medium** | 32px | 14px | 6px 12px | Default size for most interfaces |
| **Large** | 40px | 16px | 8px 16px | Prominent placements, touch-friendly contexts |

<Tip>Use Medium (32px) as the default. Ensure touch targets meet the 44×44px minimum on mobile regardless of visual size.</Tip>

## States

### Default
Tag is in its resting state with standard styling applied.

### Hover
Subtle background or border change indicates the element is interactive. Cursor changes to pointer for interactive tags/chips.

### Focus
Visible focus ring appears around the tag for keyboard navigation. Focus ring meets 3:1 contrast ratio.

### Selected (Chips)
Chip displays a selected visual treatment — typically a filled background or checkmark icon — indicating it is actively chosen.

### Disabled
Tag appears with reduced opacity and cannot be interacted with. Cursor shows not-allowed.

### Removing
Brief exit animation plays when a removable tag is dismissed (150ms fade-out).

<Info>Non-interactive tags (read-only labels) do not display hover, focus, or selected states.</Info>

## Interaction Behavior

### Click/Tap
- Selectable chips toggle between selected and unselected states
- Removable tags dismiss on close button click
- Clicking the tag body (not close button) can trigger a navigation or detail action if configured

### Remove
- Close button click removes the tag from the group
- Removal triggers an `onRemove` callback
- Optionally animate out with a 150ms fade
- Focus moves to the next tag or the container after removal

### Keyboard
- Tags are focusable via Tab
- Enter or Space toggles selection on chips
- Backspace or Delete removes a focused removable tag
- Arrow keys navigate between tags in a group

### Overflow
- When tags exceed container width, wrap to the next line
- Alternatively, show a "+N more" overflow indicator
- Overflow count is interactive and reveals hidden tags

<Warning>Always provide a way to see all tags when using overflow truncation.</Warning>

### Best Practices

<Steps>
  <Step title="Keep labels short and scannable">
    Tags should be glanceable. Use 1-3 words that clearly describe the category or value.
  </Step>
  <Step title="Use consistent sizing within a group">
    All tags in the same group should use the same size variant for visual harmony.
  </Step>
  <Step title="Provide clear removal feedback">
    When a tag is removed, animate it out and shift remaining tags smoothly.
  </Step>
  <Step title="Handle overflow gracefully">
    Use a "+N more" pattern or wrap to a new line when tags exceed the container width.
  </Step>
  <Step title="Pair with accessible labels">
    Ensure each tag has a clear text label. For icon-only tags, provide an `aria-label`.
  </Step>
</Steps>

## Engineering Implementation

### Component API

```typescript
interface TagProps {
  // Content
  label: string;
  icon?: ReactNode;
  avatar?: ReactNode;

  // Appearance
  variant?: 'filled' | 'outlined' | 'subtle';
  color?: 'default' | 'blue' | 'green' | 'red' | 'yellow' | 'purple' | 'gray';
  size?: 'sm' | 'md' | 'lg';

  // Behavior
  removable?: boolean;
  disabled?: boolean;
  onRemove?: () => void;
  onClick?: () => void;

  // Styling
  className?: string;
  ariaLabel?: string;
}

interface ChipProps {
  // Content
  label: string;
  icon?: ReactNode;

  // Appearance
  variant?: 'filled' | 'outlined';
  color?: 'default' | 'blue' | 'green' | 'red' | 'yellow' | 'purple';
  size?: 'sm' | 'md' | 'lg';

  // State
  selected?: boolean;
  disabled?: boolean;

  // Behavior
  onClick?: () => void;
  onChange?: (selected: boolean) => void;

  // Styling
  className?: string;
  ariaLabel?: string;
}
```

### Implementation Notes

**Technical Requirements:**
- Use semantic HTML — `<span>` for read-only tags, `<button>` for interactive chips
- Remove button should be a separate `<button>` element within the tag
- Support controlled and uncontrolled selection for chips
- Handle keyboard navigation within tag groups using roving tabindex
- Animate removal with 150ms fade-out transition
- Ensure minimum 44×44px touch target on mobile
- Manage focus after tag removal (move to next sibling or container)

**Overflow Handling:**
- Wrap tags to new lines by default
- Support a `maxVisible` prop to truncate with "+N more" indicator
- Overflow indicator should be interactive and reveal hidden tags

```jsx Removable Tags
function FilterBar() {
  const [filters, setFilters] = useState(['JavaScript', 'React', 'TypeScript']);

  const removeFilter = (filter) => {
    setFilters(filters.filter(f => f !== filter));
  };

  return (
    <div role="group" aria-label="Active filters">
      {filters.map(filter => (
        <Tag
          key={filter}
          label={filter}
          removable
          onRemove={() => removeFilter(filter)}
        />
      ))}
    </div>
  );
}
```

```jsx Selectable Chips
function CategoryFilter() {
  const [selected, setSelected] = useState(new Set(['electronics']));

  const toggle = (category) => {
    const next = new Set(selected);
    next.has(category) ? next.delete(category) : next.add(category);
    setSelected(next);
  };

  return (
    <div role="listbox" aria-label="Filter by category" aria-multiselectable="true">
      {['Electronics', 'Clothing', 'Books', 'Home'].map(cat => (
        <Chip
          key={cat}
          label={cat}
          selected={selected.has(cat.toLowerCase())}
          onClick={() => toggle(cat.toLowerCase())}
        />
      ))}
    </div>
  );
}
```

## Design References

### Spacing and Alignment
- 4px gap between icon and label text
- 8px gap between tags in a group
- Tags align to baseline of surrounding text when inline
- Tag groups wrap naturally with consistent gap
- Remove button positioned 4px after label text

### Interaction Consistency
- All interactive tags use 150ms ease-in-out transitions
- Hover state applies subtle background change
- Focus ring appears on keyboard navigation only (`:focus-visible`)
- Touch targets maintain 44×44px minimum on mobile
- Removal animates with fade-out and scale-down (150ms)
