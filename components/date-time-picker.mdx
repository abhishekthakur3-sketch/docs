## Overview

The Date/Time Picker system provides structured date and time selection across the product. It includes Calendar views, Date Picker inputs, Time Picker inputs, and a combined Bottom Sheet for mobile. The system supports single date selection, multi-date selection, and time input with both small and large size variants.

## Sub-Components

| Component | Description |
|-----------|-------------|
| Calendar | Monthly grid view for date selection (320px width) |
| Date Picker | Input field that expands to show calendar |
| Time Picker | Input field for time selection with scroll wheels |
| Bottom Sheet | Mobile combined date/time picker overlay |
| Number Cells | Individual day cells within the calendar grid |
| Text Cells | Time value display cells (date and time text) |

---

## Calendar Variants

| Variant | Description | Use When |
|---------|-------------|----------|
| Default | No date selected, current month displayed | Initial state |
| Selected | Single date highlighted | Single date selection |
| Multi Selected | Date range with start/end highlighted | Date range selection |

---

## Behavior

| Interaction | Result |
|-------------|--------|
| Click date input | Expands calendar dropdown |
| Click time input | Expands time selector |
| Click a day cell | Selects the date, updates input |
| Navigate months | Arrow buttons cycle through months |
| Select date range | Click start date, then end date |
| Click outside | Closes expanded picker |
| Keyboard navigation | Arrow keys move between days |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use Date Picker for date-specific inputs
    - Use Time Picker for time-specific inputs
    - Use Bottom Sheet for mobile date/time selection
    - Disable unavailable dates clearly
    - Show the current date as a visual anchor
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use a calendar for selecting from a small set of dates
    - Don't hide the selected date from the input field
    - Don't allow selection of disabled dates
    - Don't mix date and time in a single picker without clear separation
    - Don't use pickers for non-temporal data
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="grid"` for calendar, `role="gridcell"` for day cells |
| Keyboard | Arrow keys navigate days, Enter selects, Escape closes |
| Focus | Visible focus ring on active day cell |
| Screen reader | Announce selected date, month/year context, disabled dates |
| Label | Associate picker with `aria-label` describing the expected input |

## Developer Handoff

<CodeGroup>
```tsx API
interface DatePickerProps {
  value?: Date;
  onChange?: (date: Date) => void;
  variant?: 'input' | 'active' | 'expanded';
  minDate?: Date;
  maxDate?: Date;
  disabled?: boolean;
  placeholder?: string;
}

interface TimePickerProps {
  value?: string;
  onChange?: (time: string) => void;
  variant?: 'input' | 'active' | 'expanded';
  format?: '12h' | '24h';
  disabled?: boolean;
}

interface CalendarProps {
  variant?: 'default' | 'selected' | 'multi_selected';
  selectedDate?: Date;
  startDate?: Date;
  endDate?: Date;
  onSelect?: (date: Date) => void;
  minDate?: Date;
  maxDate?: Date;
  size?: 'sm' | 'lg';
}
```

```tsx Usage
// Date Picker
<DatePicker
  placeholder="Select date"
  onChange={(date) => console.log(date)}
/>

// Time Picker
<TimePicker
  format="12h"
  onChange={(time) => console.log(time)}
/>

// Calendar with selection
<Calendar
  variant="selected"
  selectedDate={new Date()}

/>

<Calendar
  variant="multi_selected"
  startDate={startDate}
  endDate={endDate}
```
