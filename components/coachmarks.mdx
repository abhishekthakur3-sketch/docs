## Overview

Coachmarks are onboarding and guidance components that appear as contextual overlays pointing to UI elements. They include short descriptions, optional visuals, and navigation actions to help users move through a guided experience. Commonly used during product onboarding, feature discovery, or when introducing new functionality.

| # | Element | Required | Description |
|---|---------|----------|-------------|
| 1 | Container | Yes | Overlay surface with background, border-radius, shadow |
| 2 | Arrow | Yes | Directional pointer connecting to highlighted UI element |
| 3 | Image Area | No | Visual illustration or screenshot (Large size only) |
| 4 | Content Area | Yes | Contains title, description, badges, CTAs |
| 5 | Badge (primary) | No | Contextual tag (e.g., "New", "v2.4") |
| 6 | Badge (secondary) | No | Additional contextual tag |
| 7 | Title | Yes | Main heading describing the feature |
| 8 | Description | Yes | Supporting text explaining the feature |
| 9 | CTA Button (primary) | No | Primary action (Next, Got it, Done) |
| 10 | CTA Button (secondary) | No | Secondary action (Skip, Back) |

## Arrow Positions

| Direction | Alignments | Description |
|-----------|-----------|-------------|
| Top | Left, Mid, Right | Arrow points up from coachmark |
| Bottom | Left, Mid, Right | Arrow points down from coachmark |
| Left | Top, Mid, Bottom | Arrow points left from coachmark |
| Right | Top, Mid, Bottom | Arrow points right from coachmark |

<Tip>Choose arrow position based on the target element's location. Ensure the coachmark doesn't obscure the element it's pointing to.</Tip>

## Behavior

| Behavior | Description |
|----------|-------------|
| Trigger | Appears on page load, after action, or programmatically |
| Backdrop | Optional semi-transparent overlay to focus attention |
| Highlight | Target element remains visible above backdrop |
| Navigation | Next/Back/Skip buttons move through steps |
| Dismiss | Skip button or clicking outside dismisses the tour |
| Sequential | Steps appear one at a time in defined order |
| Persistence | Remember completed tours — don't show again |

---

## Coachmark vs Tooltip vs Modal

| Aspect | Coachmark | Tooltip | Modal |
|--------|----------|---------|-------|
| Purpose | Onboarding, feature education | Quick contextual info | Critical decisions |
| Trigger | Automatic (tour) | Hover/focus | User action |
| Duration | Until dismissed/completed | While hovering | Until user responds |
| Content | Title + description + image + CTAs | 1–2 sentences | Full content |
| Navigation | Multi-step (Next/Skip) | None | Confirm/Cancel |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Keep guidance short and focused
    - Highlight one element per step
    - Use images when they improve clarity
    - Provide clear Skip and Next actions
    - Remember completed tours (don't repeat)
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't show too many steps (max 5–7)
    - Don't use long paragraphs
    - Don't block critical UI interactions
    - Don't overuse across the interface
    - Don't use for self-explanatory features
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Keyboard | Tab to navigate CTAs; Escape to dismiss |
| Focus | Move focus to coachmark on appearance |
| Screen reader | Announce title and description via `aria-live` |
| Contrast | Text ≥ 4.5:1 against coachmark background |
| Timing | No auto-dismiss — user controls progression |
| Skip | Always provide a way to skip the entire tour |

## Developer Handoff

<CodeGroup>
```tsx API
interface CoachmarkProps {
  variant?: 'white' | 'black';
  size?: 'sm' | 'lg';
  arrow?: 'top-left' | 'top-mid' | 'top-right' | 'bottom-left' | 'bottom-mid' | 'bottom-right' | 'left-top' | 'left-mid' | 'left-bottom' | 'right-top' | 'right-mid' | 'right-bottom';
  title: string;
  description: string;
  image?: string;
  badges?: string[];
  targetRef: RefObject<HTMLElement>;
  step?: number;
  totalSteps?: number;
  onNext?: () => void;
  onSkip?: () => void;
  onBack?: () => void;
  onDone?: () => void;
  backdrop?: boolean;
}
```

```tsx Usage
// Simple coachmark
<Coachmark
  title="New feature!"
  description="Track shipments in real-time."
  arrow="bottom-left"
  targetRef={buttonRef}
  onDone={() => setShowCoachmark(false)}
/>

// Multi-step tour
<CoachmarkTour
  steps={[
    { title: "Dashboard", description: "Your overview starts here.", targetRef: dashRef, arrow: "bottom-left" },
    { title: "Shipments", description: "Track all active orders.", targetRef: shipRef, arrow: "right-top" },
    { title: "Settings", description: "Customize your preferences.", targetRef: settingsRef, arrow: "left-mid" }
  ]}
  onComplete={() => markTourComplete()}
  onSkip={() => markTourSkipped()}
/>

// Large with image
<Coachmark
  size="lg"
  variant="black"
  title="Real-time tracking"
  description="Monitor all shipments live."
  image="/onboarding/tracking.png"
  badges={["New", "v2.4"]}
  arrow="top-mid"
  targetRef={mapRef}
  backdrop
/>
```

```tsx ARIA
// Accessible coachmark implementation
<div
  role="dialog"
  aria-label="Feature tour step 1 of 3"
  aria-describedby="coachmark-desc"
>
  <h3>New feature!</h3>
  <p id="coachmark-desc">Track shipments in real-time.</p>
  <button onClick={onSkip}>Skip</button>
  <button onClick={onNext}>Next</button>
</div>
```
