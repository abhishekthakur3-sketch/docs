## Overview

Radio buttons are selection controls that allow users to choose exactly one option from a mutually exclusive group. When one radio button is selected, any previously selected option in the same group is automatically deselected. They are commonly used in forms, surveys, settings, and any context where a single choice is required.

## States

| State | Visual | Behavior |
|-------|--------|----------|
| Default (Unselected) | Empty circle, `#D4D4D4` border | Ready for input |
| Hover | Darker border `#A3A3A3` | Cursor pointer, 150ms transition |
| Selected | Variant-colored border + inner dot | Active selection |
| Focused | 2px focus ring, 2px offset | Keyboard navigation (`:focus-visible`) |
| Disabled | 40% opacity, `#E5E5E5` border | Non-interactive, removed from tab order |
| Error | `#DC2626` border, error message below group | Validation failed |

---

## Interaction Behavior

| Behavior | Description |
|----------|-------------|
| Selection | Click/tap radio or label selects it; deselects others in group |
| Keyboard — Arrow keys | Up/Left selects previous; Down/Right selects next in group |
| Keyboard — Tab | Moves focus to/from the radio group (not between radios) |
| Keyboard — Space | Selects the focused radio if not already selected |
| No deselect | Once a radio is selected, it cannot be deselected without choosing another |
| Disabled | Cannot select; removed from arrow key navigation; 40% opacity |

## Form Behavior

| Scenario | Behavior |
|----------|----------|
| Required group | Shows error state on submit if no option selected |
| Default selection | One option can be pre-selected via `defaultValue` |
| Validation timing | On submit by default; configurable to on-change |
| Error display | Red border on all radios + error message below group |
| Error clearing | Error clears when user selects any option |
| Disabled options | Individual radios can be disabled within an active group |
| Form reset | Returns to `defaultValue` or no selection |

## Radio vs Checkbox vs Toggle vs Dropdown

| Control | Selection | Options | Use Case |
|---------|-----------|---------|----------|
| Radio | Single | 2–5 visible | Mutually exclusive, all options visible |
| Checkbox | Multiple | Any count | Independent multi-select |
| Toggle | Binary | 1 | Immediate on/off |
| Dropdown | Single | 5+ | Long list, space-constrained |

---

## Edge Cases

| Scenario | Expected Behavior |
|----------|-------------------|
| Long labels | Label wraps; radio aligns to top of first line |
| Multi-line descriptions | Helper text below label, max 2 lines, muted |
| No default selection | Group starts empty; user must explicitly choose |
| Pre-selected + disabled | Shows selected state at 40% opacity; cannot change |
| Dynamic options | Preserve selection if selected option still exists |
| RTL layouts | Label appears to the left; layout mirrors |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Use for mutually exclusive choices (2–5 options)
    - Always show all options — don't hide behind interactions
    - Use clear, descriptive labels
    - Group with `fieldset` + `legend` for accessibility
    - Consider a default selection when appropriate
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't use a single radio button alone (use checkbox)
    - Don't use for multi-select (use checkboxes)
    - Don't use for 6+ options (use dropdown)
    - Don't mix variants within the same group
    - Don't allow deselecting all options in a required group
  </Card>
</CardGroup>

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Semantic HTML | Use native `<input type="radio">` with shared `name` |
| Keyboard | Arrow keys navigate within group; Tab moves to/from group |
| Focus ring | 2px solid, 2px offset, ≥ 3:1 contrast |
| Screen reader | Announces: role, label, state, position (e.g., "1 of 3") |
| Groups | Use `<fieldset>` + `<legend>` for radio groups |
| Disabled | `aria-disabled="true"`, skipped in arrow key navigation |
| Error | Link error message via `aria-describedby` on fieldset |
| Role | `role="radiogroup"` on container, `role="radio"` on each |

  helperText?: string;

  name: string;
  value?: string;
  defaultValue?: string;
  label?: string;
  helperText?: string;
  error?: string;
  required?: boolean;
  disabled?: boolean;
  orientation?: 'vertical' | 'horizontal';
