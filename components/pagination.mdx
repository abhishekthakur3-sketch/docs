## Overview

Pagination is a navigation pattern for browsing large datasets across multiple pages. It combines page number cells, navigation controls (Previous/Next), and contextual text (e.g., "Showing 200 results") into a structured bar. The system is built from four foundational elements that assemble into a complete pagination experience.

## Component Structure

Pagination is assembled from four foundational elements:

| Component | Description |
|-----------|-------------|
| Pagination Number Cell | Individual page number button — shows page number, supports active/hover/pressed/disabled states |
| Pagination Text Cell | Navigation control (Previous/Next) with optional leading/trailing icons |
| Pagination Text Group | Contextual dataset info (e.g., "Showing 200 results", "Page 3 of 8") |
| Pagination Number Cell Group | Assembled row of page numbers with prev/next arrows and ellipsis |

These combine into the published Pagination component:

| Section | Content |
|---------|---------|
| Left | Text Group — dataset context (e.g., "Showing 200 results") |
| Center | Number Cell Group — page numbers + navigation arrows |
| Right | Text Group — navigation controls (e.g., "Previous / Next") |

## Behavior

- Clicking a page number loads that page and highlights it as active
- Previous/Next controls navigate sequentially
- Previous is disabled on page 1; Next is disabled on the last page
- Ellipsis (`...`) appears when there are too many pages to display
- Pagination updates dynamically as users navigate
- The component should always show the first page, last page, and pages around the current page

---

## Pagination vs Infinite Scroll vs Load More

| Feature | Pagination | Infinite Scroll | Load More |
|---------|-----------|----------------|-----------|
| Navigation | Jump to any page | Continuous scroll | Append on click |
| URL support | Yes (page param) | Difficult | Difficult |
| Performance | Predictable | Can degrade | Moderate |
| Best for | Tables, search results | Social feeds, galleries | Content listings |
| SEO | Good | Poor | Moderate |

## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Place pagination at the bottom of the data container
    - Always highlight the active page clearly
    - Show contextual info (total results, current page)
    - Disable Previous on first page, Next on last page
    - Use ellipsis for large page counts
    - Use consistent size within a single table
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use pagination for small datasets (under 1 page)
    - Don't use pagination with infinite scroll simultaneously
    - Don't hide the current page indicator
    - Don't show all page numbers for very large datasets
    - Don't use pagination for continuous content streams
    - Don't place pagination above the data
  </Card>
</CardGroup>

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Landmark | Wrap in `<nav aria-label="Pagination">` |
| Current page | `aria-current="page"` on the active page button |
| Labels | Each page button: `aria-label="Page 3"` or `aria-label="Go to page 3"` |
| Disabled | `aria-disabled="true"` on Previous (page 1) and Next (last page) |
| Keyboard | `Tab` to navigate between controls, `Enter` to activate |
| Announcement | Screen readers should announce "Page 3 of 8" on navigation |
| List structure | Use `<ol>` for page numbers to convey sequence |

<Info>Always use `aria-current="page"` on the active page number so screen readers announce which page the user is currently viewing.</Info>

## Developer Handoff

<CodeGroup>
```tsx API
import { Pagination } from '@tarmac/design-system';

interface PaginationProps {
  currentPage: number;                      // Current active page (1-indexed)
  totalPages: number;                       // Total number of pages
  totalItems?: number;                      // Total items (for context text)
  size?: 'large' | 'medium' | 'small';     // Default: 'medium'
  style?: 'black' | 'blue' | 'red';        // Default: 'black'
  showContext?: boolean;                    // Show "Showing X results"
  showPageInfo?: boolean;                   // Show "Page X of Y"
  onPageChange: (page: number) => void;    // Page change callback
}
```

```tsx Usage
// Basic pagination
<Pagination
  currentPage={3}
  totalPages={8}
  onPageChange={(page) => setCurrentPage(page)}
/>

// With context text
<Pagination
  currentPage={1}
  totalPages={20}
  totalItems={200}
  showContext
  showPageInfo
  onPageChange={handlePageChange}
/>

// Small, blue style
<Pagination
  currentPage={5}
  totalPages={12}
  size="small"
  style="blue"
  onPageChange={handlePageChange}
/>

// In a table footer
<Table data={data}>
  {/* table rows */}
  <TableFooter>
    <Pagination
      currentPage={page}
      totalPages={Math.ceil(total / perPage)}
      totalItems={total}

      onPageChange={setPage}
    />
  </TableFooter>
</Table>
```
