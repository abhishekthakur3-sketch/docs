## Overview

Motion in TARMAC is a functional tool that guides users, provides feedback, and creates a sense of quality. Every animation serves a purpose — it's never decorative.

## Motion Philosophy

| Principle | Description |
|-----------|-------------|
| Purpose-Driven | Every animation guides attention, provides feedback, shows relationships, or maintains context. |
| Natural and Physical | Motion follows real-world physics — objects have weight, momentum, gravity, and friction. |
| Fast and Efficient | Animations are quick (150–300ms). Users can interrupt them. Motion can be reduced or disabled. |

## Core Motion Patterns

<AccordionGroup>
  <Accordion title="The Runway — Smooth Acceleration">
    Like aircraft on a runway, elements accelerate and decelerate smoothly.

    Easing: `cubic-bezier(0.4, 0.0, 0.2, 1)`

    Use for: Page transitions, modal appearances, large element movements.
  </Accordion>
  <Accordion title="The Bounce — Subtle Spring">
    Subtle spring-like motion that adds personality without distraction.

    Easing: `cubic-bezier(0.68, -0.55, 0.265, 1.55)`

    Use for: Button presses, toggle switches, success confirmations.
  </Accordion>
  <Accordion title="The Fade — Gentle Opacity">
    Gentle opacity changes that don't draw unnecessary attention.

    Easing: `ease-in-out`

    Use for: Tooltips, hover states, loading states.
  </Accordion>
  <Accordion title="The Slide — Directional Movement">
    Directional movement that shows spatial relationships.

    Easing: `cubic-bezier(0.4, 0.0, 0.2, 1)`

    Use for: Drawers, panels, dropdown menus, carousel navigation.
  </Accordion>
</AccordionGroup>

## Duration Guidelines

| Category | Duration | Examples |
|----------|----------|----------|
| Micro-interactions | 100–150ms | Hover states, focus indicators, button presses |
| Standard transitions | 200–300ms | Modal dialogs, dropdown menus, tab switches |
| Complex animations | 300–500ms | Page transitions, multi-step sequences, large movements |

## Accessibility

<Warning>
  Always respect the `prefers-reduced-motion` setting. Never use flashing, strobing, parallax scrolling, or auto-playing animations.
</Warning>

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## Testing Motion

| Check | How |
|-------|-----|
| Reduced motion | Enable `prefers-reduced-motion` in OS settings |
| Slower devices | Test on low-end hardware or throttle CPU |
| Interaction blocking | Verify animations don't block user input |
| Cross-browser | Test in Chrome, Firefox, Safari, Edge |
