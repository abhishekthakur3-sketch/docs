# Movement Metaphors

Motion in TARMAC isn't just decoration—it's a functional tool that guides users, provides feedback, and creates a sense of quality.

## Motion Philosophy

### Purpose-Driven
Every animation serves a purpose:
- **Guide attention** to important changes
- **Provide feedback** for user actions
- **Show relationships** between elements
- **Maintain context** during transitions

### Natural and Physical
Motion should feel natural, following real-world physics:
- Objects have weight and momentum
- Movement respects gravity and friction
- Transitions are smooth, not robotic

### Fast and Efficient
Motion should enhance, not slow down:
- Animations are quick (150-300ms typically)
- Users can interrupt animations
- Motion can be reduced or disabled

## Core Metaphors

### The Runway

Like aircraft on a runway, elements accelerate and decelerate smoothly.

**Easing**: `cubic-bezier(0.4, 0.0, 0.2, 1)`

**Use for**:
- Page transitions
- Modal appearances
- Large element movements

```css
transition: transform 300ms cubic-bezier(0.4, 0.0, 0.2, 1);
```

### The Bounce

Subtle spring-like motion that adds personality without distraction.

**Easing**: `cubic-bezier(0.68, -0.55, 0.265, 1.55)`

**Use for**:
- Button presses
- Toggle switches
- Success confirmations

```css
transition: transform 200ms cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

### The Fade

Gentle opacity changes that don't draw unnecessary attention.

**Easing**: `ease-in-out`

**Use for**:
- Tooltips
- Hover states
- Loading states

```css
transition: opacity 150ms ease-in-out;
```

### The Slide

Directional movement that shows spatial relationships.

**Easing**: `cubic-bezier(0.4, 0.0, 0.2, 1)`

**Use for**:
- Drawers and panels
- Dropdown menus
- Carousel navigation

```css
transition: transform 250ms cubic-bezier(0.4, 0.0, 0.2, 1);
```

## Duration Guidelines

<Tabs>
  <Tab title="Micro-interactions">
    **100-150ms**
    - Hover states
    - Focus indicators
    - Button presses
  </Tab>
  
  <Tab title="Standard Transitions">
    **200-300ms**
    - Modal dialogs
    - Dropdown menus
    - Tab switches
  </Tab>
  
  <Tab title="Complex Animations">
    **300-500ms**
    - Page transitions
    - Multi-step sequences
    - Large element movements
  </Tab>
</Tabs>

## Accessibility Considerations

### Respect User Preferences

Always respect the `prefers-reduced-motion` setting:

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### Avoid Triggers

<Warning>
  - No flashing or strobing effects
  - No parallax scrolling
  - No auto-playing animations
  - Provide pause controls for long animations
</Warning>

## Implementation

### Design Tokens

Use motion tokens for consistency:

```typescript
const motion = {
  duration: {
    fast: '150ms',
    normal: '250ms',
    slow: '350ms',
  },
  easing: {
    standard: 'cubic-bezier(0.4, 0.0, 0.2, 1)',
    decelerate: 'cubic-bezier(0.0, 0.0, 0.2, 1)',
    accelerate: 'cubic-bezier(0.4, 0.0, 1, 1)',
    bounce: 'cubic-bezier(0.68, -0.55, 0.265, 1.55)',
  },
};
```

### React Components

Use Framer Motion for complex animations:

```tsx
import { motion } from 'framer-motion';

<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.3, ease: [0.4, 0.0, 0.2, 1] }}
>
  Content
</motion.div>
```

## Testing Motion

- Test with reduced motion enabled
- Verify animations on slower devices
- Check that animations don't block interaction
- Ensure animations work across browsers
