## Voice

Our voice is consistent across all communications.

| Trait | Description |
|-------|-------------|
| Professional Yet Approachable | We're experts, but we don't talk down. We explain complex concepts clearly without oversimplifying. |
| Confident But Humble | We're proud of what we've built, but we acknowledge there's always room to improve. |
| Clear and Direct | We value clarity over cleverness. We say what we mean without jargon or fluff. |
| Helpful and Supportive | We're here to help teams succeed. Our tone is encouraging and solution-oriented. |

---

## Tone by Context

<Tabs>
  <Tab title="Documentation">
    Clear and instructional. Step-by-step guidance, technical but accessible.

    *Example: "Use the `variant` prop to change the button style."*
  </Tab>
  <Tab title="Marketing">
    Inspiring and aspirational. Benefit-focused, energetic but not hyperbolic.

    *Example: "Build better products, faster."*
  </Tab>
  <Tab title="Error Messages">
    Helpful and actionable. Never blaming, always solution-oriented.

    *Example: "The `size` prop expects 'small', 'medium', or 'large'. You provided 'tiny'."*
  </Tab>
  <Tab title="Release Notes">
    Informative and concise. Highlight impact, technical when needed.

    *Example: "Added dark mode support to all components."*
  </Tab>
</Tabs>

## Writing Guidelines

| Guideline | What It Means |
|-----------|---------------|
| Be Concise | Short sentences and paragraphs. Remove unnecessary words. Get to the point. |
| Be Specific | Use concrete examples, exact values, and measurements. Show, don't just tell. |
| Be Consistent | Same terms for same concepts. Follow established patterns. Consistent formatting. |
| Be Inclusive | Gender-neutral language. Avoid idioms that don't translate. Consider global audiences. |

## Terminology

| Use This | Not This |
|----------|----------|
| Component | Widget, element |
| Design token | Variable, constant |
| Pattern | Template, layout |
| Variant | Type, style |
| Prop | Property, attribute |

### Capitalization Rules

| Context | Format | Example |
|---------|--------|---------|
| Brand name | UPPERCASE | TARMAC |
| Component names | PascalCase | Button, TextField |
| Prop names | camelCase | variant, isDisabled |
| Token names | kebab-case | color-primary, spacing-md |

## Voice Checklist

Before publishing any content, verify:

- Is it clear and easy to understand?
- Is it helpful and actionable?
- Is it consistent with our voice?
- Is it free of jargon and buzzwords?
- Is it inclusive and accessible?
